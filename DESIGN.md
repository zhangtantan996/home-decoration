# home-decoration DESIGN.md

本文件是仓库默认设计决策层。

- `AGENTS.md` 负责开发、验证、安全和架构硬规则。
- `docs/FRONTEND_ENGINEERING_GUIDE.md` 负责前端 token、组件、样式门禁的工程落地。
- 本文件负责页面气质、视觉层级和跨端设计取向。

## 1. 设计目标

`home-decoration` 是覆盖用户、商家、管理、履约和交易的多端装修平台。默认气质是专业、温暖、克制、有秩序，不做炫技型界面。

- 用户侧：建立信任，降低决策压力，突出阶段和结果。
- 商家侧：提高执行效率，突出状态变化，减少操作噪音。
- 管理侧：强调审查、风险、状态和批量决策能力。
- 多端：业务语义一致，页面形态按端适配。

## 2. 页面类型

### 用户交易 / 订单 / 支付 / 项目流程

- 先说明当前处于哪一步，再说明下一步做什么。
- 金额、节点、风险、时间信息必须可扫描。
- 单页只保留一个主 CTA。
- 步骤条、节点卡、状态徽标、支付确认区要层次明显。

### 商家工作台 / 履约后台

- 信息密度高，但主次清楚。
- 筛选区、列表区、详情区、操作区职责分离。
- 重点状态变化不能被普通信息淹没。
- 页面适合长时间连续作业，不靠强视觉刺激吸引注意力。

### Admin 治理 / 审核 / 风控

- 风险信息优先于装饰效果。
- 审核信息支持快速对比和判断。
- 危险操作、驳回原因、异常状态必须显著。
- 列表页优先批量处理效率，详情页优先审查链路完整性。

### 营销 / 品牌 / 官网

- 首屏必须有明确主叙事，不堆等权重卡片。
- CTA 控制为一个主 CTA 和一个辅助 CTA。
- 优先用真实场景、流程和结果建立信任。
- 避免空洞大 Hero、三列卡片、一排 logo 的通用模板感。

## 3. Mobile Baseline

`mini/` 和 `mobile/` 统一采用 iOS-like 简洁风格，不为 Android 单独做 Material 版本。

默认规则：

- 白底、浅灰层级、轻边框。
- 低饱和状态色，品牌色只做重点强调。
- 少阴影，圆角克制，避免重投影和过度玻璃拟态。
- 底部主操作贴近安全区，按钮位置稳定。
- 操作路径短，反馈直接。
- 长按、左滑等手势必须有显式按钮替代。

允许的平台差异：

- 安全区、状态栏、键盘避让。
- 微信小程序和 React Native 的原生导航限制。
- 系统控件能力差异。

禁止默认引入：

- Android Material 专属大色块和强阴影。
- FAB 泛滥。
- 复杂手势作为唯一入口。
- 为了“高级感”滥用模糊、玻璃、悬浮层。

## 4. Visual System

- 视觉值来自 `shared/design-tokens/tokens.json` 和生成产物。
- 颜色先服务语义，再服务装饰。
- 标题层级必须清楚，说明文字不能比主操作更抢眼。
- 留白用于分组和层级，不为“空”而空。
- 卡片必须有信息组织用途，不把所有内容都卡片化。
- 动效只服务进入、状态变化和反馈确认；后台与流程页动效更克制。

## 5. Conflict Order

出现冲突时按以下顺序决策：

1. 业务正确性
2. 可读性与可操作性
3. 多端业务语义一致
4. 现有品牌资产与 design tokens
5. 局部视觉创意
