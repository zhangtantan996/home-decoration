## 当前问题清单

### P0 阻塞问题（必须立即解决）
1. **Tinode 服务器未部署**
   - 影响：所有 Tinode 功能无法使用
   - 解决：启动 docker-compose.tinode.yml

2. **用户同步机制缺失**
   - 影响：新用户无法使用聊天功能
   - 解决：在注册/登录时调用 SyncUserToTinode

3. **环境变量密钥未配置**
   - 影响：Token 生成失败
   - 解决：生成并配置 TINODE_UID_ENCRYPTION_KEY 和 TINODE_AUTH_TOKEN_KEY

### P1 高优先级问题
4. **前端运行时测试缺失**
   - 影响：不确定集成是否正常工作
   - 解决：Mobile/Admin 真机测试

5. **微信小程序未实现**
   - 影响：小程序用户无法使用聊天
   - 解决：2周开发周期（复用 Mobile 方案）

6. **历史消息迁移脚本缺失**
   - 影响：旧消息丢失
   - 解决：编写迁移脚本（Chat 表 → Tinode）

### P2 中优先级问题
7. **WebSocket Hub 代码未清理**
   - 影响：代码混乱，维护困难
   - 解决：延迟3个月清理（确保 Tinode 稳定后）

8. **腾讯云IM 前端未实现**
   - 影响：无法快速回滚
   - 解决：保持代码可用，但不优先开发

## 架构问题分析

### 1. 双数据库架构风险
**现状**：
```
home_decoration DB (业务数据)
    ↓
tinode DB (聊天数据)
```

**问题**：
- ❌ 跨库事务复杂
- ❌ 用户同步可能失败
- ❌ 备份恢复复杂

**建议**：
- ✅ 保持双库架构（数据隔离优势大于复杂度）
- ✅ 实现用户同步重试机制
- ✅ 独立备份策略

### 2. 密钥管理风险
**现状**：
- 密钥存储在环境变量（.env 文件）
- 无密钥轮换机制
- 密钥泄露风险高

**建议**：
- ✅ 短期：使用 .env 文件（开发环境）
- ✅ 中期：迁移到 AWS Secrets Manager / Vault
- ✅ 长期：实现密钥轮换机制

### 3. 前端架构混乱
**现状**：
- Admin: TinodeService.ts 存在，但无 WebSocket 代码
- Mobile: TinodeService.ts + WebSocket 代码共存
- Mini: 完全未实现

**建议**：
- ✅ 创建 IM 抽象适配层（IMServiceInterface）
- ✅ 工厂模式切换 Tinode / 腾讯云IM
- ✅ 统一三端实现模式
