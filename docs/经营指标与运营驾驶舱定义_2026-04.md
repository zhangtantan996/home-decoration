# 经营指标与运营驾驶舱定义（2026-04）

> 依赖真相源： [产品需求文档(PRD).md](./产品需求文档(PRD).md)、[BUSINESS_FLOW.md](./BUSINESS_FLOW.md)

## 1. 结论

当前阶段不需要一个“什么都看”的大而全 BI，而需要一套能支撑经营判断的统一口径。

本文件解决 4 个问题：

- 北极星到底看什么
- 一级经营看板该怎么分
- 埋点与业务真相谁说了算
- 周报、复盘、运营分析中心应该围绕哪些指标展开

## 2. 北极星口径

### 2.1 唯一北极星口径

当前唯一北极星口径定义为：

**完工且无重大争议的项目数**

原因：

- 只看预约，不代表成交
- 只看支付，不代表履约完成
- 只看完工，不代表平台真的交付了可信结果
- 完工且无重大争议，最能代表“成交 + 履约 + 治理”同时成立

### 2.2 北极星拆解指标

为了驱动日常经营，需要同步看 5 个核心拆解指标：

- 有效预约数
- 设计确认数
- 工长确认数
- 完工项目数
- 争议率 / 退款率

这些指标不是新的北极星，而是北极星的前置和约束指标。

## 3. 六组一级经营看板

### 3.1 拉新看板

重点看：

- 进入用户数
- 留资 / 登录转化
- 预约转化
- 渠道与城市分布

### 3.2 设计成交看板

重点看：

- 有效预约数
- 量房 / 预算推进率
- 方案提交率
- 设计确认率

### 3.3 施工成交看板

重点看：

- 设计确认后进入工长选择的比例
- 工长报价提交率
- 工长确认率
- 项目创建率

### 3.4 履约质量看板

重点看：

- 开工率
- 阶段验收通过率
- 工期兑现率
- 完工率

### 3.5 供给质量看板

重点看：

- 入驻通过率
- 首次响应率
- 首次提案率
- 首次成交率
- 商家分层分布

### 3.6 售后治理看板

重点看：

- 退款申请量与退款率
- 投诉量与争议率
- 提现异常与对账异常
- 平均处理时长与积压量

## 4. 双漏斗定义

### 4.1 用户侧漏斗

统一看这条主链：

- 进入
- 留资 / 登录
- 预约
- 量房 / 预算
- 方案提交
- 设计确认
- 工长确认
- 项目创建
- 完工
- 评价 / 转介绍

### 4.2 商家侧漏斗

统一看这条供给链：

- 入驻申请
- 审核通过
- 资料补全
- 首次响应
- 首次提案
- 首次成交
- 首次完工
- 稳定履约

## 5. 真相源原则

### 5.1 前端行为事件的角色

前端行为事件只解决：

- 用户从哪里来
- 看了什么
- 卡在了哪一步
- 哪个页面或来源导致掉点

### 5.2 服务端业务事件的角色

服务端业务事件才是以下结果类指标的真相源：

- 支付
- 退款
- 提现
- 仲裁
- 项目创建
- 验收
- 完工

### 5.3 真相源使用规则

统一规则：

- 前端事件看上下文
- 服务端事件看结果真相
- 财务、争议、治理类指标一律以后端为准

## 6. 切片维度

经营分析和周报只保留当前最有用的切片维度：

- 城市
- 渠道
- 商家角色
- 服务类型
- 项目类型

这几个维度已经足够回答当前阶段最重要的问题：

- 哪个城市更容易成交
- 哪个渠道带来的是有效预约
- 哪类商家更能稳定履约
- 哪类服务和项目更容易出问题

## 7. 不建议追的假指标

以下指标当前不应作为经营判断主依据：

- 纯 PV
- 无转化的收藏 / 曝光
- 与成交和履约无关的表面活跃
- 没有业务上下文的页面停留时长

这些指标最多用来辅助解释，不能用来决定主线资源分配。

## 8. 埋点与驾驶舱使用规则

### 8.1 埋点规划怎么用本文档

后续做埋点时，先从本文件倒推：

1. 先确定北极星与一级看板
2. 再确定用户侧和商家侧漏斗节点
3. 最后才补页面行为事件

### 8.2 运营分析中心怎么用本文档

后续做 Admin 运营分析中心时，首页和模块顺序应优先对应这 6 组一级看板，而不是先堆技术报表。

### 8.3 周报复盘怎么用本文档

周报复盘必须先看：

1. 北极星及 5 个拆解指标
2. 六组一级看板的环比变化
3. 哪个漏斗掉点最严重
4. 是用户问题、供给问题，还是治理问题

## 9. 本文档以后怎么用

后续遇到下面这些问题，优先引用本文档：

- 埋点该怎么做
- 运营分析中心该看什么
- 每周复盘用什么指标
- 哪个漏斗掉点最严重
- 什么指标值得追，什么指标只是表面热闹

如果问题继续深入到交易状态和业务对象边界，再回到 [产品需求文档(PRD).md](./产品需求文档(PRD).md) 与 [BUSINESS_FLOW.md](./BUSINESS_FLOW.md)。
