# 线上域名入口现状与目标态（2026-03-20）

## 现状清单

- 服务器：`root@hezeyunchuang.com`
- 宿主机 Nginx 当前启用文件：`/etc/nginx/sites-available/home-decoration`
- 当前仅监听 `80`，未启用 `443`
- 生产 Web 容器入口：`127.0.0.1:8888 -> prod_web:8888`
- 生产 API 容器仅内网暴露：`prod_api:8080`
- 当前线上已按三域分工收敛：
  - 根域 `hezeyunchuang.com` / `www.hezeyunchuang.com`：官网 + 用户端 `/app/` + 商家端 `/merchant/`
  - 后台域 `admin.hezeyunchuang.com`：后台 `/admin/`
  - API 域 `api.hezeyunchuang.com`：API / uploads / static / Tinode
- 当前构建产物中：
  - 官网独立来自 `website/`
  - 用户端独立来自 `web/`，挂在 `/app/`
  - 商家端来自 `merchant/`
  - 后台来自 `admin/`
- 当前三套前端已统一直连 `http://api.hezeyunchuang.com/api/v1`

## 风险点

- 线上仍是 HTTP-only，正式生产流量未启用 TLS
- 浏览器若缓存旧版前端资源，可能还会请求历史相对地址 `'/api/v1'`；刷新后恢复正常

## 建议目标态

### 三域职责

- `hezeyunchuang.com` / `www.hezeyunchuang.com`
  - 承接官网、用户端 `/app/`、商家端 `/merchant/`
- `admin.hezeyunchuang.com`
  - 仅承接管理后台页面
- `api.hezeyunchuang.com`
  - 仅承接 API、uploads、static、Tinode、`/v0/*`

### 入口收敛原则

- 根域和后台域下的 `/api/*`、`/uploads/*`、`/static/*`、`/tinode/*`、`/v0/*` 统一返回 `404`
- 根域访问 `/admin/*` 时只允许跳转到 `admin.hezeyunchuang.com`
- 后台域访问 `/merchant/*` 时只允许跳转到根域
- 官网固定在根域 `/`
- 用户端固定在根域 `/app/`
- 商家端固定在根域 `/merchant/`
- 前端构建统一使用 `http://api.hezeyunchuang.com/api/v1`

## 本次执行方案

1. 备份远端宿主机 Nginx 配置与生产 Compose / 容器 Nginx 配置

- Web 容器：恢复 `deploy/docker-compose.prod.yml`、`deploy/nginx/nginx.conf`、`deploy/Dockerfile.frontend` 备份后重建 `prod_web`
- 回滚验证：
  - `http://hezeyunchuang.com/`
  - `http://hezeyunchuang.com/app/`
  - `http://admin.hezeyunchuang.com/admin/login`
  - `http://api.hezeyunchuang.com/api/v1/health`
