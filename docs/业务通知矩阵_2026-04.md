# 业务通知矩阵（2026-04）

## 目标
- 统一 `mini / web / merchant / admin` 的站内通知口径
- 固定业务节点的 `type / 接收角色 / actionUrl / 分类`
- 后续新增通知必须先对齐本矩阵，再落代码

## 分类规则
- `system`：入驻审核、案例审核、账号/系统提醒
- `project`：预约、方案、施工报价、阶段验收、完工、投诉、争议、仲裁
- `payment`：待支付、支付成功、退款、提现

## 端侧承接要求
- `mini`：必须同步 `actionUrl -> mini route` 映射；无法映射时显示“仅通知”
- `web`：通知中心直接使用 canonical path 跳转
- `merchant`：顶部下拉与 `/notifications` 页必须共用同一批类型文案
- `admin`：顶部下拉是摘要，`/notifications` 是完整处理面
