## 常见问题故障排除

### 高延迟（> 2 秒）

**可能原因**:
- 网络拥塞
- Tinode 服务器过载
- WebSocket 连接问题
- 设备性能问题

**调试步骤**:
1. 检查 Tinode 服务器日志: `docker logs decorating_tinode`
2. 检查网络延迟: `ping localhost`（如果本地）或 ping 服务器 IP
3. 在浏览器开发者工具中监控 WebSocket 帧（Network → WS）
4. 检查设备的 CPU/内存使用情况

### 消息丢失

**可能原因**:
- WebSocket 断开
- Topic 订阅问题
- 消息队列溢出
- 服务器端问题

**调试步骤**:
1. 检查两个设备的连接状态
2. 验证控制台日志中的 topic 订阅
3. 检查 Tinode 服务器消息保留设置
4. 查看消息序列号（seq）是否有间隙

### 消息重复

**可能原因**:
- 重试逻辑触发多次发送
- 消息去重不工作
- Topic 订阅问题

**调试步骤**:
1. 检查消息 seq 号（应该是唯一的）
2. 查看发送逻辑中的重试机制
3. 检查 Tinode SDK 消息缓存
