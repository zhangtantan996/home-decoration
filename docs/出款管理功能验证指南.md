## 数据结构

### 出款单字段

```typescript
interface PayoutOrder {
  id: number;                    // 出款单ID
  bizType: string;               // 业务类型
  bizId: number;                 // 业务ID
  providerId: number;            // 服务商ID
  providerName?: string;         // 服务商名称
  channel: string;               // 出款渠道
  amount: number;                // 出款金额（分）
  outPayoutNo: string;           // 平台出款单号
  providerPayoutNo?: string;     // 渠道出款单号
  status: string;                // 状态：created/processing/paid/failed
  failureReason?: string;        // 失败原因
  scheduledAt?: string;          // 计划时间
  paidAt?: string;               // 完成时间
  retryCount: number;            // 重试次数
  createdAt: string;             // 创建时间
}
```

## 状态说明

| 状态 | 说明 | 颜色 | 操作 |
|------|------|------|------|
| created | 已创建 | default | 查看详情 |
| processing | 处理中 | processing | 查看详情 |
| paid | 已完成 | success | 查看详情 |
| failed | 失败 | error | 查看详情、重试 |

## 常见问题

### Q1: 菜单中看不到"自动出款"选项
**A**: 执行 `server/scripts/add_payout_menu.sql` 脚本添加菜单配置，或重新运行 `seed_rbac_full.go`。

### Q2: 点击"重试"按钮没有反应
**A**: 检查当前登录账号是否有 `finance:transaction:approve` 权限。安全审计员和只读用户无此权限。

### Q3: 列表为空
**A**: 确认数据库中是否有出款单数据。可以通过以下SQL查询：
```sql
SELECT * FROM payout_orders ORDER BY created_at DESC LIMIT 10;
```

### Q4: API返回404
**A**: 检查后端路由是否正确注册，确认 `server/internal/router/router.go` 中包含出款相关路由。
