# 商业运营文档索引（2026-04）

## 1. 文档目的

本组文档不重写交易主链 PRD，而是在 [产品需求文档(PRD).md](./产品需求文档(PRD).md) 与 [BUSINESS_FLOW.md](./BUSINESS_FLOW.md) 之上，补齐以下判断层：

- 当前阶段到底处于哪一段
- 当前主商业模型是什么
- 小程序优先时，增长与成交该怎么打
- 商家供给该怎么运营和治理
- 经营指标、埋点、周报该看什么

一句话判断：**项目已不是“缺页面”的阶段，而是进入“收敛复杂平台、压实成交与履约闭环”的阶段。**

## 2. 文档真相优先级

### 2.1 真相优先级

1. [产品需求文档(PRD).md](./产品需求文档(PRD).md) / [BUSINESS_FLOW.md](./BUSINESS_FLOW.md)
2. 本轮 4 份商业运营专项文档
3. 现有 MVP / Web / Merchant 分析文档
4. 代码与实际运营现状补充事实

### 2.2 检索顺序

后续做商业、产品、运营建议时，默认按以下顺序查阅：

1. 先看本索引，确认问题属于哪一类
2. 再看对应专项文档
3. 如涉及交易状态、角色边界、项目创建条件，再回到 PRD / BUSINESS_FLOW
4. 如文档与实现有出入，以代码和当前发布面事实补充说明，但不能绕开 PRD 主基线

## 3. 当前阶段判断

当前阶段定义为：**平台型产品中前期**。

现阶段重点不是横向扩功能，而是收敛 4 件事：

- 主成交端是否足够强
- 设计成交到施工成交之间是否能桥接
- 商家供给是否能稳定履约
- 经营指标是否足以支撑日常运营和复盘

## 5. 本轮覆盖范围

本轮只定义以下真相：

- 主商业模型
- 小程序优先的增长与成交抓手
- 商家供给运营与治理
- 经营指标与驾驶舱口径
- 新旧战略文档如何并存不冲突

## 6. 本轮不覆盖范围

以下内容不在本轮文档体系内，不顺带扩写：

- 埋点代码实现
- Admin 运营分析中心页面实现
- 支付、IM、短信、提现代码改造
- 新增领域实体、重写状态机、改 PRD 主交易基线
- 主材商城升级为当前主经营引擎

## 7. 使用规则

### 7.1 必守基线

后续所有建议都不能推翻以下 3 条：

- 设计确认与工长确认是两个独立成交点
- 工长确认才创建项目
- 主材商当前不进入主交易闭环

### 7.2 推荐输出方式

后续如果你让我继续做建议、排期、优先级判断，默认输出结构按下面来：

1. 先给阶段判断
2. 再给是否服务主商业模型
3. 再给是补增长、补供给还是补治理
4. 最后才给实现建议或排期建议
