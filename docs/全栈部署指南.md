# 全栈项目部署指南 (Full Stack Deployment Guide)

本指南详细说明如何将 Home Decoration 项目的各个部分（后台、Web 端、管理后台、移动端 App）部署到生产环境。

## 1. 架构总览 (Architecture)

项目采用 **前后端分离** + **容器化部署** 的架构。

*   **服务端 (Server)**:
    *   运行在 Linux 服务器上。
    *   使用 Docker 容器运行 Go API 服务。
    *   使用 Docker 容器运行 PostgreSQL 和 Redis (生产环境建议使用云数据库，但本项目提供全容器化方案)。
*   **Web 前端 (Admin & Mobile Web)**:
    *   静态文件构建 (`html/css/js`)。
    *   使用 Docker 容器运行 Nginx 进行托管和反向代理。
*   **移动端 App (iOS & Android)**:
    *   构建为安装包 (`.apk`, `.ipa`)。
    *   运行在用户手机上，通过 HTTP 请求与服务端通信。

## 2. 服务端部署 (Server Deployment)

### 2.1 准备工作
*   一台 Linux 服务器 (Ubuntu/CentOS)。
*   已安装 [Docker](https://docs.docker.com/engine/install/) 和 [Docker Compose](https://docs.docker.com/compose/install/)。
*   Git。

### 2.2 部署步骤

1.  **克隆代码**:
    ```bash
    git clone https://your-repo-url/home_decoration.git
    cd home_decoration
    ```

2.  **进入部署目录**:
    ```bash
    cd deploy
    ```

3.  **配置环境变量 (可选)**:
    *   修改 `docker-compose.prod.yml` 中的环境变量（如数据库密码 `DB_PASSWORD`）。
    *   *建议*: 生产环境不要使用默认密码。

4.  **启动服务**:
    ```bash
    # 构建并后台启动
    docker-compose -f docker-compose.prod.yml up -d --build
    ```

5.  **验证**:
    *   查看容器状态: `docker-compose -f docker-compose.prod.yml ps`
    *   查看日志: `docker-compose -f docker-compose.prod.yml logs -f`

### 2.3 访问地址
*   **管理后台**: `http://<服务器IP>/admin/`
*   **H5 网页**: `http://<服务器IP>/mobile/`
*   **API 接口**: `http://<服务器IP>/api/`

## 3. 移动端 App 打包 (Mobile App Build)

移动端 App 需要单独打包成安装文件。

### 3.1 修改 API 地址
在打包前，必须将 App 的 API 请求地址指向你的线上服务器。

1.  打开 `mobile/src/services/api.ts` (或相应配置文件)。
2.  修改 `BASE_URL`:
    ```typescript
    // 将 localhost 修改为你的服务器 IP 或域名
    export const BASE_URL = 'http://<服务器IP>/api'; 
    // 如果配置了域名和 HTTPS (推荐)
    // export const BASE_URL = 'https://api.yourdomain.com';
    ```

### 3.2 Android 打包 (.apk)

1.  **生成签名密钥** (如果还没有):
    ```bash
    keytool -genkeypair -v -keystore my-upload-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
    # 将生成的 .keystore 文件放入 mobile/android/app/ 目录
    ```

2.  **配置签名**:
    *   在 `mobile/android/gradle.properties` 中添加密钥密码。
    *   在 `mobile/android/app/build.gradle` 中配置 `signingConfigs`。

3.  **构建 Release 包**:
    ```bash
    cd mobile/android
    ./gradlew assembleRelease
    ```
4.  **产物**: 文件位于 `mobile/android/app/build/outputs/apk/release/app-release.apk`。

### 3.3 iOS 打包 (.ipa)

*需要 macOS 环境和 Apple Developer 账号。*

1.  进入 `mobile/ios` 目录，安装依赖: `pod install`。
2.  使用 Xcode 打开 `HomeDecorationApp.xcworkspace`。
3.  配置 **Signing & Capabilities**，选择你的开发者团队。
4.  修改 Build Configuration 为 **Release**。
5.  在菜单栏选择 **Product > Archive**。
6.  Archive 完成后，点击 **Distribute App** 导出 IPA 或上传 TestFlight。

## 4. 常见问题 (FAQ)

*   **Q: 数据库数据如何备份？**
    *   A: 数据卷挂载在 Docker Volume `db_data_prod`。可以使用 `docker run --rm -v db_data_prod:/volume -v $(pwd):/backup alpine tar cvf /backup/backup.tar /volume` 进行备份。

*   **Q: 如何配置 HTTPS？**
    *   A: 建议在服务器上安装 Nginx (宿主机)，使用 Let's Encrypt 申请证书，然后配置反向代理指向 Docker 容器的 80 端口。或者修改 `deploy/nginx/nginx.conf` 并在容器内挂载证书。

*   **Q: App 报错主要是什么原因？**
    ```bash
    # 4. App 没有网络权限（Android Manifest 已配置）。
    ```

## 5. 进阶配置与优化

### 5.1 Android 打包特殊配置 (HTTP与JSBundle)
> 此部分摘自原《Android打包指南》

1.  **开启明文传输 (HTTP)**:
    由于服务器暂未配置 HTTPS，已在 `mobile/android/app/src/main/AndroidManifest.xml` 中配置：
    ```xml
    <application android:usesCleartextTraffic="true" ...>
    ```

2.  **强制重新打包 JS Bundle (防止代码未更新)**:
    在打包 APK 前，建议手动执行以下命令确保 JS 代码是最新的：
    ```powershell
    npx react-native bundle --platform android --dev false --entry-file index.js --bundle-output android/app/src/main/assets/index.android.bundle --assets-dest android/app/src/main/res
    ```

### 5.2 服务器内存优化 (配置 Swap)
> 此部分摘自原《Docker部署详解》。**如果服务器内存只有 2G，必须执行！**

由于 `npm build` 极其消耗内存，建议开启 4G 虚拟内存：
```bash
# 创建 4G 的 Swap 文件
dd if=/dev/zero of=/swapfile bs=1M count=4096
# 设置权限
chmod 600 /swapfile
# 格式化并启用
mkswap /swapfile
swapon /swapfile
# 永久生效
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### 5.3 日常更新流 (Git Flow)

**本地开发**:
1. `git add .`
2. `git commit -m "feat: 更新内容"`
3. `git push`

**服务器更新**:
1. `cd /data/www/home-decoration`
2. `git pull`
3. `cd deploy`
4. `docker-compose -f docker-compose.prod.yml up -d --build` (自动检测变更并重构)
