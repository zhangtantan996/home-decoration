# 对账差异处理流程实现总结

## 实现概述

补充了对账系统的差异处理完整流程，让Admin能够标记差异状态、记录处理说明，并实现了自动告警升级机制。

---

## 实现内容

### 1. 数据模型扩展

**文件**: `server/internal/model/reconciliation.go`

**新增字段**:
- `HandleStatus`: 处理状态（pending/investigating/resolved/ignored）
- `IgnoreReason`: 忽略原因
- `Solution`: 解决方案

**新增常量**:
```go
const (
    DifferenceStatusPending       = "pending"
    DifferenceStatusInvestigating = "investigating"
    DifferenceStatusResolved      = "resolved"
    DifferenceStatusIgnored       = "ignored"
)
```

---

### 2. Service层方法

**文件**: `server/internal/service/reconciliation_service.go`

**新增方法**:

1. **InvestigateDifference**: 标记差异为调查中
   - 输入: DifferenceID, AdminID, Notes
   - 更新状态为 `investigating`
   - 记录操作人和备注

2. **IgnoreDifference**: 忽略差异
   - 输入: DifferenceID, AdminID, Reason（必填）
   - 更新状态为 `ignored`
   - 记录操作人和忽略原因

3. **ResolveDifferenceEnhanced**: 解决差异（增强版）
   - 输入: DifferenceID, AdminID, Solution（必填）, Notes
   - 更新状态为 `resolved`
   - 标记 `resolved=true`
   - 记录解决时间、操作人、解决方案和处理说明

4. **GetPendingDifferencesOverdue**: 获取超时未处理的差异
   - 输入: hours（默认24小时）
   - 查询状态为 `pending` 且创建时间超过指定小时数的差异
   - 用于告警升级定时任务

---

### 3. Handler层API

**文件**: `server/internal/handler/admin_reconciliation_handler.go`

**新增接口**:

1. **AdminReconciliationDifferenceInvestigate**
   - 路由: `POST /api/v1/admin/reconciliation/differences/:id/investigate`
   - 权限: `finance:transaction:approve` + Admin重认证
   - 请求体: `{"notes": "调查备注"}`

2. **AdminReconciliationDifferenceIgnore**
   - 路由: `POST /api/v1/admin/reconciliation/differences/:id/ignore`
   - 权限: `finance:transaction:approve` + Admin重认证 + 必须提供原因
   - 请求体: `{"reason": "忽略原因（必填）"}`

3. **AdminReconciliationDifferenceResolve**
   - 路由: `POST /api/v1/admin/reconciliation/differences/:id/resolve-enhanced`
   - 权限: `finance:transaction:approve` + Admin重认证 + 必须提供解决方案
   - 请求体: `{"solution": "解决方案（必填）", "notes": "处理说明"}`

**更新接口**:

4. **AdminReconciliationDifferences** (差异列表)
   - 新增返回字段: `handleStatus`, `ignoreReason`, `solution`

---

### 4. 告警升级定时任务

**文件**: `server/internal/cron/reconciliation_alert_cron.go`

**功能**:
- 每小时执行一次
- 查询状态为 `pending` 且创建时间超过24小时的差异
- 为每个超时差异创建高优先级告警
- 告警类型: `payment_reconciliation_difference`
- 告警级别: `high`

**告警内容**:
```
对账差异超过24小时未处理
- 类型: amount_mismatch
- 订单号: ORDER_20260420_123456
- 金额差异: 100.00元
- 创建时间: 2026-04-19T10:00:00Z
```

**告警跳转**:
```
/admin/reconciliation/{recordId}/differences?differenceId={diffId}
```

## 技术亮点

### 1. 状态机设计

差异处理状态流转:
```
pending → investigating → resolved
pending → ignored
```

已解决的差异不能再次操作，保证数据一致性。

### 2. 权限控制

所有差异处理操作都需要:
- `finance:transaction:approve` 权限
- Admin重认证（敏感操作）
- 必填参数校验（原因/解决方案）

### 3. 审计日志

所有操作都会记录:
- 操作人（`resolved_by`）
- 操作时间（`resolved_at`）
- 操作内容（`notes`, `reason`, `solution`）

### 4. 告警升级

自动化告警机制:
- 定时检查超时差异
- 创建高优先级告警
- 提供快速跳转链接

### 5. 事务安全

所有数据库操作都使用GORM的事务机制，保证原子性。
