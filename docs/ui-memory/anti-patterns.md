# UI Anti-Patterns Memory

用途：
- 记录已经验证过“不好用、不好看、不适合本项目”的 UI 做法
- 让后续页面设计先避坑，而不是反复踩同样的问题

适合记录的内容：
- 被用户明确否定的页面结构
- 明显降低可读性的信息排布
- 过度 AI 味、过度卡片化、过度装饰的表达
- 不适合 `home-decoration` 业务场景的视觉或交互套路
- 已在真实页面中验证失败的布局、CTA、状态表达方式

记录格式建议：
- 反例名称
- 出现模块 / 页面类型
- 不好的点
- 为什么不好
- 以后要避免什么
- 可替代方向

写入规则：
- 只记录有复用价值的负面经验
- 不记录一次性的个人情绪化反馈
- 如果某个坏点已经升级为全项目长期禁用规则，再考虑写入 `DESIGN.md`
