# 家装平台产品迭代方案

> 已过期：本文档包含“多工长报价PK”等历史阶段规划，不代表当前报价 ERP 主链。现行施工报价以 `QuantityBase -> QuoteList -> QuoteSubmission -> 用户确认快照 -> ChangeOrder -> Settlement / Payout` 为准。

## 一、背景与目标

### 1.1 项目现状

当前平台已具备完整的家装交易基础设施：

**已实现核心功能**：
- ✅ 用户端（小程序 + App）：预约、设计确认、施工选择、项目进度、聊天
- ✅ 商家端（Web）：接单、报价、项目管理、财务管理
- ✅ 管理后台（Web）：审核、财务、风控、系统管理
- ✅ 后端服务：托管支付、WebSocket聊天、多身份系统、RBAC权限

**技术栈**：
- 用户端：React Native 0.83 (App) + Taro 3.x (小程序)
- 商家端/管理后台：React 18.3.1 + Ant Design
- 后端：Go + Gin + PostgreSQL + Redis

### 1.2 战略目标

将平台从"装修信息撮合"升级为"装修交易基础设施"，实现：
1. **GMV增长**：提升成交转化率和客单价
2. **履约效率**：透明化施工过程，降低投诉率
3. **平台抽佣**：建立可持续的商业模式
4. **复购裂变**：通过口碑和推荐实现自增长

## 二、MVP阶段规划（0-3个月）

### 2.1 核心目标

**聚焦成交链路**：让用户从"看设计师"到"签约付款"的转化率提升50%

### 2.2 必做功能（按优先级）

#### P0 - 智能报价系统（Week 1-2）

**目标**：用户提交户型/需求后，自动生成报价区间，提高下单率

**功能点**：
- 根据面积、风格、区域自动生成半包/全包报价区间
- 工期预估（基于历史数据）
- 材料预算明细
- 风险项提醒（如老房拆改、水电改造）

**技术实现**：
- 后端：新增 `QuoteEstimateService`，基于规则引擎生成报价
- 数据库：新增 `quote_templates` 表存储报价模板
- 小程序端：新增"智能报价"页面

**商业价值**：提升30%的咨询转化率

---

#### P1 - 多工长报价PK系统（Week 3-4）

**目标**：让3个工长同时报价，用户一键选择

**功能点**：
- 用户发起报价需求，系统推送给3个匹配工长
- 工长48小时内提交报价
- 用户查看对比表：价格、工期、评分、案例
- 一键选择并支付定金

**技术实现**：
- 后端：扩展 `QuoteService`，支持多工长报价
- 商家端：新增"报价任务"推送和提交页面
- 小程序端：新增"报价对比"页面

**商业价值**：提升平台议价权，提高成交率20%

---

#### P2 - 在线签约 + 定金支付（Week 5-6）

**目标**：微信内直接签合同 + 支付定金，锁定订单

**功能点**：
- 电子合同生成（集成第三方电子签章）
- 用户/商家在线签约
- 定金支付（微信支付）
- 合同存档和查看

**技术实现**：
- 后端：集成电子签章API（如e签宝、法大大）
- 数据库：新增 `contracts` 表
- 小程序端：新增"合同签署"页面

**商业价值**：锁单能力提升50%，减少流失

---

#### P3 - 限时成交机制（Week 7）

**目标**：催单，提高成交率

**功能点**：
- 报价48小时有效期
- 今日签约送监理服务
- 倒计时提醒

**技术实现**：
- 后端：报价增加过期时间字段
- 小程序端：报价页面增加倒计时组件

**商业价值**：提升15%的成交率

---

## 七、技术架构演进

### 7.1 当前架构

#### 第四阶段（9-12个月）
- 引入AI能力：智能报价、智能客服

## 十、风险与应对

### 10.1 技术风险

**风险**：高并发场景下系统崩溃

**应对**：
- 引入消息队列削峰
- 数据库读写分离
- Redis缓存热点数据

---

### 10.2 业务风险

**风险**：商家供给不足

**应对**：
- 前期人工BD优质商家
- 提供商家培训和工具
- 建立商家激励机制

---

### 10.3 合规风险

**风险**：托管支付需要支付牌照

**应对**：
- 接入第三方支付（如连连支付、汇付天下）
- 申请支付牌照（长期）

## 十二、总结

### 12.1 核心策略

1. **MVP聚焦成交**：智能报价、多工长PK、在线签约
2. **第二阶段聚焦履约**：工地直播、节点付款、验收系统
