二、根本原因
================================================================================

【问题 1: 环境不匹配】
---------------------
你当前使用的是 **Gemini Code**，而不是 **Claude Code**。

/omo 命令是 Claude Code 的专有斜杠命令，只能在 Claude Code 界面中使用。
就像你现在使用的 Gemini Code 有自己的命令一样，/omo 是 Claude Code 特有的。


【问题 2: myclaude 安装不完整】
-------------------------------
虽然 ~/.claude 目录存在，但关键的 bin 目录缺失：
  - 缺少: ~/.claude/bin/codeagent-wrapper.exe
  - 结果: 无法通过 CLI 调用 OMO 功能


【问题 3: 概念混淆】
-------------------
教程中的配置步骤是为了在 **Claude Code** 中使用 /omo 命令。
但你实际使用的是 **Gemini Code**，这是两个不同的产品。

-------------------

# 永久添加 (推荐)
[Environment]::SetEnvironmentVariable(
    'PATH',
    "$env:USERPROFILE\.claude\bin;" + [Environment]::GetEnvironmentVariable('PATH', 'User'),
    'User'
)


步骤 5: 验证命令可用
--------------------
