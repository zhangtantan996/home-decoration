# 小程序包体积记录（Phase 5 观测）

目的：在不做分包的前提下，用数据判断是否需要启动“分包专项里程碑”。

## 记录方式
1. 执行构建：`cd mini && npm run build:weapp`
2. 打开微信开发者工具 → 详情 → 本地设置/编译信息（以工具版本为准）查看包体积
3. 将“主包体积 / 总包体积 / 备注（本次大改动）”写入下表

## 触发分包专项的建议阈值
- 当主包体积逼近微信限制（以开发者工具提示为准）或启动明显变慢时，开启分包专项
- 分包专项前置约束：
  - TabBar 页面必须留主包
  - `subPackages[].root` 下的页面必须是实际子路径，禁止 `../` 跨目录引用
