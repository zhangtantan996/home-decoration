# Mini 设置功能 MVP 上线前置范围说明

> 日期：2026-04-27
> 范围：小程序 `pages/settings/*`、`pages/profile/edit` 以及对应用户设置接口。
> 目标：按“小程序可上线、用户可用、风险可控、合规可解释”判断设置功能范围，而不是按当前是否已经开发完成来决定是否上线。

## 1. 判断原则

设置功能虽然不是成交主链，但它承接账号、安全、隐私、通知、客服这些上线基础能力。MVP 不能只看“做完了什么”，而要看“缺了什么会影响上线”。

### 1.1 必须上线标准

满足任一条件，即使当前未完成，也应列为上线前置：

- 影响小程序审核、隐私合规、用户协议闭环。
- 影响用户登录后基本账号管理。
- 影响交易通知、订单/预约/项目处理提醒。
- 影响用户遇到问题后的联系与反馈路径。
- 影响账号安全、换绑、退出、注销等基本权利。
- 用户一旦看到入口，就必须真实可用，不能是空入口或假开关。

### 1.2 可以后置标准

满足以下条件，可以 MVP 后续迭代：

- 不影响用户完成预约、报价、方案确认、订单、项目、售后主链。
- 不影响小程序审核和隐私合规基础要求。
- 没有明确业务消费点，上线后用户改变设置也不会产生真实效果。
- 需要额外系统能力支撑，例如多语言、主题系统、推送订阅、设备会话治理、地图定位。

### 1.3 禁止上线形态

以下形态不应出现在一期线上：

- “暂未开放”的空入口。
- 保存成功但业务不生效的开关。
- 用户关闭后平台仍继续按原逻辑处理的隐私开关。
- 只展示技术字段、不解释业务含义的设置项。
- 缺少协议、客服、注销说明的账号设置页。

## 2. 一期必须上线的设置能力

### 2.1 设置首页

必须上线。

业务原因：用户登录后必须能找到账号管理、隐私协议、通知控制、客服反馈和退出登录。

必须展示：

- 个人资料
- 账号安全
- 通知提醒
- 隐私设置
- 意见反馈
- 联系客服
- 关于我们
- 退出登录

不建议在首页展示：

- 实名认证：不放设置首页；放入“账号安全”二级页，并在支付前强制触发。
- 登录设备：可放账号安全二级页；如果未打通，不放首页。
- 通用设置：字号、语言、深色模式不属于上线前置，不放首页。

### 2.2 个人资料

必须上线。

业务原因：用户需要维护基础身份展示；头像、昵称会影响个人中心、预约/需求沟通、商家识别。

必须支持：

- 头像上传或更换
- 昵称修改
- 生日选择
- 个人简介
- 保存资料

上线前必须补齐或确认：

- 头像选择取消不提示失败。
- 上传失败展示用户可读错误。
- 昵称为空时不能保存。
- 保存后个人页、登录态用户信息能同步刷新。

### 2.3 账号安全

必须上线。

业务原因：手机号登录是当前核心登录方式，账号安全页必须承接手机号、密码、注销和登录态管理。

必须支持：

- 当前手机号脱敏展示
- 修改手机号
- 修改登录密码
- 实名认证状态与认证入口
- 注销账号

可放二级但不强制首页展示：

- 登录设备管理

不应展示：

- “实名认证：暂未开放”这种空入口。

实名认证一期规则：

- 采用身份证二要素核验：真实姓名 + 身份证号。
- 不要求身份证照片，不做人脸/活体；人脸/活体放二期评估。
- 不在注册、浏览、提交需求、创建预约时强制。
- 凡是会发起支付、创建支付单、拉起支付、绑定收付款账户、提现账户的入口，必须先完成实名认证。
- 页面只展示脱敏姓名、身份证尾号、认证状态和失败原因。
- 隐私政策必须覆盖姓名、身份证号及第三方实名核验服务。

### 2.4 修改手机号

必须上线。

业务原因：手机号是登录、通知、预约、订单沟通的核心凭证，用户必须能换绑。

必须支持：

- 输入新手机号
- 发送验证码
- 倒计时
- 提交换绑
- 换绑成功后更新本地登录用户手机号

依赖：

- `POST /auth/send-code` 支持 `change_phone`
- `POST /user/change-phone`

上线前需要确认：

- 验证码真实可用或测试环境有明确测试码策略。
- 新手机号已被占用时有清晰提示。
- 换绑后旧登录态不会出现手机号显示不一致。

### 2.5 修改登录密码

必须上线。

业务原因：即使当前主登录方式是验证码/微信手机号，账号设置中存在密码能力时，必须允许用户维护密码；否则应彻底隐藏密码登录相关能力。

必须支持：

- 当前密码
- 新密码
- 确认新密码
- 最小长度校验
- 两次密码一致校验

依赖：

- `POST /user/change-password`

上线前需要确认：

- 若用户从未设置密码，旧密码是否必填的产品规则要明确。
- 密码规则前后端一致。

### 2.6 注销账号

必须上线。

业务原因：注销账号属于用户基础权利和合规要求，不能因为实现成本高而完全缺失。MVP 可以做简化注销，但不能没有入口和规则说明。

必须支持：

- 注销风险说明
- 当前手机号验证码
- 二次确认弹窗
- 注销成功后清理登录态并回到个人页或登录入口

依赖：

- `POST /auth/send-code` 支持 `delete_account`
- `POST /user/delete-account`

上线前必须由产品/法务确认：

- 注销后历史订单、预约、项目、支付、退款、争议记录是否保留。
- 用户端是否继续展示历史记录，还是全部不可见。
- 合规留存周期和隐私政策口径。

### 2.7 通知提醒

必须上线。

业务原因：平台交易主链依赖通知提醒用户处理预约、报价、订单、项目和退款。用户也必须能控制核心通知类别。

一期必须支持且真实生效：

- 系统通知
- 项目进度提醒
- 订单与支付提醒

后端必须真实消费：

- `notifySystem`
- `notifyProject`
- `notifyPayment`

上线前需要确认：

- 关闭项目提醒后，不再创建项目类站内通知。
- 关闭支付提醒后，不再创建支付/退款类站内通知。
- 关闭系统通知后，不再创建系统类站内通知。
- 不影响必须依法或交易安全保留的站内记录；如有不可关闭通知，需要在文案里说明。

一期不展示：

- 营销通知
- 声音提醒
- 震动提醒
- 订阅消息/推送授权开关

原因：这些不是当前站内通知 MVP 的必要能力，且需要额外推送授权和端能力支撑。

### 2.8 隐私设置

必须上线，但一期可以是轻量隐私中心。

业务原因：小程序上线必须有用户协议、隐私政策和用户可理解的数据处理入口。

一期必须展示：

- 用户协议
- 隐私政策
- 联系客服或隐私问题反馈入口

建议增加但不复杂化：

- “隐私问题与账号数据处理请联系客服”入口。

一期不展示未生效开关：

- 个性化推荐开关
- 定位授权开关
- 手机号可见开关
- 数据导出

原因：这些必须有真实业务消费和协议说明后才能上线，不能先放假开关。

### 2.9 意见反馈

必须上线。

业务原因：试运营期必须给用户反馈问题的通道，否则订单、支付、预约、售后异常无法闭环。

必须支持：

- 反馈分类
- 问题描述
- 联系方式选填
- 最多 4 张截图
- 提交反馈

依赖：

- 上传接口
- `POST /user/feedback`

上线前需要确认：

- 后台或运营能看到反馈记录并处理。
- 反馈截图涉及隐私时，隐私政策已覆盖。

### 2.10 联系客服 / 关于我们

必须上线。

业务原因：家装交易客单高、周期长，一期必须有人工兜底入口。

必须提供：

- 客服电话或微信号
- 服务时间
- 服务城市/区域
- 平台介绍
- 业务边界说明

## 3. 未完成但仍是上线前置的功能

| 功能 | 为什么必须上线 | 如果当前未完成，最低可接受形态 |
| --- | --- | --- |
| 隐私设置 | 小程序审核和用户数据合规基础 | 至少有用户协议、隐私政策、客服/隐私问题入口 |
| 注销账号 | 用户账号权利和合规要求 | 简化注销流程可以，但必须有验证码确认和留存说明 |
| 修改手机号 | 登录凭证和交易联系凭证 | 必须支持验证码换绑 |
| 核心通知开关 | 交易提醒和用户控制权 | 只做系统/项目/支付三类，且后端真实消费 |
| 意见反馈 | 试运营问题闭环 | 可先不做复杂工单，但必须能提交并让运营看到 |
| 联系客服 | 交易兜底 | 可先静态展示客服方式，但不能缺失 |
| 关于我们 | 平台可信背书和审核信息 | 可先静态页，但文案必须准确 |

## 4. 可以后续迭代的功能

| 功能 | 后续原因 | 上线条件 |
| --- | --- | --- |
| 登录设备管理 | 不影响交易主链，但属于账号安全增强 | 登录时记录设备、能识别当前设备、移除后会话真实失效 |
| 人脸/活体认证 | 一期采用身份证二要素，暂不做人脸 | 支付风控或监管要求明确后再接入 |
| 通用设置-字体 | 不影响交易主链 | 接入全局 UI token 或页面根 class |
| 通用设置-语言 | 不影响当前西安试点 | 接入完整 i18n，不只是保存字段 |
| 通用设置-清理缓存 | 低频工具能力 | 可放弱入口或后续上线 |
| 个性化推荐 | 暂无推荐算法消费链路 | 推荐系统上线且隐私政策覆盖 |
| 定位跟踪 | 当前西安试点不接地图定位 | 接地图定位、权限弹窗、隐私政策、关闭后不再采集 |
| 手机号可见 | 缺少公开展示场景和边界 | 明确哪些角色可见、关闭后全链路遵守 |
| 营销通知 | MVP 不做营销触达 | 营销体系、退订规则和合规文案明确 |
| 声音/震动提醒 | 站内通知无法保证端能力真实生效 | 接入端推送或设备通知能力后再做 |
| 深色模式 | 无完整主题体系 | 全局主题 token 完成后再上线 |

## 5. 一期推荐页面结构

设置首页推荐结构：

- 账号与安全
  - 个人资料
  - 账号安全
- 偏好设置
  - 通知提醒
  - 隐私设置
- 帮助与支持
  - 意见反馈
  - 联系客服
  - 关于我们
- 底部
  - 退出登录

账号安全二级页推荐结构：

- 当前手机号
- 修改手机号
- 修改登录密码
- 实名认证
- 注销账号
- 登录设备管理（仅在设备链路打通后展示）

隐私设置推荐结构：

- 用户协议
- 隐私政策
- 隐私问题联系客服

## 6. 现在需要你提供或确认的信息

必须提供：

1. 客服电话或微信号。
2. 客服工作时间。
3. 关于我们最终文案：平台名称、服务城市、服务边界。
4. 注销账号口径：注销后历史订单、预约、项目、支付、退款、争议记录如何处理。
5. 隐私协议最终版本：是否覆盖头像、手机号、地址、订单、反馈截图、实名信息。

需要你确认：

1. 实名认证二要素供应商账号、密钥和正式合同。
2. 登录设备管理是否必须一期上线。如果是，需要补登录设备记录和会话失效；如果不是，隐藏入口。
3. 通用设置是否必须一期上线。如果字体/语言不做真实全局生效，建议隐藏。
4. 通知开关关闭后，是否允许仍保留“交易安全必达通知”。如果允许，需要补文案说明。

## 7. 当前建议结论

一期必须上线：

- 个人资料
- 账号安全
- 修改手机号
- 修改登录密码
- 实名认证：支付和资金动作前置门禁
- 注销账号
- 通知提醒（三类真实开关）
- 隐私设置（协议 + 隐私问题入口）
- 意见反馈
- 联系客服
- 关于我们
- 退出登录

一期不建议上线：

- 登录设备管理，除非补齐设备记录和会话失效。
- 人脸/活体认证。
- 通用设置，除非字体/语言真实全局生效。
- 所有未被业务消费的隐私/推荐/定位/营销/深色模式开关。
