# 节点付款系统

## 概述

节点付款系统实现了按施工节点分期付款的功能，平台托管资金，用户确认节点完成后平台放款给商家。

## 功能特性

- ✅ 创建4节点付款计划（开工30%、水电30%、中期30%、验收10%）
- ✅ 用户支付节点款项到托管账户
- ✅ 用户确认节点完成后平台放款
- ✅ 实时查看节点付款状态
- ✅ 商家查看节点收款情况

## 技术架构

### 后端 (Go)

**文件结构**:
```
server/internal/
├── service/
│   ├── milestone_payment_service.go      # 节点付款服务
│   └── milestone_payment_service_test.go # 单元测试
├── handler/
│   └── milestone_payment_handler.go      # API Handler
└── router/
    └── router.go                         # 路由注册
```

**核心服务**:
- `CreateMilestonePaymentPlan()` - 创建节点付款计划
- `PayMilestone()` - 支付节点款项（使用事务+悲观锁）
- `ReleaseMilestonePayment()` - 确认完成并放款
- `GetMilestonePaymentStatus()` - 获取付款状态

**安全保障**:
- 使用数据库事务保证原子性
- 使用悲观锁（FOR UPDATE）防止并发问题
- 验证用户权限和项目状态
- 记录完整的交易日志

## API 接口

### 1. 创建节点付款计划

**请求**:
```http
POST /api/v1/projects/:id/milestone-payment-plan
Authorization: Bearer <token>

{
  "constructionQuote": 100000,
  "customPlans": [  // 可选：自定义计划
    {
      "name": "开工节点",
      "seq": 1,
      "percentage": 30,
      "criteria": "施工队进场，材料到位"
    }
  ]
}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "milestones": [
      {
        "id": 1,
        "projectId": 1,
        "name": "开工节点",
        "seq": 1,
        "amount": 30000,
        "percentage": 30,
        "status": 0,
        "criteria": "施工队进场，材料到位，开工交底完成"
      }
    ]
  }
}
```

### 2. 支付节点款项

**请求**:
```http

Authorization: Bearer <token>

{
  "paymentType": "wechat"  // wechat, alipay, balance

```

**响应**:
```json
{
  "code": 200,
```

## 业务流程

### 1. 创建节点付款计划

```
项目确认施工报价
    ↓
调用 CreateMilestonePaymentPlan
    ↓
创建4个节点（开工30%、水电30%、中期30%、验收10%）
    ↓
节点状态：待支付
```

### 2. 用户支付节点款项

```
用户选择节点
    ↓
调用 PayMilestone
    ↓
验证项目和节点状态
    ↓
创建或获取托管账户（悲观锁）
    ↓
更新托管账户余额
    ↓
更新节点状态为"已支付待验收"
    ↓
创建交易记录
    ↓
提交事务
```

### 3. 用户确认节点完成并放款

```
商家提交节点验收
    ↓
用户确认节点完成
    ↓
调用 ReleaseMilestonePayment
    ↓
调用 SettlementService.ReleaseMilestone
    ↓
创建结算单
    ↓
执行放款（扣减托管账户，增加商家收入）
    ↓
更新节点状态为"已放款"
    ↓
发送通知
```

## 数据模型

### Milestone (节点)

```go
type Milestone struct {
    Base
    ProjectID          uint64     // 项目ID
    Name               string     // 节点名称
    Seq                int8       // 顺序
    Amount             float64    // 金额
    Percentage         float32    // 百分比
    Status             int8       // 状态：0待支付 1已支付待验收 2已验收 3已放款
    Criteria           string     // 验收标准
    PaidAt             *time.Time // 支付时间
    ReleaseScheduledAt *time.Time // 计划放款时间
    ReleasedAt         *time.Time // 实际放款时间
}
```

### EscrowAccount (托管账户)

```go
type EscrowAccount struct {
    Base
    ProjectID       uint64  // 项目ID
    UserID          uint64  // 账户所有者
    TotalAmount     float64 // 总金额
    FrozenAmount    float64 // 冻结金额
    AvailableAmount float64 // 可用金额
    ReleasedAmount  float64 // 已放款金额
    Status          int8    // 状态：0待激活 1正常 2冻结 3已清算
}
```

### Transaction (交易记录)

```go
type Transaction struct {
    Base
    OrderID     string     // 订单号
    EscrowID    uint64     // 托管账户ID
    MilestoneID uint64     // 节点ID
    Type        string     // 类型：deposit, withdraw, transfer, refund
    Amount      float64    // 金额
    FromUserID  uint64     // 付款用户
    ToUserID    uint64     // 收款用户
    Status      int8       // 状态：0处理中 1成功 2失败
    CompletedAt *time.Time // 完成时间
}
```

## 安全注意事项

1. **事务保证**: 所有金钱操作必须在事务中执行
2. **悲观锁**: 使用 `FOR UPDATE` 防止并发问题
3. **权限验证**: 验证用户是项目所有者
4. **状态检查**: 验证节点和托管账户状态
5. **审计日志**: 记录所有交易操作
