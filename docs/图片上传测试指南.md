## 测试场景

### 场景 5.1: 测试后端上传 API

**目标**: 验证后端上传端点功能正常

**步骤**:
```bash
# 1. 准备测试图片
# 下载或使用任何 JPEG 图片，保存为 test.jpg

# 2. 获取认证 token
TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone": "13800138000", "code": "123456"}' \
  | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 3. 测试上传
curl -X POST http://localhost:8080/api/v1/upload \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@test.jpg"

# 预期响应:
# {
#   "code": 0,
#   "message": "success",
#   "data": {
#     "url": "/uploads/chat/202601/xxxxx.jpg"
#   }
# }

# 4. 验证文件可访问
curl -I http://localhost:8080/uploads/chat/202601/xxxxx.jpg
# 预期: HTTP 200 OK
```

**验收清单**:
- [ ] 上传端点响应 200 OK
- [ ] 响应包含 `url` 字段
- [ ] 文件 URL 可访问
- [ ] 在浏览器中访问 URL 时图片显示
- [ ] 文件存储在正确目录（`./uploads/chat/YYYYMM/`）

**故障排除**:
- **如果 401 未授权**: Token 过期或无效，重新登录
- **如果 500 错误**: 检查服务器日志，验证上传目录权限
- **如果文件不可访问**: 检查静态文件服务配置

## 代码实现说明

### Tinode 图片消息格式（Drafty）

```typescript
{
  txt: '[图片]',  // 后备文本
  ent: [{
    tp: 'IM',     // 图片类型
    data: {
      mime: 'image/jpeg',
      val: '/uploads/chat/202601/xxxxx.jpg',  // 图片 URL
      width: 1920,   // 可选
      height: 1080   // 可选
    }
  }]
}
```

### 上传流程

```
1. 用户选择图片 → launchImageLibrary() 或 launchCamera()
2. 获取图片 URI → file:///path/to/image.jpg
3. 调用 TinodeService.sendImageMessage(topicName, imageUri)
4. TinodeService.uploadFile() → Tinode LargeFileHelper
5. Tinode 上传到服务器 → 返回 URL
6. TinodeService 发布带图片 URL 的 Drafty 消息
7. 消息同步到所有参与者
8. 接收方从 URL 显示图片
```

### 后端上传端点

```
POST /api/v1/upload
Authorization: Bearer <token>
Content-Type: multipart/form-data

Body: file=<binary>

响应:
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "/uploads/chat/202601/xxxxx.jpg"
  }
}
```
