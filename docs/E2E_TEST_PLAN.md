## 测试场景

### 场景1: Mobile App基础功能测试

#### 1.1 登录和连接

**步骤**：
```bash
# 1. 启动Mobile App
cd mobile
npm start

# 2. 在另一个终端启动iOS模拟器
npm run ios

# 或启动Android模拟器
npm run android
```

**测试点**：
- [ ] 登录成功后获取tinodeToken
- [ ] TinodeService自动初始化
- [ ] 连接到Tinode服务器成功
- [ ] 会话列表加载

**预期结果**：
```
[Tinode] 初始化中...
[Tinode] WebSocket 已连接
[Tinode] 登录成功
[Tinode] 会话列表: X条
```

**验收标准**：
- [ ] 无错误日志
- [ ] 连接状态显示"已连接"
- [ ] 会话列表正常显示

---

#### 1.2 发送文本消息

**步骤**：
1. 账号A登录Mobile App
2. 进入与账号B的会话
3. 发送文本消息："测试消息1"
4. 观察消息状态变化

**测试点**：
- [ ] 消息发送中状态（pending）
- [ ] 消息发送成功状态（sent）
- [ ] 消息显示在聊天界面
- [ ] 时间戳正确

**预期结果**：
```
消息状态: pending → sent
消息内容: "测试消息1"
时间: 20:15
```

**验收标准**：
- [ ] 消息成功发送
- [ ] 状态变化正确
- [ ] 界面显示正常

---

#### 1.3 接收消息

**步骤**：
1. 保持账号A在线
2. 使用账号B登录另一台设备
3. 账号B发送消息给账号A
4. 观察账号A是否收到

**测试点**：
- [ ] 实时接收消息
- [ ] 消息内容正确
- [ ] 发送者信息正确
- [ ] 未读数更新

**预期结果**：
```
[Tinode] 📩 收到消息: {from: "usr2", content: "你好"}
会话列表未读数: +1
```

**验收标准**：
- [ ] 消息实时到达（<2秒）
- [ ] 内容完整无误
- [ ] 未读数正确

---

#### 1.4 已读回执

**步骤**：
1. 账号A打开与账号B的会话
2. 查看账号B发送的消息
3. 观察账号B端的消息状态

**测试点**：
- [ ] 账号A查看消息后，未读数清零
- [ ] 账号B端消息状态变为"已读"
- [ ] 已读回执实时更新

**预期结果**：
```
账号A: 未读数 1 → 0
账号B: 消息状态 sent → read (✓✓)
```

**验收标准**：
- [ ] 已读状态正确
- [ ] 实时更新（<2秒）

```
账号A发送: "你在吗？" (账号B离线)

```

**验收标准**：
- [ ] 离线消息不丢失
