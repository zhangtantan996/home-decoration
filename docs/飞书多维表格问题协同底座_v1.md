# 飞书多维表格问题协同底座 v1

## 目标

为 `home-decoration` 提供一套可落地的飞书多维表格底座，用于统一承载：

- 问题提报
- 研发缺陷跟踪
- 项目协同与指派
- 截止时间与结果回填

第一版只做：

- 飞书开放平台 API 直连
- 多维表格结构初始化
- 字段、视图、基础权限约束

第一版不做：

- 飞书 MCP
- 自动通知
- 自动同步
- 复杂自动编排

## 需要的飞书准备

在飞书开放平台准备一个应用，并获取：

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- `FEISHU_FOLDER_TOKEN`（可选，用于指定多维表格创建到哪个文件夹）

建议申请并核对与多维表格相关的能力：

- 获取 tenant access token
- 创建多维表格
- 创建数据表
- 创建字段
- 创建视图
- 创建/读取/更新记录

说明：

- 分享链接不是稳定控制入口
- 正式控制以开放平台 API 和 token 为准
- 凭证只放本地环境，不进仓库

## 建表方式

### 1. 先预览结构

```bash
node scripts/feishu/setup-bitable-issue-base.mjs --dry-run
```

这个命令不会访问飞书，只会输出待创建的：

- base 名称
- 表数量
- 每张表的字段
- 每张表的视图

### 2. 填写环境变量

将 [feishu.bitable.example.env](/Volumes/tantan/AI_project/home-decoration/env/feishu.bitable.example.env) 复制为本地私有环境文件，至少填：

```bash
FEISHU_APP_ID=...
FEISHU_APP_SECRET=...
FEISHU_FOLDER_TOKEN=...
FEISHU_BITABLE_NAME=home-decoration 问题提报与协同底座
```

### 3. 正式创建

```bash
export FEISHU_APP_ID=...
export FEISHU_APP_SECRET=...
export FEISHU_FOLDER_TOKEN=...
export FEISHU_BITABLE_NAME='home-decoration 问题提报与协同底座'
node scripts/feishu/setup-bitable-issue-base.mjs --create
```

创建成功后，脚本会输出：

- `appToken`
- base 名称
- 各表 `tableId`

### 4. 同步已存在的旧基座

如果已经有一版旧表，不要重建，直接原地同步：

```bash
export FEISHU_APP_ID=...
export FEISHU_APP_SECRET=...
export FEISHU_BITABLE_APP_TOKEN=...
node scripts/feishu/setup-bitable-issue-base.mjs --sync-existing --app-token "$FEISHU_BITABLE_APP_TOKEN"
```

这个模式会做 4 件事：

- 将旧英文表名原地改成中文表名
- 删除每张表自动生成的 `表格视图 1`
- 修正错误或临时视图名，例如 `按负责人-测试`
- 给关键视图补真正的筛选配置，不再只是“名字不同内容一样”

### 5. 填充演示数据

```bash
export FEISHU_APP_ID=...
export FEISHU_APP_SECRET=...
export FEISHU_BITABLE_APP_TOKEN=...
node scripts/feishu/seed-bitable-demo.mjs --app-token "$FEISHU_BITABLE_APP_TOKEN"
```

演示数据支持中英文旧表名映射，但推荐先执行一次 `--sync-existing`，保持表名和视图名一致。

## 表结构

### `问题池`

用途：

- 主问题表
- 承载提报、指派、处理、验证、关闭

关键字段：

- `编号`
- `标题`
- `类型`
- `来源`
- `模块`
- `业务域`
- `优先级`
- `严重度`
- `状态`
- `提交人`
- `指派给`
- `协作者`
- `计划开始时间`
- `截止时间`
- `实际完成时间`
- `当前结果`
- `复现步骤`
- `期望结果`
- `实际结果`
- `相关链接`
- `关联项目任务`
- `验证状态`
- `验证人`
- `备注`

视图：

- `全部问题`
- `待我处理`
- `待我验证`
- `本周到期`
- `P0/P1`
- `按模块看板`
- `阻塞项`
- `已解决待关闭`

### `项目协同`

用途：

- 项目任务与里程碑协同

### `成员目录`

用途：

- 维护可指派成员与角色

### `验证记录`

用途：

- 验收与验证记录

## 当前实现边界

当前脚本已覆盖：

- 获取 tenant token
- 创建多维表格
- 创建数据表
- 创建字段
- 创建视图
- 同步已存在基座
- 删除默认空白视图
- 修正关键视图筛选
- 预留记录查询/新增/更新/upsert 能力

当前未覆盖：

- 飞书消息通知
- 表单视图
- 自动同步仓库任务状态
- 更细的视图筛选和权限策略自动配置
