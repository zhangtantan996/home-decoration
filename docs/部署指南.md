## 环境要求

### 基础环境

| 组件 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| Go | 1.21+ | 1.23+ | 后端 API 服务 |
| Node.js | 18+ | 20+ | 前端构建工具 |
| PostgreSQL | 14+ | 15+ | 主数据库 |
| Redis | 6+ | 6.2+ | 缓存和会话存储 |
| Docker | 20+ | 24+ | 容器化部署（可选） |
| Docker Compose | 2.0+ | 2.20+ | 本地开发环境（可选） |

### 操作系统

- Linux (Ubuntu 20.04+, CentOS 8+)
- macOS 12+
- Windows 10+ (需要 WSL2)

### 开发工具（可选）

- **Air**: Go 热重载工具 (`go install github.com/cosmtrek/air@latest`)
- **微信开发者工具**: 小程序开发必需
- **Android Studio**: Android 应用构建
- **Xcode**: iOS 应用构建（仅 macOS）

```bash
CREATE DATABASE home_decoration;

CREATE USER home_decoration_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE home_decoration TO home_decoration_user;
```

## 数据库迁移

### 迁移文件位置

所有正式迁移文件位于 `server/migrations/` 目录。

### 执行顺序

数据库迁移文件按版本号顺序执行。以下是关键迁移文件：

#### 核心迁移（按版本号顺序）

```bash
# 进入数据库
psql -U postgres -d home_decoration

# 执行迁移（示例）
\i server/migrations/001_create_notifications_table.sql
\i server/migrations/002_create_user_identities.sql
\i server/migrations/002_add_proposal_versioning.sql
\i server/migrations/003_add_booking_refund_fields.sql
\i server/migrations/004_init_fee_configs.sql
```

#### 高风险链路补洞（必须执行）

```bash
# 高风险链路最新补洞（认证/短信审计/商家入驻）
\i server/migrations/v1.6.9_reconcile_high_risk_schema_guard.sql
```

#### 完整迁移列表

建议按以下顺序执行所有迁移：

1. `001_create_notifications_table.sql`
2. `002_create_user_identities.sql`
3. `002_add_proposal_versioning.sql`
4. `003_add_booking_refund_fields.sql`
5. `004_init_fee_configs.sql`
6. `20251231_add_proposal_id_to_projects.sql`
7. `20260121_add_social_features.sql`
8. `20260122_backfill_inspiration_visibility.sql`
9. `add_case_management_menu.sql`
10. `add_user_login_lock_fields.sql`
11. `v1.1.0_add_provider_fields.sql`
12. `v1.1.1_add_provider_subtype.sql`
13. ... (按版本号继续)
14. `v1.6.9_reconcile_high_risk_schema_guard.sql` ⚠️ **必须执行**
15. `v1.7.0_add_quote_subsystem.sql`
16. ... (继续后续版本)

### 批量执行脚本

```bash
# 创建批量执行脚本
cat > /tmp/run_migrations.sh << 'EOF'
#!/bin/bash
set -e

DB_USER="postgres"
DB_NAME="home_decoration"
MIGRATIONS_DIR="server/migrations"

echo "开始执行数据库迁移..."

for sql_file in $(ls -1 $MIGRATIONS_DIR/*.sql | sort -V); do
  echo "执行: $sql_file"
  psql -U $DB_USER -d $DB_NAME -f $sql_file
  if [ $? -eq 0 ]; then
    echo "✓ 成功: $sql_file"
  else
    echo "✗ 失败: $sql_file"
    exit 1
  fi
done

echo "所有迁移执行完成！"
EOF

chmod +x /tmp/run_migrations.sh
/tmp/run_migrations.sh
```

### 验证迁移

```bash
# 检查表是否创建成功
psql -U postgres -d home_decoration -c "\dt"

# 检查关键表结构
psql -U postgres -d home_decoration -c "\d users"
psql -U postgres -d home_decoration -c "\d providers"
psql -U postgres -d home_decoration -c "\d projects"
```

### Schema Guard 检查

```bash
# 本地开发环境检查 schema 健康状态
npm run db:check
```

## 配置文件说明

### 1. 后端配置 (`server/config.yaml`)

```yaml
server:
  host: "0.0.0.0"
  port: "8080"
  mode: "debug"  # 生产环境改为 "release"
  public_url: "http://localhost:8080"

database:
  host: "localhost"
  port: "5432"
  user: "postgres"
  password: "${DATABASE_PASSWORD}"  # 从环境变量读取
  dbname: "home_decoration"
  sslmode: "disable"  # 生产环境改为 "require"

redis:
  host: "localhost"
  port: "6380"
  password: "${REDIS_PASSWORD}"
  db: 0

jwt:
  secret: "${JWT_SECRET}"  # 从环境变量读取
  expire_hour: 8

wechat_mini:
  app_id: "${WECHAT_MINI_APPID}"
  app_secret: "${WECHAT_MINI_SECRET}"
```

### 2. 环境变量配置

创建 `.env` 文件（不要提交到 Git）：

```bash
# 数据库
DATABASE_PASSWORD=your_db_password
DATABASE_HOST=localhost
DATABASE_PORT=5432

# Redis
REDIS_PASSWORD=your_redis_password
REDIS_HOST=localhost
REDIS_PORT=6380

# JWT
JWT_SECRET=your_jwt_secret_key_min_32_chars

# 微信小程序
WECHAT_MINI_APPID=your_wechat_appid
WECHAT_MINI_SECRET=your_wechat_secret

# 短信服务（阿里云）
SMS_PROVIDER=aliyun
SMS_ACCESS_KEY_ID=your_aliyun_access_key
SMS_ACCESS_KEY_SECRET=your_aliyun_secret
SMS_SIGN_NAME=your_sms_sign_name
SMS_TEMPLATE_CODE=your_template_code

# 对象存储（可选）
STORAGE_DRIVER=local  # 或 oss
STORAGE_OSS_ENDPOINT=your_oss_endpoint
STORAGE_OSS_ACCESS_KEY_ID=your_oss_key
STORAGE_OSS_ACCESS_KEY_SECRET=your_oss_secret
STORAGE_OSS_BUCKET=your_bucket_name
```

### 3. 前端配置

#### Admin 管理后台 (`.env`)

```bash
VITE_API_URL=http://localhost:8080/api/v1
```

#### 微信小程序 (`.env`)

```bash

```

## 启动服务

### 方式一：Docker Compose（推荐）

```bash
# 启动所有服务（数据库、Redis、后端、Admin）
docker-compose -f docker-compose.local.yml up -d

# 查看日志
docker-compose -f docker-compose.local.yml logs -f

# 停止服务
docker-compose -f docker-compose.local.yml down
```

### 方式二：手动启动

#### 1. 启动后端 API

```bash
cd server

# 方式 A: 直接运行
go run ./cmd/api

# 方式 B: 使用 Air 热重载（推荐开发环境）
air

# 方式 C: 编译后运行
make build
./bin/api
```

后端服务默认运行在 `http://localhost:8080`

#### 2. 启动 Admin 管理后台

```bash
cd admin
npm run dev
```

Admin 默认运行在 `http://localhost:5173`

#### 3. 启动微信小程序

```bash
cd mini
npm run dev:weapp
```

然后使用微信开发者工具导入 `mini/dist` 目录。

#### 4. 启动商家端（如果存在）

```bash
cd merchant
npm run dev
```

#### 5. 启动移动端（React Native）

```bash
cd mobile

# 启动 Metro bundler
npm start

# 在另一个终端运行 Android
npm run android

# 或运行 iOS（仅 macOS）
npm run ios
```

```bash
# 数据库
DB_USER=postgres

DB_NAME=home_decoration

# Redis
REDIS_PASSWORD=strong_redis_password

# 后端
SERVER_MODE=release
JWT_SECRET=strong_jwt_secret_min_32_chars

# 微信小程序
WECHAT_MINI_APPID=production_appid
WECHAT_MINI_SECRET=production_secret
```

### 3. Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # Admin 管理后台
    location /admin {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://api:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket 代理
    location /api/v1/ws {
        proxy_pass http://api:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash

psql -U postgres -d home_decoration < backup_20260420_120000.sql
```

## 常见问题排查

### 1. 数据库连接失败

**错误信息**：
```
failed to connect to database: dial tcp [::1]:5432: connect: connection refused
```

**解决方案**：
- 检查 PostgreSQL 是否运行：`ps aux | grep postgres`
- 检查端口是否正确：`netstat -an | grep 5432`
- 检查 `config.yaml` 中的数据库配置
- 检查环境变量 `DATABASE_HOST` 和 `DATABASE_PORT`

### 2. Redis 连接失败

**错误信息**：
```
redis: connection refused
```

**解决方案**：
- 检查 Redis 是否运行：`redis-cli ping`
- 检查端口是否正确（默认 6379 或配置的 6380）
- 检查 Redis 密码配置

### 3. 端口冲突

**错误信息**：
```
bind: address already in use
```

**解决方案**：

**macOS/Linux**：
```bash
# 查找占用端口的进程
lsof -i :8080

# 杀死进程
kill -9 <PID>
```

**Windows**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 杀死进程
taskkill /F /PID <PID>
```

### 4. Go 编译错误

**错误信息**：
```
package xxx is not in GOROOT
```

**解决方案**：
```bash
# 清理模块缓存
go clean -modcache

# 重新下载依赖
go mod download

# 更新依赖
go mod tidy
```

### 5. 前端依赖安装失败

**错误信息**：
```
npm ERR! code ERESOLVE
```

**解决方案**：
```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install

# 或使用 --legacy-peer-deps
npm install --legacy-peer-deps
```

### 6. 数据库迁移失败

**错误信息**：
```
ERROR: relation "xxx" already exists
```

**解决方案**：
- 检查迁移是否已执行：`\dt` 查看表列表
- 跳过已执行的迁移文件
- 或重建数据库：
  ```bash
  dropdb home_decoration
  createdb home_decoration
  # 重新执行所有迁移
  ```

### 7. JWT Token 验证失败

**错误信息**：
```
invalid token
```

**解决方案**：
- 检查 `JWT_SECRET` 环境变量是否设置
- 确保前后端使用相同的 JWT_SECRET
- 检查 token 是否过期（默认 8 小时）
- 清除浏览器缓存和 localStorage

### 8. 微信小程序无法连接后端

**错误信息**：
```
request:fail
```

**解决方案**：
- 检查 `TARO_APP_API_BASE` 配置
- 微信开发者工具中开启"不校验合法域名"
- 确保后端服务正在运行
- 检查网络代理设置

### 9. Docker 容器无法启动

**错误信息**：
```
Error response from daemon: driver failed
```

**解决方案**：
```bash
# 查看容器日志
docker-compose -f docker-compose.local.yml logs api

# 重建容器
docker-compose -f docker-compose.local.yml up -d --force-recreate

# 清理未使用的资源
docker system prune -a
```

### 10. Schema Guard 检查失败

**错误信息**：
```
Schema drift detected in high-risk tables
```

**解决方案**：
```bash
# 执行高风险链路补洞迁移
psql -U postgres -d home_decoration -f server/migrations/v1.6.9_reconcile_high_risk_schema_guard.sql

# 重新检查
npm run db:check
```

## 性能优化建议

1. **数据库索引**：确保所有迁移文件中的索引已创建
2. **Redis 缓存**：启用 Redis 缓存热点数据
3. **静态资源 CDN**：生产环境使用 CDN 加速静态资源
4. **Gzip 压缩**：Nginx 启用 gzip 压缩
5. **数据库连接池**：调整 GORM 连接池大小

---

## 安全建议

1. **不要提交敏感信息**：`.env` 文件加入 `.gitignore`
2. **使用强密码**：数据库、Redis、JWT Secret 使用强密码
3. **启用 HTTPS**：生产环境必须使用 HTTPS
4. **定期备份**：每日备份数据库
5. **更新依赖**：定期更新依赖包修复安全漏洞
