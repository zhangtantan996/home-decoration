# 在线签约 + 定金支付系统

## 功能概述

本系统实现了在线签约和定金支付功能，支持用户和商家在微信小程序内直接签署合同并支付定金。

## 核心功能

### 1. 合同管理

- **创建合同**: 商家可以为预约、需求或项目创建合同
- **电子签章**: 支持用户和商家在线签署合同（当前使用模拟签章）
- **合同下载**: 签署完成后可下载合同文件
- **合同状态查询**: 实时查询合同签署和支付状态

### 2. 定金支付

- **微信支付**: 支持微信小程序内支付定金
- **支付宝支付**: 支持支付宝H5支付定金
- **支付回调**: 自动处理支付成功回调，更新合同状态

### 3. 通知系统

- 用户签署合同后通知商家
- 商家签署合同后通知用户
- 定金支付成功后通知双方

```

Authorization: Bearer {user_token}
```

## 合同状态流转

```
draft (草稿)
  ↓
pending_sign (待签署)
  ↓
signed (已签署) - 双方都签署完成
  ↓
confirmed (已确认) - 定金支付完成
  ↓
active (生效中)
  ↓
completed (已完成)
```

## 电子签章集成

当前使用模拟签章实现。如需集成真实的电子签章服务，请修改以下文件：

- `server/internal/service/contract_service.go` 中的 `createESignFlow` 方法
- 支持的电子签章提供商：
  - e签宝 (esign)
  - 法大大 (fadada)
  - 上上签

### 集成示例（e签宝）

```go
func (s *ContractService) createESignFlow(contract *model.Contract) error {
    client := esign.NewClient(apiKey, apiSecret)
    
    // 创建签署流程
    flow, err := client.CreateSignFlow(&esign.SignFlowRequest{
        BusinessScene: "装修合同签署",
        FlowConfig: esign.FlowConfig{
            NotifyUrl: "https://yourdomain.com/api/v1/esign/callback",
        },
    })
    if err != nil {
        return err
    }
    
    // 添加签署文档
    doc, err := client.AddDocument(flow.FlowID, &esign.DocumentRequest{
        FileContent: contract.ContractContent,
        FileName:    fmt.Sprintf("%s.pdf", contract.ContractNo),
    })
    if err != nil {
        return err
    }
    
    // 添加签署方
    _, err = client.AddSigners(flow.FlowID, []esign.Signer{
        {
            SignerType: "PERSON",
            SignerAccount: getUserAccount(contract.UserID),
            SignOrder: 1,
        },
        {
            SignerType: "PERSON",
            SignerAccount: getProviderAccount(contract.ProviderID),
            SignOrder: 2,
        },
    })
    
    contract.ESignFlowID = flow.FlowID
    contract.ESignProvider = "esign"
    
    return repository.DB.Model(contract).Updates(map[string]interface{}{
        "esign_flow_id": flow.FlowID,
        "esign_provider": "esign",
    }).Error
}
```

## 支付回调处理

支付成功后会自动调用 `HandleContractDepositPaymentSuccess` 方法，该方法会：

1. 更新合同的定金支付状态
2. 更新合同状态为 `confirmed`
3. 如果有关联项目，更新项目状态
4. 发送通知给用户和商家

## 注意事项

1. **安全性**: 
   - 所有接口都需要JWT认证
   - 合同内容应该加密存储
   - 支付接口需要防重放攻击

2. **事务处理**:
   - 定金支付使用数据库事务确保数据一致性
   - 支付回调处理使用悲观锁防止并发问题

3. **错误处理**:
   - 所有错误都有明确的错误信息
   - 支付失败会自动回滚

4. **通知**:
   - 关键操作都会发送站内通知
   - 可以扩展为短信或微信模板消息
