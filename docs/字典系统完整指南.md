## 系统概述

### 为什么需要字典系统？

**现状问题**:
- ❌ 装修风格、户型等选项硬编码在前端代码中
- ❌ 修改选项需要改代码、重新部署
- ❌ 同一字典在不同页面定义不一致
- ❌ 无法动态添加新选项

**解决方案**:
- ✅ 统一在数据库管理所有枚举数据
- ✅ 管理员通过界面动态增删改查
- ✅ Redis 缓存，查询速度快
- ✅ 前端 DictSelect 组件，开箱即用

### 核心功能

- 📊 管理 12 类字典分类（装修风格、户型、预算等）
- 🔧 管理员可动态增删改查字典值
- ⚡ Redis 缓存，查询速度快
- 🔒 权限控制、审计日志
- 🎯 前端 DictSelect 组件，一行代码替换硬编码

### 支持的字典类型

| 分类代码 | 中文名称 | 优先级 | 使用场景 |
|---------|---------|-------|---------|
| `style` | 装修风格 | P0 | 作品、预约、商家擅长 |
| `layout` | 户型 | P0 | 作品、预约、项目 |
| `budget_range` | 预算区间 | P0 | 预约、项目筛选 |
| `renovation_type` | 装修类型 | P1 | 预约、订单（全包/半包） |
| `work_type` | 工种类型 | P1 | 工长信息（瓦工/电工） |
| `provider_sub_type` | 商家类型 | P1 | 商家入驻（个人/公司） |
| `phase_type` | 施工阶段 | P2 | 项目进度 |
| `material_category` | 材料分类 | P2 | 主材门店 |
| `review_tag` | 评价标签 | P2 | 商家评价 |
| `certification_type` | 资质类型 | P2 | 商家认证 |
| `after_sales_type` | 售后类型 | P2 | 售后申请 |
| `service_area` | 服务区域 | P1 | 商家服务范围 |

### 数据库设计

#### 字典分类表 (dictionary_categories)

```sql
CREATE TABLE dictionary_categories (
    id BIGSERIAL PRIMARY KEY,
    code VARCHAR(50) UNIQUE NOT NULL,           -- 分类代码 (style, layout)
    name VARCHAR(100) NOT NULL,                 -- 分类名称（装修风格, 户型）
    description TEXT,
    sort_order INT DEFAULT 0,                   -- 排序
    enabled BOOLEAN DEFAULT true,               -- 是否启用
    icon VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 字典值表 (system_dictionaries)

```sql
CREATE TABLE system_dictionaries (
    id BIGSERIAL PRIMARY KEY,
    category_code VARCHAR(50) NOT NULL,         -- 所属分类
    value VARCHAR(100) NOT NULL,                -- 存储值（现代简约）
    label VARCHAR(100) NOT NULL,                -- 显示标签（现代简约）
    description TEXT,
    sort_order INT DEFAULT 0,
    enabled BOOLEAN DEFAULT true,
    extra_data JSONB,                           -- 扩展数据
    parent_value VARCHAR(100),                  -- 父级值（支持多级）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uk_dict_category_value UNIQUE(category_code, value)
);
```

### API 接口

#### 公开接口（无需认证）

**获取字典选项**:
```
GET /api/v1/dictionaries/:category

响应:
{
  "code": 0,
  "data": [
    {"value": "现代简约", "label": "现代简约"},
    {"value": "北欧风格", "label": "北欧风格"}
  ]
}
```

**获取所有分类**:
```
GET /api/v1/dictionaries/categories
```

#### 管理接口（需管理员权限）

- `GET /api/v1/admin/dictionaries` - 分页查询
- `POST /api/v1/admin/dictionaries` - 创建字典值
- `PUT /api/v1/admin/dictionaries/:id` - 更新
- `DELETE /api/v1/admin/dictionaries/:id` - 删除

## 快速部署

### 步骤 1: 数据库迁移

```bash
# 1. 创建表结构
psql -U postgres -d home_decoration -f server/scripts/migrate_dictionaries.sql

# 2. 初始化数据（12个分类，80+字典值）
psql -U postgres -d home_decoration -f server/scripts/init_dictionaries.sql

# 3. 验证数据（可选）
psql -U postgres -d home_decoration -f server/scripts/validate_dictionaries.sql
```

### 步骤 2: 配置后端路由

#### 2.1 修改 `server/cmd/api/main.go`

在 `handler.InitHandlers(cfg)` 后添加：

```go
// 初始化数据字典相关
dictRepo := repository.NewDictionaryRepository(repository.DB)
dictCache := service.NewDictCacheService()
dictService := service.NewDictionaryService(dictRepo, dictCache)
dictHandler := handler.NewDictionaryHandler(dictService)
```

在文件顶部 import 中添加：
```go
"home-decoration-server/internal/service"
```

修改 router.Setup 调用：
```go
r := router.Setup(cfg, hub, wsHandler, dictHandler)
```

#### 2.2 修改 `server/internal/router/router.go`

**修改函数签名**（第12行）：
```go
func Setup(cfg *config.Config, hub *ws.Hub, wsHandler *ws.Handler, dictHandler *handler.DictionaryHandler) *gin.Engine {
```

**添加公开路由**（约第60行，material-shops之后）：
```go
// 数据字典（公开接口）
v1.GET("/dictionaries/categories", dictHandler.GetAllCategories)
v1.GET("/dictionaries/:category", dictHandler.GetDictOptions)
```

**添加管理路由**（约第300行，admin.POST("/cases")之后）：
```go
// 数据字典管理
admin.GET("/dictionaries", dictHandler.ListDicts)
admin.POST("/dictionaries", dictHandler.CreateDict)
admin.PUT("/dictionaries/:id", dictHandler.UpdateDict)
admin.DELETE("/dictionaries/:id", dictHandler.DeleteDict)

// 字典分类管理（可选）
admin.GET("/dictionaries/categories", dictHandler.ListCategories)
admin.POST("/dictionaries/categories", dictHandler.CreateCategory)
admin.PUT("/dictionaries/categories/:code", dictHandler.UpdateCategory)
admin.DELETE("/dictionaries/categories/:code", dictHandler.DeleteCategory)
```

## 使用指南

### 管理员：添加新字典值

1. 访问 `系统管理 > 数据字典`
2. 选择分类（如"装修风格"）
3. 点击"新增"按钮
4. 填写表单：
   - 值：`侘寂风格`
   - 标签：`侘寂风格`
   - 描述：`日式侘寂美学`
   - 排序：`11`
5. 点击"确定"
6. 前端缓存自动刷新

### 开发者：使用 DictSelect 组件

#### 基本用法

```tsx
import { DictSelect } from '@/components/DictSelect';

// 在 Form 中使用
<Form.Item label="装修风格" name="style">
    <DictSelect category="style" placeholder="请选择装修风格" />
</Form.Item>
```

#### 多选模式

```tsx
<DictSelect
    category="style"
    mode="multiple"
    placeholder="选择擅长的设计风格"
/>
```

#### 页面改造示例

## 扩展与维护

### 缓存策略

#### Redis 缓存设计

**缓存Key规范**:
```
格式：dict:{category_code}
示例：dict:style, dict:layout
```

**TTL**:
- 默认：3600秒（1小时）
- 高频字典：7200秒（2小时）

#### 缓存更新流程

1. **查询流程**：
   - 先查 Redis
   - 未命中则查数据库
   - 写入 Redis 缓存

2. **更新流程**：
   - 先更新数据库
   - 删除 Redis 缓存
   - 下次查询时重建缓存

### 添加新字典分类

```sql
-- 1. 插入分类
INSERT INTO dictionary_categories (code, name, description, sort_order)
VALUES ('room_style', '房间风格', '针对不同房间的风格分类', 13);

-- 2. 插入字典值
INSERT INTO system_dictionaries (category_code, value, label, sort_order) VALUES
('room_style', '卧室现代', '卧室现代', 1),
('room_style', '客厅简约', '客厅简约', 2);

-- 3. 前端直接使用
<DictSelect category="room_style" />
```

### 迁移历史数据

```sql
-- 检查不兼容的数据
SELECT DISTINCT style FROM provider_cases
WHERE style NOT IN (
    SELECT value FROM system_dictionaries WHERE category_code = 'style'
);

-- 批量修复（如果发现"北欧"需要改为"北欧风格"）
UPDATE provider_cases
SET style = '北欧风格'
WHERE style = '北欧';
```

## 故障排除

### 问题 1: 接口 404

**症状**: `GET /api/v1/dictionaries/style` 返回 404

**原因**: 路由未正确配置

**解决**:
1. 检查 `router.go` 是否添加了路由
2. 检查 `main.go` 是否初始化了 `dictHandler`
3. 重启后端服务

### 问题 2: 前端无数据

**症状**: DictSelect 组件无数据

**排查步骤**:
```bash
# 1. 检查后端 API
curl http://localhost:8080/api/v1/dictionaries/style

# 2. 检查数据库
psql -U postgres -d home_decoration
SELECT * FROM system_dictionaries WHERE category_code='style';

# 3. 清除前端缓存
# 浏览器控制台执行
useDictStore.getState().clearAllDicts()
```

### 问题 3: 缓存未刷新

**症状**: 修改字典值后前端未更新

**解决**:
1. 检查管理页面是否调用了 `clearDict(categoryCode)`
2. 手动清除缓存：浏览器控制台执行 `useDictStore.getState().clearAllDicts()`
3. 刷新页面

### 问题 4: Redis 连接失败

**症状**: 后端日志显示 `Redis connection failed`

**解决**:
1. 检查 Redis 是否启动：`redis-cli ping`
2. 检查配置：`server/config.yaml`
3. 系统会自动降级到直接查数据库，不影响业务
