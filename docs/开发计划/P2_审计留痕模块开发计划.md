> **优先级**: P2 (中优先级)
> **工作量**: 2人日
> **完成时间**: 第三周
> **依赖模块**: 所有业务模块

---

## 一、模块概述

### 1.1 业务目标

完善审计留痕机制，确保所有关键操作都有审计日志，支持管理员查询和追溯。

### 1.2 功能范围

1. **审计日志记录**
   - 自动记录关键操作
   - 记录操作人、操作时间、操作内容
   - 记录操作前后状态

2. **审计日志查询**
   - 查询审计日志列表
   - 筛选操作类型和时间
   - 导出审计日志

## 三、技术设计

### 3.1 数据模型

#### 3.1.1 审计日志表 (audit_logs)

```sql
CREATE TABLE audit_logs (
    id BIGSERIAL PRIMARY KEY,
    operator_type VARCHAR(20) NOT NULL, -- user/admin/system
    operator_id BIGINT,
    operation_type VARCHAR(50) NOT NULL, -- create_booking/confirm_proposal/freeze_funds/etc
    resource_type VARCHAR(50) NOT NULL, -- booking/project/order/etc
    resource_id BIGINT NOT NULL,
    before_state JSONB, -- 操作前状态
    after_state JSONB, -- 操作后状态
    ip_address VARCHAR(50),
    user_agent TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_audit_logs_operator ON audit_logs(operator_type, operator_id);
CREATE INDEX idx_audit_logs_operation_type ON audit_logs(operation_type);
CREATE INDEX idx_audit_logs_resource ON audit_logs(resource_type, resource_id);
CREATE INDEX idx_audit_logs_created_at ON audit_logs(created_at);
```

### 3.2 后端接口设计

#### 3.2.1 审计日志接口

**GET /api/v1/admin/audit-logs**

查询审计日志

**请求参数**:
- `operationType`: 操作类型
- `resourceType`: 资源类型
- `startDate`: 开始日期
- `endDate`: 结束日期
- `page`: 页码
- `pageSize`: 每页数量

**响应**:
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 2001,
        "operatorType": "admin",
        "operatorId": 1,
        "operationType": "freeze_funds",
        "resourceType": "project",
        "resourceId": 123,
        "beforeState": {...},
        "afterState": {...},
        "createdAt": "2026-03-17T22:00:00Z"
      }
    ],
    "total": 500,
    "page": 1,
    "pageSize": 20
  }
}
```
