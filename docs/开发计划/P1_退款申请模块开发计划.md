# P1_退款申请模块开发计划

> **优先级**: P1 (高优先级)
> **工作量**: 3人日
> **完成时间**: 第二周
> **依赖模块**: 预约模块、项目模块

---

## 一、模块概述

### 1.1 业务目标

完善退款申请流程，支持用户在不同阶段申请退款，管理员审核后执行退款，确保资金安全。

### 1.2 功能范围

1. **退款申请功能**
   - 用户提交退款申请
   - 填写退款原因
   - 上传证据材料

2. **退款审核功能**
   - 管理员审核退款申请
   - 批准/拒绝退款
   - 执行退款

### 1.3 业务流程

```
用户申请退款 → 管理员审核 → 批准/拒绝 → 执行退款 → 通知用户
```

## 三、技术设计

### 3.1 数据模型

#### 3.1.1 退款申请表 (refund_applications)

```sql
CREATE TABLE refund_applications (
    id BIGSERIAL PRIMARY KEY,
    booking_id BIGINT REFERENCES bookings(id),
    project_id BIGINT REFERENCES projects(id),
    user_id BIGINT NOT NULL REFERENCES users(id),
    refund_type VARCHAR(20) NOT NULL, -- intent_fee/design_fee/construction_fee/full
    refund_amount DECIMAL(10,2) NOT NULL,
    reason TEXT NOT NULL,
    evidence TEXT[], -- 证据材料URL数组
    status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending/approved/rejected/completed
    admin_id BIGINT REFERENCES admins(id),
    admin_notes TEXT, -- 审核意见
    approved_at TIMESTAMP,
    rejected_at TIMESTAMP,
    completed_at TIMESTAMP, -- 退款完成时间
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_refund_applications_booking_id ON refund_applications(booking_id);
CREATE INDEX idx_refund_applications_project_id ON refund_applications(project_id);
CREATE INDEX idx_refund_applications_user_id ON refund_applications(user_id);
CREATE INDEX idx_refund_applications_status ON refund_applications(status);
```

### 3.2 后端接口设计

#### 3.2.1 退款申请接口

**POST /api/v1/bookings/:id/refund**

提交退款申请

**请求参数**:
```json
{
  "refundType": "intent_fee",
  "reason": "设计方案不满意，申请退款",
  "evidence": ["https://oss.example.com/evidence1.jpg"]
}
```

**响应**:
```json
{
  "code": 0,
  "message": "退款申请已提交",
  "data": {
    "refundApplication": {
      "id": 999,
      "bookingId": 456,
      "refundAmount": 1000,
      "status": "pending",
      "createdAt": "2026-03-17T20:00:00Z"
    }
  }
}
```

---

**GET /api/v1/refunds/my**

查询我的退款申请

**响应**:
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 999,
        "refundType": "intent_fee",
        "refundAmount": 1000,
        "status": "pending",
        "createdAt": "2026-03-17T20:00:00Z"
      }
    ]
  }
}
```

---

#### 3.2.2 退款审核接口

**GET /api/v1/admin/refunds**

查询退款申请列表

**请求参数**:
- `status`: pending/approved/rejected/completed
- `page`: 页码
- `pageSize`: 每页数量

**响应**:
```json
{
  "code": 0,
  "data": {
    "list": [...],
    "total": 20,
    "page": 1,
    "pageSize": 20
  }
}
```

---

**GET /api/v1/admin/refunds/:id**

查询退款申请详情

**响应**:
```json
{
  "code": 0,
  "data": {
    "refundApplication": {
      "id": 999,
      "user": {...},
      "booking": {...},
      "refundType": "intent_fee",
      "refundAmount": 1000,
      "reason": "设计方案不满意",
      "evidence": [...],
      "status": "pending"
    }
  }
}
```

---

**POST /api/v1/admin/refunds/:id/approve**

批准退款

**请求参数**:
```json
{
  "adminNotes": "退款理由合理，批准退款"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "退款已批准，正在执行"
}
```

---

**POST /api/v1/admin/refunds/:id/reject**

拒绝退款

**请求参数**:
```json
{
  "adminNotes": "退款理由不充分，拒绝退款"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "退款已拒绝"
}
```
