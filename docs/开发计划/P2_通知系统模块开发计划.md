# P2_通知系统模块开发计划

> **优先级**: P2 (中优先级)
> **工作量**: 3人日
> **完成时间**: 第三周
> **依赖模块**: 所有业务模块

---

## 一、模块概述

### 1.1 业务目标

完善通知系统，确保所有关键节点都发送通知，支持多渠道通知（短信、推送、站内信）。

### 1.2 功能范围

1. **通知发送功能**
   - 自动发送通知
   - 支持多渠道（短信、推送、站内信）
   - 支持模板化

2. **通知查询功能**
   - 查询通知列表
   - 标记已读
   - 删除通知

---

## 二、需求分析

### 2.1 用户故事

**US-1**: 作为系统，我希望能够自动发送通知，以便用户及时了解进展。

**验收标准**:
- ✅ 所有关键节点自动发送通知
- ✅ 支持多渠道（短信、推送、站内信）
- ✅ 支持模板化
- ✅ 发送失败自动重试

---

**US-2**: 作为用户，我希望能够查看通知，以便了解进展。

**验收标准**:
- ✅ 可以查看通知列表
- ✅ 可以标记已读
- ✅ 可以删除通知
- ✅ 支持分页查询

---

## 三、技术设计

### 3.1 后端接口设计

#### 3.1.1 通知发送接口

**POST /api/v1/notifications/send**

发送通知（内部接口）

**请求参数**:
```json
{
  "userId": 123,
  "type": "proposal_submitted",
  "channels": ["sms", "push", "in_app"],
  "data": {
    "proposalId": 456,
    "providerName": "张设计师"
  }
}
```

---

#### 3.1.2 通知查询接口

**GET /api/v1/notifications**

查询通知列表

**请求参数**:
- `page`: 页码
- `pageSize`: 每页数量

**响应**:
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 3001,
        "type": "proposal_submitted",
        "title": "设计方案已提交",
        "content": "张设计师已提交设计方案，请查看",
        "isRead": false,
        "createdAt": "2026-03-17T23:00:00Z"
      }
    ],
    "total": 50,
    "page": 1,
    "pageSize": 20
  }
}
```

---

**PUT /api/v1/notifications/:id/read**

标记已读

**响应**:
```json
{
  "code": 0,
  "message": "已标记为已读"
}
```

---

**DELETE /api/v1/notifications/:id**

删除通知

**响应**:
```json
{
  "code": 0,
  "message": "删除成功"
}
```
