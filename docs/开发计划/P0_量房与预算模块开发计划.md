# P0_量房与预算模块开发计划

> ⚠️ 已过期（2026-04-16）  
> 本文档仅保留历史背景，请以 `docs/产品需求文档(PRD).md` 与 `docs/BUSINESS_FLOW.md` 的当前主链口径为准。

> **优先级**: P0 (关键路径)
> **工作量**: 3人日
> **完成时间**: 第一周
> **依赖模块**: 预约模块、设计方案模块

## 一、模块概述

### 1.1 业务目标

完善设计沟通阶段的量房记录和预算确认功能，使用户与设计师能够完整体验从预约到设计方案提交的全流程。

### 1.2 功能范围

1. **量房记录功能**
   - 设计师上传量房照片
   - 设计师记录房屋尺寸
   - 用户查看量房资料（只读，不作为确认节点）

2. **预算确认功能**
   - 设计师提供预算范围估算
   - 用户确认预算可接受
   - 设计意向确认

### 1.3 业务流程

```
预约创建 → 设计师承接 → 量房记录 → 预算确认 → 设计方案提交
```

## 二、需求分析

### 2.1 用户故事

**US-1**: 作为设计师，我希望能够上传量房照片和记录尺寸，以便为用户提供准确的设计方案。

**验收标准**:
- ✅ 可以上传多张量房照片（最多 20 张）
- ✅ 可以记录房屋各区域尺寸
- ✅ 可以添加量房备注
- ✅ 提交后用户可见

---

**US-2**: 作为用户，我希望能够查看量房进度和照片，以便了解设计师的工作进展。

**验收标准**:
- ✅ 可以查看量房照片
- ✅ 可以查看房屋尺寸记录
- ✅ 量房资料仅查看，不触发主链确认动作
- ✅ 正式确认从预算/沟通确认开始

---

**US-3**: 作为设计师，我希望能够提供预算范围估算，以便用户了解大致费用。

**验收标准**:
- ✅ 可以输入预算范围（最小值-最大值）
- ✅ 可以选择预算包含项（设计费/施工费/材料费）
- ✅ 可以添加预算说明
- ✅ 提交后用户可见

---

**US-4**: 作为用户，我希望能够确认预算范围，以便进入设计方案阶段。

**验收标准**:
- ✅ 可以查看预算范围
- ✅ 可以确认预算可接受
- ✅ 可以拒绝预算并终止
- ✅ 确认后进入设计方案阶段

## 三、技术设计

### 3.1 数据模型

#### 3.1.1 量房记录表 (site_surveys)

```sql
CREATE TABLE site_surveys (
    id BIGSERIAL PRIMARY KEY,
    booking_id BIGINT NOT NULL REFERENCES bookings(id),
    provider_id BIGINT NOT NULL REFERENCES providers(id),
    photos TEXT[], -- 量房照片 URL 数组
    dimensions JSONB, -- 房屋尺寸记录
    notes TEXT, -- 量房备注
    status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending/submitted/confirmed
    submitted_at TIMESTAMP,
    confirmed_at TIMESTAMP,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_site_surveys_booking_id ON site_surveys(booking_id);
CREATE INDEX idx_site_surveys_provider_id ON site_surveys(provider_id);
```

**dimensions 字段结构**:
```json
{
  "living_room": {
    "length": 5.2,
    "width": 4.8,
    "height": 2.8,
    "unit": "m"
  },
  "bedroom_1": {
    "length": 4.0,
    "width": 3.5,
    "height": 2.8,
    "unit": "m"
  }
}
```

---

#### 3.1.2 预算确认表 (budget_confirmations)

```sql
CREATE TABLE budget_confirmations (
    id BIGSERIAL PRIMARY KEY,
    booking_id BIGINT NOT NULL REFERENCES bookings(id),
    provider_id BIGINT NOT NULL REFERENCES providers(id),
    budget_min DECIMAL(10,2) NOT NULL, -- 预算最小值
    budget_max DECIMAL(10,2) NOT NULL, -- 预算最大值
    includes JSONB, -- 预算包含项
    notes TEXT, -- 预算说明
    status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending/confirmed/rejected
    confirmed_at TIMESTAMP,
    rejected_at TIMESTAMP,
    rejection_reason TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_budget_confirmations_booking_id ON budget_confirmations(booking_id);
CREATE INDEX idx_budget_confirmations_provider_id ON budget_confirmations(provider_id);
```

**includes 字段结构**:
```json
{
  "design_fee": true,
  "construction_fee": true,
  "material_fee": true,
  "furniture_fee": false
}
```

### 3.2 后端接口设计

#### 3.2.1 量房记录接口

**POST /api/v1/bookings/:id/site-survey**

提交量房记录

**请求参数**:
```json
{
  "photos": ["https://oss.example.com/photo1.jpg", "https://oss.example.com/photo2.jpg"],
  "dimensions": {
    "living_room": {
      "length": 5.2,
      "width": 4.8,
      "height": 2.8,
      "unit": "m"
    }
  },
  "notes": "客厅采光良好，建议使用浅色系"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "量房记录提交成功",
  "data": {
    "siteSurvey": {
      "id": 123,
      "bookingId": 456,
      "status": "submitted",
      "submittedAt": "2026-03-17T10:00:00Z"
    }
  }
}
```

---

**GET /api/v1/bookings/:id/site-survey**

查询量房记录

{

## 六、验收标准

### 6.1 功能验收

- ✅ 设计师可以提交量房记录
- ✅ 用户可以查看量房记录
- ✅ 量房资料节点为用户只读查看，不作为主链确认动作
- ✅ 设计师可以提交预算确认
- ✅ 用户可以确认/拒绝预算
- ✅ 确认预算后可进入设计方案阶段

### 6.2 质量验收

- ✅ 所有接口有单元测试
- ✅ 所有页面有功能测试
- ✅ 代码覆盖率 ≥ 80%

### 6.3 性能验收

- ✅ 照片上传时间 < 3s (单张)
- ✅ 接口响应时间 < 500ms
- ✅ 页面加载时间 < 2s

## 七、风险与应对

### 7.1 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 照片上传性能问题 | 🟡 中 | 使用 OSS 直传，后端只存储 URL |
| 尺寸记录格式不统一 | 🟢 低 | 使用 JSONB 存储，前端统一格式 |

### 7.2 进度风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 前端页面开发慢 | 🟡 中 | 复用现有组件，减少重复开发 |
