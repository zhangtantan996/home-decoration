# P1_暂停与争议模块开发计划

> **优先级**: P1 (高优先级)
> **工作量**: 3人日
> **完成时间**: 第二周
> **依赖模块**: 项目执行模块

---

## 一、模块概述

### 1.1 业务目标

完善项目执行中的异常处理机制，支持用户或工长暂停施工、提交争议，确保资金安全和业务流程可控。

### 1.2 功能范围

1. **暂停施工功能**
   - 用户/工长申请暂停施工
   - 冻结后续款项
   - 恢复施工

2. **争议处理功能**
   - 用户/工长提交争议
   - 冻结所有款项
   - 平台介入仲裁

### 1.3 业务流程

```
项目执行中 → 暂停申请 → 资金冻结 → 恢复施工 / 提交争议 → 仲裁
```

---

### 3.2 后端接口设计

#### 3.2.1 暂停施工接口

**POST /api/v1/projects/:id/pause**

申请暂停施工

**请求参数**:
```json
{
  "reason": "用户家中有急事，需要暂停施工2周",
  "initiator": "user" // user/foreman
}
```

**响应**:
```json
{
  "code": 0,
  "message": "暂停申请已提交",
  "data": {
    "project": {
      "id": 123,
      "status": "paused",
      "pausedAt": "2026-03-17T16:00:00Z"
    }
  }
}
```

---

**POST /api/v1/projects/:id/resume**

恢复施工

**响应**:
```json
{
  "code": 0,
  "message": "施工已恢复",
  "data": {
    "project": {
      "id": 123,
      "status": "in_progress"
    }
  }
}
```

---

#### 3.2.2 争议处理接口

**POST /api/v1/projects/:id/dispute**

提交争议

**请求参数**:
```json
{
  "reason": "施工质量不符合要求",
  "evidence": ["https://oss.example.com/evidence1.jpg", "https://oss.example.com/evidence2.jpg"]
}
```

**响应**:
```json
{
  "code": 0,
  "message": "争议已提交，平台将介入处理",
  "data": {
    "project": {
      "id": 123,
      "status": "disputed",
      "disputedAt": "2026-03-17T17:00:00Z"
    }
  }
}
```
