# P0_项目完工模块开发计划

> **优先级**: P0 (关键路径)
> **工作量**: 2人日
> **完成时间**: 第一周
> **依赖模块**: 项目执行模块、验收模块

---

## 一、模块概述

### 1.1 业务目标

完善项目完工流程，支持工长提交完工、生成灵感案例草稿，使项目能够正常进入验收和关闭阶段。

### 1.2 功能范围

1. **项目完工提交**
   - 工长提交项目完工
   - 上传完工照片
   - 填写完工说明

2. **灵感案例草稿生成**
   - 自动生成案例草稿
   - 进入审核链
   - 默认不自动公开

### 1.3 业务流程

```
项目执行 → 工长提交完工 → 生成案例草稿 → 用户验收 → 项目关闭
```

---

## 二、需求分析

### 2.1 用户故事

**US-1**: 作为工长，我希望能够提交项目完工，以便进入验收阶段。

**验收标准**:
- ✅ 可以上传完工照片（最多 30 张）
- ✅ 可以填写完工说明
- ✅ 提交后自动生成案例草稿
- ✅ 用户收到完工通知

---

**US-2**: 作为平台，我希望能够自动生成灵感案例草稿，以便沉淀优质案例。

**验收标准**:
- ✅ 完工后自动生成案例草稿
- ✅ 案例草稿包含项目信息、照片、设计师信息
- ✅ 案例草稿进入审核链
- ✅ 默认不自动公开

---

## 三、技术设计

### 3.1 数据模型

#### 3.1.1 项目表扩展 (projects)

```sql
ALTER TABLE projects ADD COLUMN IF NOT EXISTS inspiration_case_draft_id BIGINT REFERENCES case_audits(id);
ALTER TABLE projects ADD COLUMN IF NOT EXISTS completed_photos TEXT[];
ALTER TABLE projects ADD COLUMN IF NOT EXISTS completion_notes TEXT;
```

---

### 3.2 后端接口设计

#### 3.2.1 项目完工接口

**POST /api/v1/projects/:id/complete**

提交项目完工

**请求参数**:
```json
{
  "photos": ["https://oss.example.com/complete1.jpg", "https://oss.example.com/complete2.jpg"],
  "notes": "项目已按时完工，质量符合要求"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "项目完工提交成功",
  "data": {
    "project": {
      "id": 123,
      "status": "completed",
      "completedAt": "2026-03-17T15:00:00Z",
      "inspirationCaseDraftId": 456
    }
  }
}
```

---

**GET /api/v1/projects/:id/inspiration-draft**

查询灵感案例草稿

**响应**:
```json
{
  "code": 0,
  "data": {
    "caseDraft": {
      "id": 456,
      "projectId": 123,
      "title": "现代简约风格三居室",
      "photos": [...],
      "status": "pending_review",
      "createdAt": "2026-03-17T15:00:00Z"
    }
  }
}
```
