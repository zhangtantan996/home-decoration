## 1. 设计理念

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **信任感** | 通过透明化设计建立用户信任 |
| **即时反馈** | 实时状态展示，消除信息焦虑 |
| **简洁高效** | 减少操作步骤，降低认知负担 |
| **差异化体验** | 针对不同用户群体定制界面 |

### 1.2 设计参考

- 美团/大众点评：LBS服务发现
- 支付宝：资金安全感设计
- 滴滴：服务匹配与实时追踪

## 2. 设计规范

### 2.1 色彩系统

**主色调**
| 名称 | 色值 | 用途 |
|------|------|------|
| 品牌蓝 | #1890FF | 主按钮、链接 |
| 安全绿 | #52C41A | 成功、正常状态 |
| 预警黄 | #FAAD14 | 警告、滞后状态 |
| 危险红 | #FF4D4F | 错误、异常状态 |

**中性色**
| 名称 | 色值 | 用途 |
|------|------|------|
| 标题黑 | #1F1F1F | 主标题 |
| 正文灰 | #4A4A4A | 正文内容 |
| 辅助灰 | #8C8C8C | 辅助信息 |
| 背景色 | #F5F5F5 | 页面背景 |

### 2.2 字体规范

| 层级 | 大小 | 行高 | 字重 | 用途 |
|-----|------|-----|------|------|
| H1 | 28px | 40px | Bold | 页面标题 |
| H2 | 22px | 32px | Bold | 模块标题 |
| H3 | 18px | 26px | Medium | 卡片标题 |
| Body | 16px | 24px | Regular | 正文 |
| Caption | 14px | 20px | Regular | 辅助文字 |
| Small | 12px | 18px | Regular | 备注信息 |

**工人端特殊规范**: 最小字号 18px，重要信息 24px+

### 2.3 间距系统

基准单位: 8px

| 名称 | 数值 | 用途 |
|------|-----|------|
| xs | 4px | 紧凑间距 |
| sm | 8px | 小间距 |
| md | 16px | 标准间距 |
| lg | 24px | 大间距 |
| xl | 32px | 区块间距 |

### 2.4 圆角规范

| 类型 | 圆角 | 用途 |
|------|-----|------|
| 按钮 | 8px | 标准按钮 |
| 卡片 | 12px | 内容卡片 |
| 弹窗 | 16px | Modal/Dialog |
| 头像 | 50% | 圆形头像 |

## 5. 交互设计

### 5.1 滑动支付确认

**目的**: 增强用户对资金的掌控感，避免误触

**交互流程**:
1. 展示验收材料与金额
2. 用户从左向右滑动滑块
3. 滑动超过80%触发确认
4. 震动反馈 + 成功动效
5. 显示支付结果

### 5.2 AI质量检测反馈

**交互设计**:
- 检测中：旋转扫描动效
- 合格：绿色对勾 + 通过标识
- 问题：红框标记问题区域 + 问题描述浮层

### 5.3 实时工地状态

**状态指示**:
| 状态 | 颜色 | 图标 | 说明 |
|------|-----|------|------|
| 正常 | 🟢 | ✓ | 进度正常 |
| 滞后 | 🟡 | ⚠ | 延期3天内 |
| 异常 | 🔴 | ✕ | 重大问题 |
