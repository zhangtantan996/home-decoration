# 项目概览 - Home Decoration Platform

## 🎯 项目定位

**装修设计一体化平台** - 连接业主、设计师、装修公司、工头和工人的综合服务平台。

## 核心价值主张

- **业主**：找到靠谱的设计师和施工团队，全程透明监管
- **设计师**：展示作品，获取客户，提供设计服务
- **装修公司**：接单、管理项目、协调资源
- **工头/工人**：接活、展示技能、获得收入

## 🏗️ 项目结构

```
home-decoration/
├── server/          # Go 后端（Gin + GORM + PostgreSQL）
├── admin/           # React Admin 管理后台（React 18.3.1 + Ant Design）
├── mobile/          # React Native 移动应用（React 19.2.0，仅原生）
├── mini/            # 微信小程序（Taro 3.x + React 18.3.1）
├── docs/            # 产品文档（PRD、设计规范）
├── deploy/          # 部署配置（Docker、Nginx）
└── llmdoc/          # AI 优化的项目文档（本目录）
```

## 🎨 核心功能模块

### 1. 用户端（Mobile + Mini Program）
- **首页**：浏览服务商、设计灵感
- **灵感**：设计案例展示
- **进度**：项目施工进度跟踪
- **消息**：实时聊天（IM 系统）
- **我的**：个人中心、订单管理

### 2. 管理端（Admin Panel）
- **仪表盘**：数据统计、趋势分析
- **用户管理**：用户列表、详情、操作日志
- **服务商管理**：审核、认证、评级
- **项目管理**：列表视图、地图视图
- **预约管理**：预约审核、状态跟踪
- **财务管理**：托管账户、交易记录
- **风险管理**：预警、仲裁
- **系统设置**：RBAC、菜单、配置

### 3. 托管支付系统（Escrow）
- **资金托管**：业主预付款托管
- **分阶段支付**：按里程碑释放资金
- **退款机制**：超时自动退款
- **商家结算**：定时结算到商家账户
- **交易记录**：完整的审计日志

### 4. IM 系统（正在迁移）
- **旧系统**：自研 WebSocket Hub（已废弃）
- **新系统 1**：Tinode 开源 IM（集成中）
- **新系统 2**：腾讯云 IM SDK（集成中）
- **功能**：一对一聊天、群聊、消息推送

### 5. 预约系统（Booking）
- **预约流程**：选择服务商 → 选择时间 → 支付意向金 → 确认
- **超时处理**：定时任务检查超时预约，自动退款
- **状态管理**：pending → confirmed → completed → cancelled

### 6. 项目管理（Project）
- **项目阶段**：设计 → 施工 → 验收 → 完工
- **里程碑**：按阶段设置里程碑，验收后释放资金
- **任务管理**：每个阶段的具体任务
- **进度跟踪**：实时更新施工进度

### 7. 多身份切换系统（Identity）
- **一账户多身份**：用户可同时拥有业主、服务商、工人等多个身份
- **身份切换**：在不同身份之间无缝切换，自动更新 Token 和权限
- **身份申请**：用户可申请新身份，管理员审核后激活
- **审计日志**：所有身份切换操作都有完整的审计记录

## 🔐 安全特性

### 认证系统
- **JWT Token**：用户端和管理端认证
- **Refresh Token**：自动刷新机制
- **微信登录**：小程序微信登录集成
- **手机验证**：短信验证码（待集成）

### 授权系统
- **RBAC**：基于角色的权限控制
- **菜单权限**：动态菜单生成
- **API 权限**：细粒度的接口权限
- **数据权限**：用户只能访问自己的数据

### 数据安全
- **密码加密**：bcrypt 哈希
- **敏感数据加密**：AES 加密（身份证号、银行卡号）
- **SQL 注入防护**：GORM 参数化查询
- **XSS 防护**：前端输入过滤
- **CORS 白名单**：严格的跨域控制

### 财务安全
- **事务管理**：所有资金操作使用数据库事务
- **悲观锁**：防止并发修改余额
- **审计日志**：所有财务操作记录
- **双重验证**：关键操作需要二次确认

- 缓存：Redis（ElastiCache）
