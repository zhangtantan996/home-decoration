# 项目结构

**最后更新**: 2026-01-25

## Monorepo 架构

本项目采用 **Monorepo**（单仓多包）结构，将后端服务、管理后台、移动端应用及小程序集中在一个仓库中管理。这种架构的优势在于：
- **代码复用**：方便在不同子项目间共享类型定义和逻辑（如接口定义、常量等）。
- **原子提交**：跨多个端的变更可以在同一个 Git 提交中完成，保持逻辑一致性。
- **依赖一致**：统一管理第三方依赖版本，减少版本冲突。

## 目录结构总览

```text
home-decoration/
├── server/          # Go 后端 (Gin + GORM)
├── admin/           # 管理后台 (React + Ant Design)
├── mobile/          # 移动端 App (React Native)
├── mini/            # 微信小程序 (Taro)
├── deploy/          # 部署与运维配置 (Docker/Nginx)
├── tests/           # 自动化测试 (Playwright E2E)
├── documentation/   # 项目标准化文档（本目录）
├── docs/            # 历史参考文档
├── scripts/         # 辅助开发脚本
├── .sisyphus/       # AI Agent 工作空间
└── package.json     # 根工作区配置
```

## 各子项目说明

### 1. server/ - Go 后端
**职责**: 提供全平台的 REST API 和 WebSocket 实时通讯服务，处理业务逻辑、数据库交互及权限控制。
**关键目录**:
- `cmd/api/`: 应用入口点（main.go）。
- `internal/handler/`: HTTP 处理器，负责解析请求和调用服务。
- `internal/service/`: 核心业务逻辑层。
- `internal/repository/`: 数据访问层，直接操作数据库。
- `internal/model/`: 数据库模型定义。
- `pkg/`: 可导出的公共工具包。
- `migrations/`: 数据库迁移文件。

### 2. admin/ - 管理后台
**职责**: 面向平台管理员和运营人员，负责用户审核、服务商管理、财务监控及系统配置。
**关键目录**:
- `src/pages/`: 各业务模块页面（如用户列表、项目管理、财务审核）。
- `src/components/`: 通用 UI 组件。
- `src/services/`: API 请求封装。
- `src/stores/`: 基于 Zustand 的状态管理。
- `src/layouts/`: 页面布局组件。

### 3. mobile/ - 移动端
**职责**: 面向业主、设计师和工人，提供跨平台（iOS/Android/Web）的移动体验，包含工地监控、即时聊天和预约功能。
**关键目录**:
- `src/screens/`: 移动端页面（首页、消息、我的工地等）。
- `src/navigation/`: 导航路由配置。
- `src/store/`: 状态管理（含认证和聊天状态）。
- `android/` & `ios/`: 原生代码工程目录。
- `src/services/WebSocketService.ts`: 实时通讯实现。

### 4. mini/ - 微信小程序
**职责**: 轻量级入口，主要用于品牌展示、简单预约和进度查询。
**关键目录**:
- `src/pages/`: 小程序页面。
- `src/components/`: Taro 组件。
- `src/utils/request.ts`: 微信网络请求封装。

### 5. deploy/ - 部署配置
**职责**: 包含生产环境和开发环境的容器化部署配置。
- `nginx/`: Nginx 代理与静态资源配置。
- `docker-compose.prod.yml`: 生产环境编排。
- `Dockerfile.*`: 各模块的镜像构建脚本。

### 6. tests/ - E2E 测试
**职责**: 使用 Playwright 进行端到端自动化测试，模拟用户真实操作流程，确保核心链路稳定性。

## 根目录文件说明

- `package.json`: 定义 Monorepo 的 Workspaces 和全局开发依赖。
- `docker-compose.yml`: 标准 Docker 编排配置。
- `docker-compose.local.yml`: 本地开发环境配置（一键启动数据库、Redis 等中间件）。
- `AGENTS.md`: AI 开发助手的操作指引与约束规范。
- `项目结构说明.md`: 原始的项目结构参考文档（详细版）。

## 开发导航建议
- **后端逻辑变更**: 关注 `server/internal/service/`。
- **管理后台 UI 调整**: 关注 `admin/src/pages/`。
- **移动端功能迭代**: 关注 `mobile/src/screens/`。
- **环境搭建与部署**: 查看 `deploy/` 及根目录的 `docker-compose.local.yml`。
