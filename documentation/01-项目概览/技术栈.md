# 技术栈

**最后更新**: 2026-01-25

## 概览
本项目采用前后端分离的现代化技术架构，针对不同终端（管理后台、移动端、小程序）选择了最适合的工具链。核心原则是“稳定性优先、体验一致、分端优化”。

## 后端技术栈
后端基于 Go 语言构建，遵循 `handler -> service -> repository` 严谨分层架构。

| 技术 | 版本 | 用途 | 约束说明 |
|------|------|------|----------|
| Go | 1.21 🔒 | 核心开发语言 | `server/go.mod` 声明版本 |
| Gin | v1.9.1 | Web 框架 | 高性能 HTTP 路由 |
| GORM | v1.30.0 | 数据库 ORM | 复杂 SQL 映射与事务管理 |
| JWT | v5.2.0 | 认证授权 | 基于 `golang-jwt/jwt/v5` |
| Redis Client | v9.17.2 | 缓存访问 | `go-redis/v9` |
| Viper | v1.18.2 | 配置管理 | 支持多环境 yaml 配置 |

## 版本约束说明

### React 版本分裂详解 ⚠️
本项目存在明确的 React 版本分裂，**禁止统一版本**：

1. **Admin Panel (React 18.3.1)**:
   - **原因**: 核心依赖 `Ant Design 5.x` 和 `Tencent Cloud Chat SDK` 目前对 React 19 的兼容性不完整，升级会导致渲染异常或生命周期冲突。
   - **操作建议**: `package.json` 中已取消 `^` 前缀，必须保持精确版本。

2. **Mobile App (React 19.2.0)**:
   - **原因**: `React Native 0.83` 已原生支持 React 19。移动端业务较新，旨在利用 React 19 的新特性（如改进的并发渲染、更简单的 Hook 使用等）来提升交互流畅度。

### 后端约束
- **Go 1.21**: 虽部分文档提及 1.23，但目前核心逻辑依赖 `go 1.21` 特性。在未进行完整回归测试前，不得提升 `go.mod` 声明版本。
- **Gin**: 统一使用 Gin，禁止引入 Echo 或 Fiber 等其他 Web 框架以降低维护成本。
