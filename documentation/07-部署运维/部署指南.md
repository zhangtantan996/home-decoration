# 部署指南

**最后更新**: 2026-01-25

本文档详细介绍了如何将 Home Decoration 项目部署到生产环境，涵盖服务器要求、环境搭建、服务启动及验证流程。

## 服务器要求

为了保证系统的稳定运行，建议服务器满足以下配置要求。

### 硬件要求
- **CPU**: 2核+ (推荐 4核+)
- **内存**: 4GB+ (推荐 8GB+)。*注意：如果内存小于 4GB，建议配置 Swap 分区以防止构建过程中内存溢出。*
- **磁盘**: 50GB+ (SSD 优先)，用于存储数据库数据、日志及 Docker 镜像。

### 软件要求
- **操作系统**: Ubuntu 20.04/22.04 LTS 或 CentOS 7.x/8.x。
- **Docker**: Engine 20.10.x+。
- **Docker Compose**: 2.0.x+。
- **Nginx**: 用于反向代理和 HTTPS 卸载。
- **Git**: 用于拉取代码。

## 部署流程

### 1. 准备工作

#### 连接服务器
使用 SSH 连接到您的生产服务器：
```bash
ssh root@your_server_ip
```

#### 创建项目目录
建议将项目部署在 `/www` 目录下：
```bash
sudo mkdir -p /www/home_decoration_prod
sudo chown -R $USER:$USER /www
```

#### 拉取代码
```bash
cd /www/home_decoration_prod
git clone https://your-repo-url/home_decoration.git .
```

### 2. 配置环境变量

项目使用 `.env` 文件管理生产环境配置。

1. **复制模板文件**:
   ```bash
   cp .env.example .env
   ```

2. **编辑配置**:
   使用 `vi` 或 `nano` 修改 `.env` 文件，确保以下关键项已正确配置：
   - `DB_PASSWORD`: 生产环境数据库密码（切勿使用默认值）。
   - `REDIS_PASSWORD`: Redis 访问密码。
   - `JWT_SECRET`: 用于生成 Token 的随机密钥。
   - `SERVER_MODE`: 设置为 `release`。

### 3. 启动服务

我们推荐使用 Docker Compose 进行一键部署，它会自动处理各组件（API、数据库、缓存、前端）的关联。

```bash
# 进入部署目录
cd deploy

# 启动并构建镜像
docker compose -f docker-compose.prod.yml up -d --build
```

*注：首次部署可能需要较长时间下载基础镜像并构建前端资源。*

### 4. 验证部署

部署完成后，通过以下步骤确认服务运行状态：

- **检查容器状态**:
  ```bash
  docker compose -f docker-compose.prod.yml ps
  ```
  所有容器（server, admin, db, redis, nginx）应显示为 `Up` 或 `running`。

- **查看服务日志**:
  ```bash
  docker compose -f docker-compose.prod.yml logs -f server
  ```

- **健康检查接口**:
  访问 `http://your_server_ip/api/health`，若返回 `{"status": "ok"}` 则表示后端服务运行正常。

## 部署检查清单

在正式对外开放服务前，请根据此清单进行逐一核对。

### 1. 部署前准备 (Pre-deployment)
- [ ] **硬件资源**: 服务器资源满足最低要求（2核/4GB/50GB+）。
- [ ] **环境依赖**: 已安装 Docker、Docker Compose、Git 及 Nginx。
- [ ] **密钥生成**: 已使用 `openssl rand -base64 64` 生成独立的 `JWT_SECRET` 和 `ENCRYPTION_KEY`。
- [ ] **强密码策略**: 数据库（`DB_PASSWORD`）和 Redis（`REDIS_PASSWORD`）已设置为高强度随机密码。
- [ ] **配置确认**: `.env` 文件已根据 `.env.example` 创建，且关键项已修改为生产环境值。
- [ ] **文件权限**: `.env` 文件权限已设置为 `600`（仅所有者可读写）。
- [ ] **备份现状**: 若为版本升级，已完成现有数据库数据的全量备份。

### 2. 部署执行 (Deployment Execution)
- [ ] **源码拉取**: 生产环境代码已从 `main` 分支拉取到最新版本。
- [ ] **构建成功**: 执行 `docker compose build` 未出现报错。
- [ ] **容器状态**: 运行 `docker compose ps`，所有服务（server, admin, db, redis, nginx）均显示为 `running`。
- [ ] **数据库迁移**: 检查 Server 容器日志，确认 GORM 自动迁移（Auto-migration）已成功执行，无 SQL 报错。

### 3. 部署后验证 (Post-deployment Verification)
- [ ] **接口健康状况**: 访问 `http://domain/api/health` 返回 `{"status": "ok"}`。
- [ ] **前端访问**: Admin 管理后台与 Mobile Web 页面能正常加载，且静态资源无 404 错误。
- [ ] **登录验证**: 使用测试账号可成功登录 Admin 后台。
- [ ] **数据联通性**: 能正常查询基础字典数据，证明后端与数据库连接正常。
- [ ] **IM 服务**: (若启用) 确认消息收发功能及 Tinode 状态正常。

### 4. 安全与运维专项 (Security & Ops)
- [ ] **HTTPS 加密**: SSL 证书已正确配置，且 HTTP 请求已自动重定向至 HTTPS。
- [ ] **调试端点保护**: `SERVER_MODE` 已设为 `release`，且 `/api/v1/debug/*` 端点返回 404 或已限制访问。
- [ ] **安全头验证**: 响应头中包含 `X-Frame-Options`, `X-Content-Type-Options` 等安全增强项。
- [ ] **速率限制**: 登录接口的限流策略（Rate Limiting）已生效。
- [ ] **防火墙配置**: 宿主机防火墙（ufw/firewalld）已限制仅对外开放 80 和 443 端口，禁用 8888 等调试端口。
- [ ] **端口隔离**: 数据库 (5432) 和 Redis (6379) 端口未暴露到公网。
- [ ] **自动重启**: Docker 容器配置了 `restart: always` 策略。
- [ ] **监控告警**: (推荐) 已配置服务器 CPU、内存、磁盘空间的监控及告警阈值。

## 常见问题处理

- **构建时内存不足**: 如果在构建前端镜像时崩溃，请参考 `docs/全栈部署指南.md` 配置 Swap 分区。
- **端口冲突**: 确保宿主机没有其他进程占用 80 或 443 端口。
- **数据库连接失败**: 检查 `.env` 中的 `DB_HOST` 是否与 docker-compose 中的服务名一致。
