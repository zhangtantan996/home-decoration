# 常见问题

**最后更新**: 2026-01-25

本文档汇总了在搭建环境、构建项目以及本地运行过程中常见的问题及其解决方案。

## 环境问题

### Q1: Node.js 版本不匹配
**问题**: 运行 `npm install` 时报错，或者某些依赖包（如 `node-sass` 或 `sharp`）编译失败。
**解决方案**: 
- 本项目要求 **Node.js 20+**。
- 建议使用 `nvm` 管理版本：
  ```bash
  nvm install 20
  nvm use 20
  ```

### Q2: Docker 未启动或 Docker Compose 命令报错
**问题**: 运行 `npm run infra` 报错 `docker: command not found` 或 `Cannot connect to the Docker daemon`。
**解决方案**: 
- 确保已安装 Docker Desktop (Mac/Windows) 或 Docker Engine (Linux)。
- 确保 Docker 服务已启动并在后台运行。
- 在 Linux 上，可能需要使用 `sudo` 或将用户加入 `docker` 组。

### Q3: 端口被占用
**问题**: 启动服务时提示端口冲突（常见端口：5432, 6380, 8080, 5173）。
**解决方案**: 
- 检查占用端口的进程：
  ```bash
  # Mac/Linux
  lsof -i :8080
  # Windows
  netstat -ano | findstr :8080
  ```
- 杀掉对应进程，或者在配置文件（如 `.env` 或 `docker-compose.yml`）中修改端口。
- 注意：本项目 Redis 映射到了 **6380** 端口，避免与本地默认 Redis 冲突。

### Q4: Admin 面板 React 版本冲突
**问题**: Admin 项目启动报错 `Cannot read property 'createElement' of undefined`。
**解决方案**: 
- Admin 项目必须锁定使用 **React 18.3.1**（Ant Design 5.x 暂不完全支持 React 19）。
- 检查 `admin/package.json`，确保 `react` 和 `react-dom` 前没有 `^` 或 `~` 符号。
- 修复步骤：
  ```bash
  cd admin
  npm install react@18.3.1 react-dom@18.3.1
  rm -rf node_modules package-lock.json
  npm install
  ```

## 构建问题

### Q1: npm install 速度慢或下载失败
**问题**: 下载依赖包超时或报错。
**解决方案**: 
- 切换到国内镜像源（如淘宝镜像）：
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```

### Q2: Go 依赖下载失败
**问题**: `go mod download` 无法连接到 `proxy.golang.org`。
**解决方案**: 
- 设置 GOPROXY 代理：
  ```bash
  # Mac/Linux
  export GOPROXY=https://goproxy.cn,direct
  # Windows (PowerShell)
  $env:GOPROXY = "https://goproxy.cn,direct"
  ```

### Q3: 前端构建内存不足 (Out of Memory)
**问题**: Docker 构建 Admin 镜像时报错 `JavaScript heap out of memory`。
**解决方案**: 
- 增加 Node.js 的内存限制。在 Dockerfile 或环境变量中设置：
  ```bash
  export NODE_OPTIONS="--max-old-space-size=8192"
  ```
- 确保 Docker 分配给容器的内存至少有 4GB。

### Q4: 缺少 TUIKit 相关依赖
**问题**: 构建时报错 `Cannot find module '@tencentcloud/tui-core-lite'`。
**解决方案**: 
- 腾讯云 IM SDK 使用了 peerDependencies，需要手动安装：
  ```bash
  cd admin
  npm install @tencentcloud/tui-core-lite @tencentcloud/chat-uikit-engine-lite @tencentcloud/uikit-base-component-react
  ```

## 运行问题

### Q1: 数据库连接失败
**问题**: 后端启动报错 `dial tcp [::1]:5432: connect: connection refused`。
**解决方案**: 
- 检查 `server/.env` 中的 `DB_HOST` 是否正确。
- 如果在宿主机运行后端，`DB_HOST` 应为 `localhost`；如果在 Docker 中运行，应为 `db`。
- 确保数据库容器已启动且健康。

### Q2: API 跨域问题 (CORS)
**问题**: 浏览器报错 `Access to XMLHttpRequest has been blocked by CORS policy`。
**解决方案**: 
- 检查后端 `server/internal/middleware/cors.go` 是否配置了正确的允许域名。
- 开发环境下通常配置为 `*`，生产环境下请务必配置具体域名。

### Q3: 移动端 API 无法访问
**问题**: 真机调试或模拟器无法调用 API。
**解决方案**: 
- 检查 API 地址是否为 `localhost` 或 `127.0.0.1`。
- 移动端必须使用开发机的 **局域网 IP**（如 `http://192.168.1.100:8080`）。
- 确保手机和电脑在同一个 WiFi 下。

### Q4: 列表显示数据不正确（如头像、名称）
**问题**: 首页显示的设计师名称为“独立设计师”，头像显示的是封面图。
**解决方案**: 
- 这是由于 `users` 表缺少对应种子数据导致的关联查询失败。
- 重新运行数据库初始化脚本，或手动补全 `users` 表中的记录，确保 `providers.user_id` 在 `users` 表中有对应项。

## 其他问题

### 如何重置整个环境？
如果环境彻底搞乱了，可以尝试“核弹级”重置：
```bash
# 1. 停止并删除所有容器和卷
docker-compose down -v

# 2. 删除所有 node_modules
find . -name "node_modules" -type d -prune -exec rm -rf '{}' +

# 3. 重新安装并启动
npm install
npm run infra
cd server && go mod download
```
