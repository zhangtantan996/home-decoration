# IM 功能实现

**最后更新**: 2026-01-25

本文档详细介绍了装修设计一体化平台中即时通讯（IM）功能的实现方式。系统采用 [Tinode](https://tinode.co/) 作为底层 IM 引擎，通过在 Mobile (React Native) 和 Admin (React) 前端集成 Tinode SDK，实现实时消息交互。

## 核心功能

### 1. 消息功能

系统支持多种消息类型，通过 Tinode 的 Drafty 格式进行结构化消息传输。

#### 文本消息
**状态**: ✅ 已完成
支持基础的文字聊天，实时收发。

#### 图片消息
**状态**: ✅ 已完成
支持发送和预览图片。发送流程：
1. 调用业务后端 `/api/v1/upload` 接口上传图片文件。
2. 将返回的图片 URL 封装在 Drafty 的 `IM` 实体中。
3. 通过 Tinode 协议发送消息。

#### 语音消息
**状态**: ✅ 已完成
支持移动端“按住说话”功能，并在管理后台支持播放。
- **录制**: 移动端使用 `react-native-nitro-sound` (或同类库) 录制 AAC/M4A 格式音频。
- **限制**: 最大时长 60 秒，最大文件大小 5MB。
- **预览**: 在消息列表中显示 `【语音 Xs】`。
- **实现**: 使用 Drafty 的 `EX` 实体携带 `mime`, `val` (URL), `duration` 等元数据。

---

### 2. 在线状态
**状态**: ✅ 已完成
系统能够实时追踪用户的在线/离线状态。
- **实现原理**: 通过订阅 Tinode 的 `me` 主题，监听 `onContactUpdate` 回调。
- **前端显示**: 在聊天列表或对话页顶部显示用户的实时在线状态（在线/离线/最后在线时间）。

---

### 3. 打字指示器
**状态**: 🚧 开发中
用于显示对方正在输入的状态。
- **技术路径**: 利用 Tinode 的 `note` 消息（`kp` - key press）。
- **当前进度**: 已在 `TinodeService` 中预留接口，前端 UI 尚未完全适配显示逻辑。

---

### 4. 消息已读
**状态**: ✅ 已完成
支持显示消息的发送状态、送达状态和已读状态。
- **回执机制**: 接收方在查看消息后，调用 `topic.noteRead(seqId)` 向服务器发送已读回执。
- **状态同步**: 发送方通过监听 `onInfo` 事件获取 `read` 通知，从而更新本地消息状态。

## 代码示例

### 发送不同类型的消息 (TinodeService.ts)

```typescript
// 发送文本消息
async sendTextMessage(topicName: string, text: string) {
    const topic = this.tinode.getTopic(topicName);
    await topic.publish(text, true);
}

// 发送图片消息
async sendImageMessage(topicName: string, imageUri: string) {
    const uploadResult = await this.uploadFile(imageUri, 'image/jpeg');
    const content = {
        txt: '[图片]',
        fmt: [{ at: 0, len: 4, tp: 'IM', key: 0 }],
        ent: [{
            tp: 'IM',
            data: {
                mime: 'image/jpeg',
                val: uploadResult.url,
                width: uploadResult.width,
                height: uploadResult.height,
            },
        }],
    };
    await this.tinode.getTopic(topicName).publish(content, true);
}

// 发送语音消息
async sendAudioMessage(topicName: string, audioPath: string, duration: number) {
    const uploadResult = await this.uploadFile(audioPath, 'audio/mp4');
    const content = {
        txt: 'audio.m4a',
        fmt: [{ at: -1, len: 0, key: 0 }],
        ent: [{
            tp: 'EX',
            data: {
                mime: 'audio/mp4',
                val: uploadResult.url,
                duration: duration, // 毫秒
            },
        }],
    };
    await this.tinode.getTopic(topicName).publish(content, true);
}
```

### 标记消息已读

```typescript
async markAsRead(topicName: string, messageSeqId: number) {
    const topic = this.tinode.getTopic(topicName);
    await topic.noteRead(messageSeqId);
}
```

### 处理打字指示器

```typescript
// 发送“正在输入”状态
sendTypingIndicator(topicName: string) {
    const topic = this.tinode.getTopic(topicName);
    topic.noteKeyPress();
}

// 监听“正在输入”状态
topic.onInfo = (info) => {
    if (info.what === 'kp') {
        console.log('对方正在输入...');
    }
};
```

## 注意事项
1. **网络重连**: `TinodeService` 实现了自动重连机制，当 WebSocket 断开时会尝试使用缓存的 Token 重新初始化。
2. **多端同步**: Tinode 原生支持多端同步，移动端和管理后台的操作（如发送消息、标记已读）会实时同步到所有在线设备。
3. **文件上传**: IM 中的多媒体文件（图片、语音）使用业务后端的统一上传接口，不直接通过 Tinode 文件服务（以方便统一管理存储策略）。
