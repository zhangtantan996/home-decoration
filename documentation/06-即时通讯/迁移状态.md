# IM 迁移状态

**最后更新**: 2026-01-25

## 迁移进度

当前进度: **85%**

本阶段迁移目标是将即时通讯系统从腾讯云 IM (Tencent Cloud IM) 迁移至自建的 Tinode 系统，以降低成本并增强对数据的自主控制。

## 已完成项

✅ **基础架构部署**: Tinode 服务器（PostgreSQL 存储）已上线并稳定运行。
✅ **用户同步**: 后端已实现用户注册/登录时的自动同步及 Token 生成机制。
✅ **Mobile 端集成**: 移动端已完整集成 TinodeService，支持核心聊天功能。
✅ **文本消息**: 全端支持标准文本消息收发。
✅ **图片消息**: 支持图片的上传、发送及渲染（Mobile & Admin）。
✅ **语音消息**: 已实现语音消息的录制与播放适配。
✅ **在线状态**: 用户在线/离线状态实时更新。
✅ **消息已读**: 消息已读回执功能。

## 回滚方案

系统保留了腾讯云 IM 的代码作为紧急后备方案。如需回滚：

1. **后端切换**:
   - 修改环境变量，将主 IM 标识切回 `TENCENT_IM`。
   - 重启 API 服务以重新激活 `im_handler.go` 逻辑。
2. **前端切换**:
   - 在 Mobile/Admin 配置文件中切换 IM 服务提供者。
   - 重新构建并发布客户端。
3. **数据校验**:
   - 验证用户 UserSig 生成是否正常。
   - 检查历史消息访问（注：Tinode 与腾讯云 IM 消息数据不互通）。
4. **验证功能**:
   - 执行 P0 级测试脚本，确保基础收发功能恢复。

## 风险控制

- **数据隔离**: 迁移期间产生的 Tinode 消息不会同步到腾讯云 IM，回滚后可能会有数据断层。
- **配置同步**: 确保所有环境（开发、测试、生产）的 `TINODE_UID_ENCRYPTION_KEY` 等密钥一致，避免 Token 生成错误。
- **负载观测**: 在全量切换前，通过监控系统密切关注 Tinode 容器的 CPU/内存及 WebSocket 连接数。
