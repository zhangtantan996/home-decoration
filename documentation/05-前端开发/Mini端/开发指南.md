## 开发环境搭建

### 1. 启动后端 API
确保后端服务已启动且配置了微信小程序的 `AppID` 和 `Secret`。

### 2. 安装依赖
```bash
cd mini
npm install
```

### 3. 编译运行
```bash
# 微信小程序模式（持续监听）
npm run dev:weapp
```

### 4. 导入微信开发者工具
1. 打开微信开发者工具，选择“导入项目”。
2. 目录选择项目根目录下的 `mini/`（工具会自动识别 `project.config.json`）。
3. 确认 AppID 设置正确。

## 关键业务流程

### 登录链路 (MVP-Auth)
小程序登录采用“静默登录 + 手机号授权”模式：
1. **静默登录**：调用 `Taro.login()` 获取 `code`，发送至 `POST /auth/wechat/mini/login`。
2. **账号判断**：
   - 若 openid 已绑定用户，直接返回 JWT。
   - 若未绑定，返回 `bindToken` 提示需绑定手机号。
3. **手机号绑定**：调用 `getPhoneNumber` 插件获取加密 code，发送至 `POST /auth/wechat/mini/bind-phone` 完成注册与绑定。

## 注意事项

- **API 基地址**：通过环境变量 `TARO_APP_API_BASE` 控制，默认为 `http://localhost:8080/api/v1`。
- **HTTPS 限制**：真机调试需在微信后台配置合法域名，或在开发者工具中开启“不校验合法域名”。
- **分包策略**：随着业务增加，建议将非 Tab 页面移入分包以控制主包体积。
