# Mobile 端原生模块

**最后更新**: 2026-01-25

本文档介绍 Mobile 端（React Native）中使用的原生模块及其配置，主要包括相机、图片选择、语音录制等功能。

## 原生模块清单

### 相机与扫描
- **库**: `react-native-camera-kit`
- **用途**: 用于二维码扫描、实时相机预览。
- **场景**: 扫码登录、扫码绑定设备。

### 图片选择与裁剪
- **库**: `react-native-image-picker`
- **用途**: 从相册选择图片/视频，或直接调用相机拍照。
- **库**: `react-native-image-crop-picker`
- **用途**: 提供更强大的图片选择能力，支持多选、压缩及裁剪。
- **场景**: 头像上传、施工日志图片上传。

### 语音录制
- **库**: `react-native-nitro-sound`
- **用途**: 用于录制音频、播放音频。
- **场景**: IM 聊天中的语音消息录制。

### 其他常用模块
- **文件选择**: `@react-native-documents/picker`
- **地理位置**: `@react-native-community/geolocation`
- **安全存储**: `react-native-keychain`
- **文件操作**: `react-native-blob-util`

## 权限配置

使用原生功能前，必须在各平台配置文件中声明相应权限。

### Android (AndroidManifest.xml)

文件路径：`mobile/android/app/src/main/AndroidManifest.xml`

```xml
<!-- 网络权限 -->
<uses-permission android:name="android.permission.INTERNET" />

<!-- 相册读取权限 -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<!-- 相册写入权限 (API 29及以下) -->
<uses-permission
    android:name="android.permission.WRITE_EXTERNAL_STORAGE"
    android:maxSdkVersion="29"
    tools:replace="android:maxSdkVersion" />

<!-- 定位权限 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- 语音录制权限 -->
<uses-permission android:name="android.permission.RECORD_AUDIO" />

<!-- 相机权限说明：
     由于 react-native-image-picker 的特性，通常不需要在 AndroidManifest 中显式声明 CAMERA 权限，
     否则在某些版本上可能会导致运行时请求逻辑异常。
-->
<uses-permission android:name="android.permission.CAMERA" tools:node="remove" />
```

### iOS (Info.plist)

文件路径：`mobile/ios/HomeDecorationApp/Info.plist`

```xml
<key>NSCameraUsageDescription</key>
<string>我们需要相机权限以扫描二维码或拍摄照片</string>

<key>NSMicrophoneUsageDescription</key>
<string>我们需要麦克风权限以录制语音消息</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>我们需要相册权限以选择图片或从图片扫描二维码</string>

<key>NSLocationWhenInUseUsageDescription</key>
<string>我们需要地理位置权限以提供周边服务</string>
```

## 平台差异

### 1. 权限请求方式
- **Android**: 
  - 对于 Android 6.0 (API 23) 以上版本，需要使用 `PermissionsAndroid` 模块在运行时动态请求权限。
  - 对于相册访问，Android 13+ 引入了更细粒度的权限（READ_MEDIA_IMAGES 等），需注意兼容。
- **iOS**: 
  - 权限请求通常在第一次调用相关 API 时由系统自动弹出。
  - 必须在 `Info.plist` 中配置 `UsageDescription`，否则应用会崩溃。

### 2. API 行为差异
- **相机**: `react-native-image-picker` 在 iOS 上调用相机后会返回临时文件路径；在 Android 上某些情况下可能需要处理 Activity 被销毁的状态。
- **文件路径**: 
  - Android 常用 `file:///` 开头的路径。
  - iOS 通常使用相对路径或经过处理的绝对路径。
- **音频格式**: 
  - Android 常用 AAC 或 AMR 编码。
  - iOS 常用 AAC (M4A) 编码。建议统一使用 AAC 以保证双端兼容。

## 使用示例

### 图片选择 (react-native-image-picker)

```typescript
import { launchImageLibrary, launchCamera } from 'react-native-image-picker';

const handleSelectImage = async () => {
  const result = await launchImageLibrary({
    mediaType: 'photo',
    quality: 1,
  });

  if (result.didCancel) {
    console.log('User cancelled image picker');
  } else if (result.errorMessage) {
    console.log('ImagePicker Error: ', result.errorMessage);
  } else {
    const source = result.assets[0].uri;
    console.log('Selected image uri:', source);
  }
};
```

### 语音录制 (react-native-nitro-sound)

```typescript
import { useSoundRecorder, AudioEncoderAndroidType, AudioSourceAndroidType } from 'react-native-nitro-sound';

const { startRecorder, stopRecorder } = useSoundRecorder({
  onRecord: (event) => {
    console.log('Recording duration:', event.currentPosition);
  },
});

const start = async () => {
  await startRecorder({
    AudioEncoderAndroid: AudioEncoderAndroidType.AAC,
    AudioSourceAndroid: AudioSourceAndroidType.MIC,
  });
};

const stop = async () => {
  const path = await stopRecorder();
  console.log('Audio recorded at:', path);
};
```

### 二维码扫描 (react-native-camera-kit)

```typescript
import { Camera, CameraType } from 'react-native-camera-kit';

const ScanView = () => (
  <Camera
    cameraType={CameraType.Back}
    scanBarcode={true}
    onReadCode={(event) => console.log(event.nativeEvent.codeStringValue)}
    frameColor="white"
    laserColor="red"
  />
);
```
