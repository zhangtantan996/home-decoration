## 导航结构

Mobile 端采用 **React Navigation v7**，核心结构为 `Stack Navigator` 嵌套 `Bottom Tab Navigator`。

### 1. 根导航 (AppNavigator)
负责处理登录状态（Auth Flow）和全局页面跳转。

```typescript
// src/navigation/AppNavigator.tsx

const Stack = createNativeStackNavigator<RootStackParamList>();

const AppNavigator = () => {
    const { isAuthenticated } = useAuthStore();

    return (
        <NavigationContainer>
            <Stack.Navigator screenOptions={{ headerShown: false }}>
                {isAuthenticated ? (
                    // 已登录：进入主应用
                    <>
                        <Stack.Screen name="Main" component={MainTabs} />
                        <Stack.Screen name="Booking" component={BookingScreen} />
                        <Stack.Screen name="ChatRoom" component={ChatRoomScreen} />
                        {/* 其他全局 Stack 页面 */}
                    </>
                ) : (
                    // 未登录：进入登录页
                    <Stack.Screen name="Login" component={LoginScreen} />
                )}
            </Stack.Navigator>
        </NavigationContainer>
    );
};
```

### 2. 底部导航 (MainTabs)
主界面采用底部 Tab 布局，包含 5 个核心模块。

- **Home**: 首页 (HomeScreen)
- **Inspiration**: 灵感 (InspirationScreen)
- **Progress**: 进度/工地 (MySiteScreen)
- **Message**: 消息 (MessageScreen)
- **Profile**: 我的 (ProfileScreen)

### 3. 类型定义 (TypeScript)
所有路由参数都应在 `RootStackParamList` 中严格定义。

```typescript
export type RootStackParamList = {
    Main: undefined;
    Login: undefined;
    Booking: {
        providerId: string;
        providerType: 'designer' | 'worker' | 'company';
    };
    // ...
};
```

## 数据流与状态管理

使用 **Zustand** 进行全局状态管理，结合 **Axios** 处理异步请求。

### Store 示例
```typescript
// src/store/authStore.ts
import { create } from 'zustand';

interface AuthState {
  user: User | null;
  isAuthenticated: boolean;
  login: (credentials: LoginCredentials) => Promise<void>;
  logout: () => void;
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  isAuthenticated: false,
  login: async (creds) => {
    const user = await api.login(creds);
    set({ user, isAuthenticated: true });
  },
  logout: () => set({ user: null, isAuthenticated: false }),
}));
```

## 平台差异处理

由于采用纯 React Native 架构 (移除 Web 支持)，主要关注 iOS 与 Android 的差异。

### 样式适配
使用 `Platform.OS` 或 `Platform.select` 处理样式差异。

```typescript
import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  header: {
    height: Platform.OS === 'ios' ? 88 : 56,
    paddingTop: Platform.OS === 'ios' ? 44 : 0,
    ...Platform.select({
      ios: { shadowOpacity: 0.1 },
      android: { elevation: 4 },
    }),
  },
});
```

### 原生模块配置
- **Android**: 依赖 `android/app/build.gradle` 和 `AndroidManifest.xml`。
- **iOS**: 依赖 `ios/Podfile`，需运行 `cd ios && pod install` 安装依赖。

## 开发注意事项

1.  **React 19 特性**: Mobile 端使用了 React 19，可以使用新的 Hooks (如 `use`)，但需注意第三方库兼容性。
2.  **新架构 (New Architecture)**: RN 0.76+ 默认开启新架构，若遇到原生模块兼容问题，可能需要回退或寻找替代库。
3.  **安全区域**: 所有页面应使用 `react-native-safe-area-context` 进行适配，避免刘海屏遮挡。
4.  **图片资源**: 建议使用 `lucide-react-native` 矢量图标，减少 PNG/JPG 资源体积。
