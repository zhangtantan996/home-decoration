# Admin 端组件规范

**最后更新**: 2026-01-25

## 1. Ant Design 使用规范

本项目 Admin 端基于 [Ant Design 5.x](https://ant.design/docs/react/introduce-cn) 和 [Ant Design Pro Components](https://procomponents.ant.design/) 构建。

### 1.1 核心原则
- **优先使用 ProComponents**: 对于表格（CRUD）、表单、布局等复杂场景，优先使用 `@ant-design/pro-components`，以减少样板代码。
- **保持一致性**: 遵循 Ant Design 的默认设计语言，除非有明确的 UI 定制需求，否则不要随意覆盖全局样式。
- **按需引入**: 虽然 AntD 5 默认支持 Tree Shaking，但在引用图标时仍建议按需引入。

### 1.2 常用组件推荐

| 场景 | 推荐组件 | 说明 |
| :--- | :--- | :--- |
| **布局** | `ProLayout` | 提供标准的后台管理布局（菜单、面包屑、页头） |
| **数据表格** | `ProTable` | 内置搜索、筛选、排序、分页，极大简化 CRUD 开发 |
| **表单** | `ProForm` | 包括 `ModalForm`, `DrawerForm` 等，简化表单状态管理 |
| **骨架屏** | `ProSkeleton` / `Skeleton` | 页面加载时的占位显示 |
| **反馈** | `App` 包裹组件 | 使用 `App.useApp()` 获取 `message`, `modal`, `notification` 实例，确保上下文正确 |

### 1.3 代码示例

#### ProTable 使用示例
```tsx
import { ProTable, type ProColumns } from '@ant-design/pro-components';
import { Button } from 'antd';
import { PlusOutlined } from '@ant-design/icons';

interface DataType {
  id: string;
  name: string;
  status: string;
}

const columns: ProColumns<DataType>[] = [
  {
    title: '名称',
    dataIndex: 'name',
    copyable: true,
  },
  {
    title: '状态',
    dataIndex: 'status',
    valueEnum: {
      open: { text: '开启', status: 'Success' },
      closed: { text: '关闭', status: 'Error' },
    },
  },
  {
    title: '操作',
    valueType: 'option',
    render: (_, record) => [
      <a key="edit">编辑</a>,
    ],
  },
];

export default () => (
  <ProTable<DataType>
    columns={columns}
    request={async (params) => {
      // API 请求逻辑
      return { data: [], success: true };
    }}
    rowKey="id"
    toolBarRender={() => [
      <Button key="button" icon={<PlusOutlined />} type="primary">
        新建
      </Button>,
    ]}
  />
);
```

## 2. 自定义组件

位于 `admin/src/components/` 目录下的通用组件。

### 2.1 组件列表

| 组件名 | 文件名 | 用途 |
| :--- | :--- | :--- |
| **字典选择器** | `DictSelect.tsx` | 基于系统字典数据的下拉选择器，用于表单中统一选择字典项 |
| **地区级联选择** | `RegionCascader.tsx` | 省市区三级联动选择器，封装了地区数据加载逻辑 |
| **权限包裹器** | `PermissionWrapper.tsx` | 基于 RBAC 权限控制组件显隐，用于按钮级权限控制 |
| **受保护路由** | `ProtectedRoute.tsx` | 路由守卫组件，处理登录状态校验和权限拦截 |
| **通知下拉框** | `NotificationDropdown.tsx` | 顶部导航栏的消息通知下拉列表 |
| **商家通知下拉** | `MerchantNotificationDropdown.tsx` | 专门用于商家端的通知组件 |
| **占位页面** | `PlaceholderPage.tsx` | 用于空状态、403、404 等占位显示 |

### 2.2 使用示例

#### DictSelect (字典选择)
```tsx
import DictSelect from '@/components/DictSelect';

// 在 Form 中使用
<Form.Item name="status" label="状态">
  <DictSelect dictType="sys_common_status" placeholder="请选择状态" />
</Form.Item>
```

#### PermissionWrapper (权限控制)
```tsx
import PermissionWrapper from '@/components/PermissionWrapper';

<PermissionWrapper permission="system:user:add">
  <Button type="primary">新增用户</Button>
</PermissionWrapper>
```

## 3. 样式规范

### 3.1 样式方案
- **CSS-in-JS**: Ant Design 5 使用 CSS-in-JS。对于组件级别的简单样式调整，推荐使用 `style` 属性或 `emotion`/`styled-components` (如果项目中已引入)。
- **CSS Modules**: 对于复杂页面的布局样式，推荐使用 CSS Modules (`*.module.css` 或 `*.module.scss`)，避免全局样式污染。

### 3.2 命名规范
- **文件命名**: 组件文件使用 **PascalCase** (如 `UserList.tsx`)，样式文件使用同名 CSS Module (如 `UserList.module.scss`)。
- **类名命名**: 使用 **camelCase** (如 `.container`, `.headerTitle`)。
