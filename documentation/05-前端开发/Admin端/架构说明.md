## 状态管理 (Zustand)

Admin 端使用 Zustand 进行全局状态管理，主要用于处理用户认证、权限信息以及全局字典缓存。

### 核心设计模式
1. **Store 定义**: 使用 `create` 创建 Hook。
2. **持久化**: 部分 Store 使用 `persist` 中间件进行 localStorage 持久化。
3. **选择器**: 在组件中只选择需要的状态，避免不必要的重渲染。

### 代码示例

```typescript
// src/stores/authStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface User {
  id: number;
  username: string;
  permissions: string[];
}

interface AuthState {
  token: string | null;
  user: User | null;
  setToken: (token: string) => void;
  setUser: (user: User) => void;
  logout: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      token: null,
      user: null,
      setToken: (token) => set({ token }),
      setUser: (user) => set({ user }),
      logout: () => set({ token: null, user: null }),
    }),
    {
      name: 'admin-auth-storage', // localStorage key
    }
  )
);
```

### 组件中使用

```typescript
import { useAuthStore } from '@/stores/authStore';

const UserProfile = () => {
  // 只订阅 user 变化
  const user = useAuthStore((state) => state.user);
  
  if (!user) return null;
  
  return <div>你好, {user.username}</div>;
};
```

## 开发规范

1. **版本锁定**: 严禁修改 `package.json` 中 React 的版本号。
2. **UI 风格**: 优先使用 Ant Design Pro Components (`ProTable`, `ProForm`) 以保持界面一致性并提高开发效率。
3. **API 请求**: 所有后端请求必须通过 `src/services/` 目录下的服务函数发起，禁止在组件内直接调用 `axios`。
4. **权限控制**: 页面级权限在路由中配置，按钮级权限使用 `PermissionWrapper` 组件包裹。
