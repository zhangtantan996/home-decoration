# 聊天模块 API

**最后更新**: 2026-01-25

## 概述

聊天模块基于 Tinode 提供即时通讯（IM）功能，支持业主、设计师、工长及装修公司之间的实时沟通。系统支持文本、图片、语音消息以及实时在线状态展示。

**注意**: 当前系统正处于从腾讯云 IM 迁移至 Tinode 的阶段，迁移进度已达 **85%**。目前 Tinode 作为主要 IM 系统，腾讯云 IM 作为备选方案。

---

## Tinode 集成说明

### 1. 认证机制
- **认证方式**: 复用业务系统的 JWT Token。
- **用户标识**: Tinode 用户 ID 采用 `usr{id}` 格式（例如：`usr123`）。
- **Token 获取**: 登录成功后，后端会返回一个专用于 Tinode 认证的 `tinodeToken`。

### 2. 通讯协议
- **实时连接**: 采用 WebSocket 协议（`ws://` 或 `wss://`）。
- **API 接口**: 后端通过 Gin 框架封装了部分管理类和上传类接口。

### 3. 消息格式
- 消息内容采用 Tinode 的 **Drafty** 格式。
- 文本消息示例: `{"txt": "您好"}`
- 图片消息示例: `{"txt": "[图片]", "ent": [{"tp": "IM", "data": {"mime": "image/jpeg", "val": "URL"}}]}`

## API 列表

### 1. 获取聊天 Token
获取用于连接 Tinode 服务器的身份认证令牌。

**接口**: `POST /api/v1/chat/token`
**认证**: 需要业务 JWT Token
**请求参数**: 无

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "tinodeToken": "eyJhbGciOiJIUzI1NiIs...",
        "expiresIn": 604800
    }
}
```

---

### 2. 发送消息
通过后端网关发送消息（主要用于系统通知或特定业务逻辑触发的消息）。普通聊天建议直接通过 WebSocket 连接发送。

**接口**: `POST /api/v1/chat/messages`
**认证**: 需要 JWT Token
**请求参数**:
```json
{
    "topic": "usr123",
    "content": {
        "txt": "这是通过 API 发送的消息"
    }
}
```

**响应数据**:
```json
{
    "code": 0,
    "message": "消息已发送",
    "data": {
        "seqId": 45
    }
}
```

---

### 3. 上传图片
上传聊天图片到文件服务器或云存储。

**接口**: `POST /api/v1/chat/upload/image`
**Content-Type**: `multipart/form-data`
**认证**: 需要 JWT Token
**请求参数**:
- `file`: 图片文件 (File)

**响应数据**:
```json
{
    "code": 0,
    "message": "上传成功",
    "data": {
        "url": "https://cdn.example.com/chat/img/2026/01/25/abc.jpg",
        "width": 1024,
        "height": 768,
        "size": 102400
    }
}
```

---

### 4. 上传语音
上传聊天语音片段。

**接口**: `POST /api/v1/chat/upload/voice`
**Content-Type**: `multipart/form-data`
**认证**: 需要 JWT Token
**请求参数**:
- `file`: 语音文件 (File, 建议格式: .m4a 或 .amr)
- `duration`: 语音时长（秒）

**响应数据**:
```json
{
    "code": 0,
    "message": "上传成功",
    "data": {
        "url": "https://cdn.example.com/chat/voice/2026/01/25/def.m4a",
        "duration": 15,
        "size": 45000
    }
}
```

---

### 5. 获取在线状态
批量查询多个用户的在线状态。

**接口**: `GET /api/v1/chat/status`
**认证**: 需要 JWT Token
**Query 参数**:
- `userIds`: 用户 ID 列表，逗号分隔 (例如: `1,2,3`)

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": [
        {
            "userId": 1,
            "online": true,
            "lastSeen": "2026-01-25T10:30:00Z"
        },
        {
            "userId": 2,
            "online": false,
            "lastSeen": "2026-01-24T18:20:00Z"
        }
    ]
}
```

---

### 6. 获取 Tinode 用户 ID
获取指定业务用户对应的 Tinode 用户标识（用于客户端初始化或管理操作）。

**接口**: `GET /api/v1/tinode/userid/:userId`
**认证**: 需要 JWT Token
**路径参数**:
- `userId`: 业务系统用户 ID (uint64)

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "tinodeUserId": "usr123"
    }
}
```

**说明**:
- Tinode 用户 ID 格式为 `usr{id}`（例如：业务用户 ID 为 123，则 Tinode 用户 ID 为 `usr123`）
- 该接口会自动同步用户信息到 Tinode 数据库（幂等操作）
- 如果用户不存在，返回 404 错误

---

### 7. 清空聊天记录
删除指定 Topic 中的所有消息（管理员功能）。

**接口**: `DELETE /api/v1/tinode/topic/:topic/messages`
**认证**: 需要 JWT Token + 管理员权限
**路径参数**:
- `topic`: Tinode Topic 名称

**响应数据**:
```json
{
    "code": 0,
    "message": "聊天记录已清空",
    "data": null
}
```

**Topic 格式说明**:
- **私聊**: `usr{id1}_usr{id2}` 格式，较小的 ID 在前（例如：`usr123_usr456`）
- **群聊**: `grp{id}` 格式（例如：`grp789`）

**权限要求**:
- 用户必须是该 Topic 的成员
- 用户必须拥有该 Topic 的管理员权限（`admin` 或 `owner` 角色）
- 非管理员调用将返回 403 错误

**错误码**:
- `400`: Topic 格式无效
- `403`: 权限不足（需要管理员权限）
- `404`: Topic 不存在或用户不是成员
- `503`: Tinode 服务不可用

---

### 8. 刷新 Tinode Token
刷新用于 Tinode 认证的 JWT Token（当 Token 即将过期时调用）。

**接口**: `POST /api/v1/tinode/refresh-token`
**认证**: 需要业务 JWT Token
**请求参数**: 无

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "tinodeToken": "eyJhbGciOiJIUzI1NiIs...",
        "tinodeError": ""
    }
}
```

**说明**:
- Token 有效期为 7 天（604800 秒）
- 建议在 Token 过期前 1 天刷新
- 如果刷新失败，`tinodeError` 字段会包含错误信息，`tinodeToken` 为空字符串
- 刷新成功后，客户端应使用新 Token 重新连接 Tinode 服务器

## Tinode 与腾讯云 IM 的关系

**当前架构**:
- **主要方案**: Tinode（自建开源 IM 系统）
- **备用方案**: 腾讯云 IM（维护至 2026-07-24）

**迁移进度**: 85% 完成

**选择 Tinode 的原因**:
1. **开源可控**: 完全掌握源码，可根据业务需求定制
2. **成本优化**: 无需支付第三方 IM 服务费用
3. **数据安全**: 聊天数据存储在自有数据库中
4. **功能扩展**: 可灵活集成业务逻辑（如装修进度通知、支付提醒等）

**腾讯云 IM 维护策略**:
- 代码保留但不再积极维护（见 `server/internal/handler/im_handler.go`）
- 不添加新功能，仅修复关键安全问题
- 计划于 2026-07-24 进行最终评审，决定是否完全移除

## 使用示例

### 移动端集成流程 (React Native)

1. **登录并获取 Token**:
   登录业务系统，从响应中获取 `tinodeToken`。
2. **初始化 SDK**:
   ```typescript
   import { Tinode } from 'tinode-sdk';
   const tinode = new Tinode({...});
   await tinode.connect('ws://localhost:6061/v0/channels');
   await tinode.loginToken(tinodeToken);
   ```
3. **订阅会话**:
   ```typescript
   const topic = tinode.getTopic('usr123');
   await topic.subscribe();
   ```
4. **发送文本**:
   ```typescript
   await topic.publishMessage("Hello!");
   ```
5. **发送图片**:
   先调用 `/api/v1/chat/upload/image` 获取 URL，然后通过 `publishMessage` 发送 Drafty 格式内容。
6. **Token 刷新**:
   ```typescript
   // 在 Token 即将过期时调用
   const response = await fetch('/api/v1/tinode/refresh-token', {
     method: 'POST',
     headers: { 'Authorization': `Bearer ${businessToken}` }
   });
   const { tinodeToken } = await response.json();
   await tinode.loginToken(tinodeToken);
   ```
