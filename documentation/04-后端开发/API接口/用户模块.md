# 用户模块 API

**最后更新**: 2026-01-25

## 概述
用户模块（业主端）是平台的核心模块之一，主要服务于装修业主。该模块提供个人资料管理、服务商预约、装修项目进度追踪、托管资金管理以及售后服务申请等功能。

> **认证说明**: 本模块下所有接口均需在 HTTP 请求头中携带 JWT Token：
> `Authorization: Bearer {token}`

## API 列表

### 1. 个人资料管理

#### 1.1 获取个人资料
**接口**: `GET /api/v1/user/profile`

**描述**: 获取当前登录用户的详细个人信息。

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "phone": "13800138000",
    "nickname": "张三",
    "avatar": "https://example.com/avatar.jpg",
    "userType": 1,
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

#### 1.2 更新个人资料
**接口**: `PUT /api/v1/user/profile`

**描述**: 修改用户的昵称、头像等基本信息。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| nickname | string | 否 | 新昵称 |
| avatar | string | 否 | 新头像 URL |

**响应示例**:
```json
{
  "code": 0,
  "message": "更新成功",
  "data": {
    "id": 1,
    "nickname": "新昵称",
    "avatar": "https://example.com/new_avatar.jpg"
  }
}
```

### 2. 预约管理

#### 2.1 发起预约
**接口**: `POST /api/v1/bookings`

**描述**: 业主向选定的服务商（设计师、装修公司或工长）发起预约。

**请求参数**:
```json
{
  "providerId": 1,
  "providerType": "designer",
  "address": "上海市浦东新区张江高科技园区",
  "area": 120,
  "renovationType": "全屋装修",
  "budgetRange": "10-20万",
  "preferredDate": "2025-02-10",
  "phone": "13800138000",
  "notes": "希望了解北欧风格的设计方案",
  "houseLayout": "3室2厅2卫"
}
```

#### 2.2 获取预约列表
**接口**: `GET /api/v1/bookings`

**描述**: 分页获取当前用户的所有预约记录。

**Query 参数**:
- `page`: 页码（默认 1）
- `pageSize`: 每页数量（默认 20）
- `status`: 状态筛选

#### 2.3 获取预约详情
**接口**: `GET /api/v1/bookings/:id`

**描述**: 获取特定预约的详细信息，包括服务商反馈和状态。

#### 2.4 支付意向金
**接口**: `POST /api/v1/bookings/:id/pay-intent`

**描述**: 为特定预约支付意向金。支付成功后，预约将进入正式洽谈阶段。

### 3. 项目管理

#### 3.1 我的项目列表
**接口**: `GET /api/v1/projects`

**描述**: 获取业主名下的所有装修项目。

**响应示例**:
```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "id": 1,
        "name": "浦东新区120平米装修项目",
        "status": "in_progress",
        "currentPhase": "水电改造",
        "progress": 35
      }
    ],
    "total": 1
  }
}
```

#### 3.2 项目详情与进度
**接口**: `GET /api/v1/projects/:id`

**描述**: 获取项目的实时进度、阶段划分及施工方联系方式。

#### 3.3 查看施工日志
**接口**: `GET /api/v1/projects/:id/logs`

**描述**: 查看工长或设计师上传的每日施工日志及现场照片。

#### 3.4 确认验收节点
**接口**: `POST /api/v1/projects/:id/accept`

**描述**: 业主对已完工的阶段性节点（如水电、木工）进行在线验收确认。

**请求参数**:
```json
{
  "milestoneId": 1,
  "rating": 5,
  "comment": "验收通过，施工符合标准"
}
```

---

### 4. 资金托管

#### 4.1 查询托管详情
**接口**: `GET /api/v1/projects/:id/escrow`

**描述**: 查询该项目的托管资金概览。

#### 4.2 充值到托管账户
**接口**: `POST /api/v1/projects/:id/deposit`

**描述**: 业主按合同约定将资金阶段性存入平台托管。

---

### 5. 售后服务

#### 5.1 提交售后申请
**接口**: `POST /api/v1/after-sales`

**描述**: 针对装修质量问题或投诉发起售后申请。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| bookingId | int | 是 | 关联订单 ID |
| type | string | 是 | `refund`（退款）、`complaint`（投诉）、`repair`（维修） |
| reason | string | 是 | 申请原因 |
| description | string | 是 | 详细说明 |
| images | array | 否 | 照片证明 |

#### 5.2 售后详情查询
**接口**: `GET /api/v1/after-sales/:id`

**描述**: 追踪售后申请的处理进度和平台回复。

## 错误码参考

| 错误码 | 说明 |
| :--- | :--- |
| 401 | 认证失败，请重新登录 |
| 403 | 无权访问该项目/预约 |
| 404 | 资源不存在 |
| 2001 | 余额不足，请先充值 |
| 2002 | 项目状态不支持该操作 |

## 使用示例 (Curl)

### 获取个人档案
```bash
curl -X GET http://localhost:8080/api/v1/user/profile \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### 更新昵称
```bash
curl -X PUT http://localhost:8080/api/v1/user/profile \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"nickname": "阿星"}'
```
