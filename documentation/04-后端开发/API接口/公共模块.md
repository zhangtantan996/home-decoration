# 公共模块 API

**最后更新**: 2026-01-25

## 概述
公共模块提供系统级的基础数据服务，包括字典管理、行政区划、灵感图库、材料商店收藏和文件上传功能。这些接口大部分无需认证即可访问，为前端提供下拉选项、地区选择、内容浏览等基础能力。

---

## 一、字典管理 API

字典模块提供系统中各类下拉选项的数据源，如装修类型、户型、风格等。

### 常用字典分类说明

| 分类代码 | 分类名称 | 说明 | 示例值 |
| :--- | :--- | :--- | :--- |
| renovation_type | 装修类型 | 新房装修、二手房翻新等 | new_house, old_house, partial |
| house_layout | 户型 | 房屋户型结构 | one_bedroom, two_bedroom, three_bedroom |
| style | 装修风格 | 现代简约、中式、欧式等 | modern, chinese, european, nordic |
| budget_range | 预算范围 | 装修预算区间 | 5w_below, 5w_10w, 10w_20w, 20w_above |
| area_range | 面积范围 | 房屋面积区间 | 50_below, 50_80, 80_120, 120_above |
| service_type | 服务类型 | 设计、施工、监理等 | design, construction, supervision |

### 1. 获取字典选项
**接口**: `GET /api/v1/dictionaries/:category`

**描述**: 根据分类代码获取该分类下的所有字典选项（仅返回已启用的选项）。

**请求参数**:
- `category` (路径参数): 字典分类代码，如 `renovation_type`、`house_layout`、`style` 等

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "value": "modern",
      "label": "现代简约"
    },
    {
      "value": "chinese",
      "label": "中式风格"
    },
    {
      "value": "european",
      "label": "欧式风格"
    }
  ]
}
```

## 二、行政区划 API

行政区划模块提供省-市-区三级联动数据，支持懒加载和级联查询。

### 区域层级说明
- **Level 1**: 省级（省、直辖市、自治区）
- **Level 2**: 市级（地级市、直辖市的区）
- **Level 3**: 区级（区、县、县级市）

### 3. 获取所有省份
**接口**: `GET /api/v1/regions/provinces`

**描述**: 获取所有已启用的省级行政区划。

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "code": "110000",
      "name": "北京市",
      "level": 1,
      "parentCode": "",
      "sortOrder": 1,
      "enabled": true
    },
    {
      "id": 2,
      "code": "310000",
      "name": "上海市",
      "level": 1,
      "parentCode": "",
      "sortOrder": 2,
      "enabled": true
    }
  ]
}
```

---

### 4. 根据省份获取城市列表
**接口**: `GET /api/v1/regions/provinces/:provinceCode/cities`

**描述**: 根据省份代码获取该省下的所有城市（Level 2）。

**请求参数**:
- `provinceCode` (路径参数): 省份代码，如 `110000`（北京）、`310000`（上海）

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 10,
      "code": "110100",
      "name": "北京市",
      "level": 2,
      "parentCode": "110000",
      "sortOrder": 1,
      "enabled": true
    }
  ]
}
```

---

### 5. 根据城市获取区/县列表
**接口**: `GET /api/v1/regions/cities/:cityCode/districts`

**描述**: 根据城市代码获取该市下的所有区/县（Level 3）。

**请求参数**:
- `cityCode` (路径参数): 城市代码，如 `110100`（北京市）

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 100,
      "code": "110101",
      "name": "东城区",
      "level": 3,
      "parentCode": "110100",
      "sortOrder": 1,
      "enabled": true
    },
    {
      "id": 101,
      "code": "110102",
      "name": "西城区",
      "level": 3,
      "parentCode": "110100",
      "sortOrder": 2,
      "enabled": true
    }
  ]
}
```

### 6. 根据父级代码获取子区域（懒加载）
**接口**: `GET /api/v1/regions/children/:parentCode`

**描述**: 通用的懒加载接口，根据父级代码获取下一级区域。适用于级联选择器的懒加载场景。

**请求参数**:
- `parentCode` (路径参数): 父级区域代码

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 10,
      "code": "110100",
      "name": "北京市",
      "level": 2,
      "parentCode": "110000",
      "sortOrder": 1,
      "enabled": true
    }
  ]
}
```

**使用场景**:
- 前端级联选择器懒加载
- 动态加载下一级区域数据
- 减少初始加载数据量

## 三、灵感图库 API

灵感图库提供装修案例的浏览、点赞、收藏和评论功能。部分接口支持未登录访问，社交功能需要认证。

### 7. 获取灵感图库列表
**接口**: `GET /api/v1/inspiration`

**描述**: 获取灵感案例列表，支持筛选和分页。未登录用户可浏览，登录用户可查看个人点赞/收藏状态。

**认证**: 可选（使用 OptionalJWT 中间件）

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10 |
| style | string | 否 | 装修风格筛选 |
| houseLayout | string | 否 | 户型筛选 |
| areaRange | string | 否 | 面积范围筛选 |
| sortBy | string | 否 | 排序方式：latest（最新）、hot（最热）、likes（点赞数） |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "现代简约三居室",
        "coverImage": "https://example.com/image.jpg",
        "style": "modern",
        "houseLayout": "three_bedroom",
        "area": 120,
        "likeCount": 256,
        "favoriteCount": 89,
        "commentCount": 34,
        "isLiked": true,
        "isFavorited": false,
        "designer": {
          "id": 10,
          "name": "张设计师",
          "avatar": "https://example.com/avatar.jpg"
        }
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

**说明**:
- 未登录用户：`isLiked` 和 `isFavorited` 字段不返回或为 `null`
- 登录用户：返回当前用户对该案例的点赞/收藏状态

## 五、文件上传 API

文件上传接口支持图片、文档、音视频等多种格式，用于聊天、头像、案例图片等场景。

### 16. 通用文件上传
**接口**: `POST /api/v1/upload`

**描述**: 上传文件到服务器，支持多种文件格式。

**认证**: 必需（JWT Token）

**请求参数**:
- `file` (表单字段): 要上传的文件

**支持的文件格式**:

| 类型 | 扩展名 | 大小限制 |
| :--- | :--- | :--- |
| 图片 | .jpg, .jpeg, .png, .gif, .webp | 50MB |
| 文档 | .pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .txt | 50MB |
| 压缩包 | .zip, .rar | 50MB |
| 视频 | .mp4, .mov, .avi | 50MB |
| 音频 | .m4a, .aac, .mp3, .wav, .ogg | 50MB |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "https://example.com/uploads/chat/202601/chat_100_1737792000000000000.jpg",
    "path": "/uploads/chat/202601/chat_100_1737792000000000000.jpg",
    "filename": "avatar.jpg",
    "size": 102400,
    "type": ".jpg"
  }
}
```

**字段说明**:
- `url`: 完整的访问 URL（包含域名）
- `path`: 相对路径（用于存储到数据库）
- `filename`: 原始文件名
- `size`: 文件大小（字节）
- `type`: 文件扩展名

**错误码**:
| 错误码 | 说明 |
| :--- | :--- |
| 400 | 文件格式不支持或文件大小超限 |
| 401 | 未登录或 Token 无效 |
| 500 | 服务器保存文件失败 |

**使用示例 (Curl)**:
```bash
curl -X POST http://localhost:8080/api/v1/upload \
  -H "Authorization: Bearer <your_token>" \
  -F "file=@/path/to/your/image.jpg"
```

**使用示例 (JavaScript)**:
```javascript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

fetch('http://localhost:8080/api/v1/upload', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
})
.then(res => res.json())
.then(data => console.log(data));
```

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法或缺失必填字段 |
| 401 | 未认证 | 需要登录的接口未携带有效 Token |
| 404 | 资源不存在 | 请求的资源（案例、商店等）不存在 |
| 500 | 服务器错误 | 服务器内部错误，请联系管理员 |

## 使用说明

### 字典使用流程
1. 调用 `GET /api/v1/dictionaries/categories` 获取所有分类
2. 根据需要的分类代码调用 `GET /api/v1/dictionaries/:category` 获取选项
3. 前端渲染为下拉框或选择器

### 区域三级联动流程
1. 页面加载时调用 `GET /api/v1/regions/provinces` 获取省份列表
2. 用户选择省份后，调用 `GET /api/v1/regions/provinces/:provinceCode/cities` 获取城市
3. 用户选择城市后，调用 `GET /api/v1/regions/cities/:cityCode/districts` 获取区县

### 灵感图库社交功能流程
1. 未登录用户可浏览案例列表和评论
2. 登录后可进行点赞、收藏、评论操作
3. 点赞和收藏支持切换（再次点击取消）
4. 评论支持回复（通过 `parentId` 参数）

---

## 注意事项

1. **字典缓存**: 字典数据变化不频繁，建议前端缓存到 LocalStorage，减少请求次数
2. **区域数据**: 区域数据量较大，建议使用懒加载方式，按需加载下一级数据
3. **文件上传**: 
   - 上传前需先获取有效的 JWT Token
   - 注意文件大小限制（50MB）
   - 建议前端进行文件格式和大小预校验
4. **灵感图库**: 
   - 列表接口支持未登录访问，但不返回个人状态
   - 社交功能（点赞、收藏、评论）必须登录
5. **性能优化**: 
   - 灵感列表建议使用虚拟滚动或分页加载
   - 图片建议使用 CDN 加速和懒加载
