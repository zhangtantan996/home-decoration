# 商家模块 API

**最后更新**: 2026-01-25

## 概述
商家模块（Merchant）主要面向平台的服务商，包括设计师、装修公司和工长。该模块提供了从商家入驻申请、认证登录、个人信息维护、作品案例管理到预约处理、方案提交、订单跟踪及财务结算（收益与提现）的全流程功能。

## API 认证
所有商家端接口（除入驻申请和登录外）均需要在请求头中携带商家 JWT Token：
`Authorization: Bearer <your_merchant_token>`

## API 列表

### 1. 商家入驻申请

#### 1.1 提交入驻申请
**接口**: `POST /api/v1/merchant/apply`
**描述**: 服务商提交入驻申请资料，包括基本信息、资质证明及初步作品。
**认证**: 无需认证

**请求参数 (JSON)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| phone | String | 是 | 手机号 |
| code | String | 是 | 验证码 |
| applicantType | String | 是 | 申请类型：`personal` (个人), `studio` (工作室), `company` (公司) |
| realName | String | 是 | 真实姓名 |
| companyName | String | 否 | 公司/工作室名称 |
| licenseImage | String | 否 | 营业执照照片 URL |
| portfolioCases | Array | 否 | 初始作品案例列表 |

**响应示例**:
```json
{
    "code": 0,
    "message": "申请提交成功，请等待审核",
    "data": {
        "applicationId": 100,
        "status": "pending"
    }
}
```

#### 1.2 查询申请状态
**接口**: `GET /api/v1/merchant/apply/:phone/status`
**描述**: 通过手机号查询入驻申请的审核进度。

#### 2.3 更新商家信息

**描述**: 更新商家个人资料，包括公司名称、服务区域、专长等信息。
**认证**: 需要商家 Token

**请求参数 (JSON)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| name | String | 否 | 显示名称（个人设计师昵称） |

**响应示例**:

```json
    "message": "success",
```

#### 2.4 上传商家头像
**接口**: `POST /api/v1/merchant/avatar`
**描述**: 上传商家头像图片。
**认证**: 需要商家 Token
**Content-Type**: `multipart/form-data`

**请求参数 (Form)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| file | File | 是 | 头像文件（jpg/png/gif，最大2MB） |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "url": "http://cdn.com/uploads/avatars/avatar_123_1234567890.jpg",
        "path": "/uploads/avatars/avatar_123_1234567890.jpg"
    }
}
```

#### 2.5 上传通用图片
**接口**: `POST /api/v1/merchant/upload`
**描述**: 上传案例图片、封面图或附件文档。
**认证**: 需要商家 Token
**Content-Type**: `multipart/form-data`

**请求参数 (Form)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| file | File | 是 | 文件（图片/文档/压缩包，最大20MB） |

**支持格式**: jpg, jpeg, png, webp, pdf, doc, docx, xls, xlsx, ppt, pptx, txt, zip, rar

**响应示例**:
```json
{

    "message": "success",
    "data": {
        "url": "http://cdn.com/uploads/cases/case_1_1234567890.jpg",
        "path": "/uploads/cases/case_1_1234567890.jpg"
```

### 3. 作品案例管理

#### 3.1 获取作品集列表
**接口**: `GET /api/v1/merchant/cases`
**描述**: 分页获取商家已发布的所有作品案例。

#### 3.2 获取作品详情
**接口**: `GET /api/v1/merchant/cases/:id`
**描述**: 获取特定案例的详细信息，包括待审核的修改内容。
**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 案例 ID |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "id": 1,
        "auditId": 10,
        "title": "现代简约风格案例",
        "coverImage": "http://cdn.com/cover.jpg",
        "style": "现代简约",
        "layout": "三室两厅",
        "area": "120㎡",
        "price": 150000,
        "quoteTotalCent": 15000000,
        "quoteCurrency": "CNY",
        "quoteItems": "[{\"name\":\"设计费\",\"amountCent\":3000000}]",
        "year": "2024",
        "description": "案例描述...",
        "images": ["http://cdn.com/img1.jpg", "http://cdn.com/img2.jpg"],
        "sortOrder": 1,
        "createdAt": "2024-01-01T00:00:00Z",
        "status": 0,
        "actionType": "update"
    }
}
```

**字段说明**:
- `status`: 0=待审核, 1=已发布, 2=已拒绝
- `actionType`: create=新增, update=修改, delete=删除, 空=无审核
- `auditId`: 审核记录 ID（有待审核时返回）

#### 3.3 创建作品案例
**接口**: `POST /api/v1/merchant/cases`
**描述**: 发布新的设计或施工案例。
**请求示例**:
```json
{
    "title": "现代简约风格案例",
    "coverImage": "https://example.com/cover.jpg",
    "style": "现代简约",
    "area": "120㎡",
    "year": "2024",
    "description": "案例描述...",
    "images": ["https://img1.jpg", "https://img2.jpg"]
}
```

#### 3.4 更新作品案例
**接口**: `PUT /api/v1/merchant/cases/:id`
**描述**: 修改已发布的案例信息。

#### 3.5 删除作品案例
**接口**: `DELETE /api/v1/merchant/cases/:id`
**描述**: 移除不再展示的案例。

#### 3.6 作品排序
**接口**: `PUT /api/v1/merchant/cases/reorder`
**描述**: 调整案例在个人主页的展示顺序。

#### 3.7 取消审核申请
**接口**: `DELETE /api/v1/merchant/cases/audit/:auditId`
**描述**: 取消待审核或已拒绝的案例审核申请。
**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| auditId | Integer | 是 | 审核记录 ID |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "message": "已取消"
    }
}
```

**业务规则**:
- 只能取消状态为待审核(0)或已拒绝(2)的审核记录
- 取消后审核记录将被删除
- 如果是修改类型的审核，取消后将恢复显示原始内容

**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 方案 ID |

**请求参数 (JSON)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| summary | String | 否 | 方案概述 |
| designFee | Float | 否 | 设计费（元） |
| constructionFee | Float | 否 | 施工费（元） |
| materialFee | Float | 否 | 材料费（元） |
| estimatedDays | Integer | 否 | 预计工期（天） |


**业务规则**:
- 只有待确认(1)或已拒绝(3)状态的方案才能修改
- 修改后方案状态自动改为待确认(1)
- 不改变方案版本号

- 取消后方案记录将被删除

##### 4.2.4 重新发起方案
**接口**: `POST /api/v1/merchant/proposals/:id/reopen`
**描述**: 将已取消的方案重新发起，状态改为待确认。
**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 方案 ID |

**业务规则**:
- 只能重新发起已取消(4)状态的方案
- 重新发起后状态改为待确认(1)

##### 4.2.5 重新提交方案（新版本）
**接口**: `POST /api/v1/merchant/proposals/resubmit`
**描述**: 基于已拒绝的方案创建新版本，版本号自动递增。
**认证**: 需要商家 Token

**请求参数 (JSON)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| originalProposalId | Integer | 是 | 原方案 ID |
| summary | String | 是 | 方案概述 |
| designFee | Float | 是 | 设计费（元） |
| constructionFee | Float | 是 | 施工费（元） |
| materialFee | Float | 是 | 材料费（元） |
| estimatedDays | Integer | 是 | 预计工期（天） |
| attachments | String | 否 | 附件 URL |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "proposal": {
            "id": 2,
            "bookingId": 10,
            "designerId": 5,
            "summary": "现代简约设计方案（修订版）",
            "designFee": 45000,
            "constructionFee": 180000,
            "materialFee": 140000,
            "estimatedDays": 85,
            "status": 1,
            "version": 2,
            "createdAt": "2024-01-05T00:00:00Z"
        },
        "message": "方案 v2 已提交，等待用户确认"
    }
}
```

**业务规则**:
- 只能基于已拒绝(3)状态的方案创建新版本
- 新方案版本号 = 原版本号 + 1
- 新方案状态为待确认(1)
- 原方案保持已拒绝状态不变
- 新方案关联相同的预约(bookingId)

##### 4.2.6 获取拒绝信息
**接口**: `GET /api/v1/merchant/proposals/:id/rejection-info`
**描述**: 查看方案被拒绝的原因和历史记录。
**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 方案 ID |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "proposalId": 1,
        "status": 3,
        "version": 1,
        "rejectedAt": "2024-01-03T10:30:00Z",
        "rejectionReason": "预算超出预期，希望降低材料费用",
        "rejectionCount": 1,
        "canResubmit": true,
        "history": [
            {
                "version": 1,
                "submittedAt": "2024-01-01T00:00:00Z",
                "rejectedAt": "2024-01-03T10:30:00Z",

```

**字段说明**:

- `history`: 该预约下所有方案的拒绝历史

**业务规则**:
- 只能查看属于当前商家的方案
- 累计拒绝 3 次后进入争议处理流程

### 6. IM 集成

商家端支持两种 IM 系统集成：**Tinode**（主要）和**腾讯云 IM**（备用）。

#### 6.1 获取腾讯云 IM 签名
**接口**: `GET /api/v1/merchant/im/usersig`
**描述**: 获取腾讯云 IM 登录所需的 UserSig 签名（备用方案）。
**认证**: 需要商家 Token
**状态**: 备用方案，不在生产环境使用

**响应示例**:
```json
{
    "code": 0,
    "data": {
        "sdkAppId": 1400123456,
        "userId": "123",
        "userSig": "eJwtzM0KgCAQBOB3mXMR3X..."
    }
}
```

**字段说明**:
- `sdkAppId`: 腾讯云 IM 应用 ID
- `userId`: 商家关联的用户 ID（全局唯一）
- `userSig`: 签名，有效期 7 天

**业务规则**:
- 自动将商家信息同步到腾讯云 IM
- 使用商家关联的 User ID 作为 IM 账号
- 昵称优先使用 User.Nickname，其次使用 Provider.CompanyName
- 头像使用完整 URL

#### 6.2 获取 Tinode 用户 ID
**接口**: `GET /api/v1/merchant/tinode/userid/:userId`
**描述**: 获取指定用户的 Tinode 用户 ID，用于发起聊天。
**认证**: 需要商家 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| userId | Integer | 是 | 目标用户 ID |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "tinodeUserId": "usr123"
    }
}
```

**字段说明**:
- `tinodeUserId`: Tinode 用户 ID，格式为 `usr{id}`

**业务规则**:
- 自动同步目标用户到 Tinode 数据库
- 操作是幂等的，可重复调用
- 用于商家主动联系用户场景

**Tinode 与腾讯云 IM 的关系**:
- **主要系统**: Tinode（自托管开源方案，迁移进度 85%）
- **备用系统**: 腾讯云 IM（保留至 2026-07-24，仅安全修复）
- **用户 ID 映射**: 业务用户 ID → Tinode 用户 ID (`usr{id}`)
- **话题命名**: 私聊使用 `usr{id1}_usr{id2}`（小 ID 在前），群聊使用 `grp{id}`
