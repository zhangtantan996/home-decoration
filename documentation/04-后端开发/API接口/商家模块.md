# 商家模块 API

**最后更新**: 2026-01-25

## 概述
商家模块（Merchant）主要面向平台的服务商，包括设计师、装修公司和工长。该模块提供了从商家入驻申请、认证登录、个人信息维护、作品案例管理到预约处理、方案提交、订单跟踪及财务结算（收益与提现）的全流程功能。

## API 认证
所有商家端接口（除入驻申请和登录外）均需要在请求头中携带商家 JWT Token：
`Authorization: Bearer <your_merchant_token>`

---

## API 列表

### 1. 商家入驻申请

#### 1.1 提交入驻申请
**接口**: `POST /api/v1/merchant/apply`
**描述**: 服务商提交入驻申请资料，包括基本信息、资质证明及初步作品。
**认证**: 无需认证

**请求参数 (JSON)**:
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| phone | String | 是 | 手机号 |
| code | String | 是 | 验证码 |
| applicantType | String | 是 | 申请类型：`personal` (个人), `studio` (工作室), `company` (公司) |
| realName | String | 是 | 真实姓名 |
| companyName | String | 否 | 公司/工作室名称 |
| licenseImage | String | 否 | 营业执照照片 URL |
| portfolioCases | Array | 否 | 初始作品案例列表 |

**响应示例**:
```json
{
    "code": 0,
    "message": "申请提交成功，请等待审核",
    "data": {
        "applicationId": 100,
        "status": "pending"
    }
}
```

#### 1.2 查询申请状态
**接口**: `GET /api/v1/merchant/apply/:phone/status`
**描述**: 通过手机号查询入驻申请的审核进度。

---

### 2. 商家认证与资料

#### 2.1 商家登录
**接口**: `POST /api/v1/merchant/login`
**描述**: 商家通过手机号和验证码登录。

#### 2.2 获取商家信息
**接口**: `GET /api/v1/merchant/info`
**描述**: 获取当前登录商家的详细个人资料、评价统计及服务状态。
**认证**: 需要商家 Token

---

### 3. 作品案例管理

#### 3.1 获取作品集列表
**接口**: `GET /api/v1/merchant/cases`
**描述**: 分页获取商家已发布的所有作品案例。

#### 3.2 获取作品详情
**接口**: `GET /api/v1/merchant/cases/:id`
**描述**: 获取特定案例的详细信息。

#### 3.3 创建作品案例
**接口**: `POST /api/v1/merchant/cases`
**描述**: 发布新的设计或施工案例。
**请求示例**:
```json
{
    "title": "现代简约风格案例",
    "coverImage": "https://example.com/cover.jpg",
    "style": "现代简约",
    "area": "120㎡",
    "year": "2024",
    "description": "案例描述...",
    "images": ["https://img1.jpg", "https://img2.jpg"]
}
```

#### 3.4 更新作品案例
**接口**: `PUT /api/v1/merchant/cases/:id`
**描述**: 修改已发布的案例信息。

#### 3.5 删除作品案例
**接口**: `DELETE /api/v1/merchant/cases/:id`
**描述**: 移除不再展示的案例。

#### 3.6 作品排序
**接口**: `PUT /api/v1/merchant/cases/reorder`
**描述**: 调整案例在个人主页的展示顺序。

---

### 4. 业务协作管理

#### 4.1 预约管理
- **获取预约列表**: `GET /api/v1/merchant/bookings`
- **获取预约详情**: `GET /api/v1/merchant/bookings/:id`
**描述**: 查看业主发起的预约请求及其详细需求（如房屋面积、预算、偏好等）。

#### 4.2 方案管理 (Proposals)
- **提交设计/施工方案**: `POST /api/v1/merchant/proposals`
- **获取方案列表**: `GET /api/v1/merchant/proposals`
**描述**: 针对业主预约提交响应，包括报价明细（设计费、施工费、材料费）及附件。

#### 4.3 订单管理
- **获取订单列表**: `GET /api/v1/merchant/orders`
- **描述**: 管理已签约的项目订单，查看支付进度及项目状态。

---

### 5. 财务管理

#### 5.1 收益概览与明细
- **概览**: `GET /api/v1/merchant/income/summary`
- **流水**: `GET /api/v1/merchant/income/list`

#### 5.2 提现管理
- **申请提现**: `POST /api/v1/merchant/withdraw`
- **获取提现记录**: `GET /api/v1/merchant/withdraw/list`

#### 5.3 银行账户管理
- **获取账户列表**: `GET /api/v1/merchant/bank-accounts`
- **添加账户**: `POST /api/v1/merchant/bank-accounts`
- **删除账户**: `DELETE /api/v1/merchant/bank-accounts/:id`
- **设置默认账户**: `PUT /api/v1/merchant/bank-accounts/:id/default`

## 使用示例

### 1. 商家入驻申请示例 (cURL)
```bash
curl -X POST http://localhost:8080/api/v1/merchant/apply \
     -H "Content-Type: application/json" \
     -d '{
           "phone": "13800138000",
           "code": "123456",
           "applicantType": "studio",
           "realName": "张三",
           "companyName": "名匠设计工作室"
         }'
```

### 2. 发布案例示例 (cURL)
```bash
curl -X POST http://localhost:8080/api/v1/merchant/cases \
     -H "Authorization: Bearer <TOKEN>" \
     -H "Content-Type: application/json" \
     -d '{
           "title": "极简北欧风",
           "style": "北欧",
           "area": "90㎡",
           "coverImage": "http://cdn.com/cover.jpg"
         }'
```

### 3. 获取工作台统计数据
**接口**: `GET /api/v1/merchant/dashboard`
**响应结构**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "todayBookings": 3,
        "pendingProposals": 5,
        "activeProjects": 8,
        "totalRevenue": 1200000,
        "rating": 4.8
    }
}
```
