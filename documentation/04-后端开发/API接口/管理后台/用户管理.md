# 管理后台 - 用户管理 API

**最后更新**: 2026-01-25

## 概述
用户管理模块为管理员提供用户数据的查询、创建、更新和状态管理功能。管理员可以查看所有用户列表、用户详情，创建新用户，更新用户信息，以及管理用户状态（启用/禁用/删除）。

## 用户类型说明
| 类型值 | 说明 | 备注 |
| :--- | :--- | :--- |
| 1 | 业主 | 普通用户，发起装修需求 |
| 2 | 服务商 | 设计师、装修公司、工长 |
| 3 | 工人 | 施工工人 |
| 4 | 管理员 | 平台管理员 |

## 用户状态说明
| 状态值 | 说明 | 备注 |
| :--- | :--- | :--- |
| 1 | 正常 | 用户可正常登录和使用 |
| 2 | 禁用 | 用户被暂时禁用，无法登录 |
| 3 | 已删除 | 用户已被删除（软删除） |

---

## API 列表

### 1. 用户列表
**接口**: `GET /api/v1/admin/users`

**描述**: 获取用户列表，支持分页、关键词搜索和用户类型筛选。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10，最大 100 |
| keyword | string | 否 | 搜索关键词（匹配手机号或昵称） |
| userType | int8 | 否 | 用户类型筛选：1-业主，2-服务商，3-工人，4-管理员 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 123,
        "phone": "13800138000",
        "nickname": "张三",
        "avatar": "http://example.com/avatar.jpg",
        "userType": 1,
        "status": 1,
        "createdAt": "2026-01-20T10:30:00Z",
        "updatedAt": "2026-01-25T15:45:00Z"
      },
      {
        "id": 124,
        "phone": "13900139000",
        "nickname": "李四",
        "avatar": "",
        "userType": 2,
        "status": 1,
        "createdAt": "2026-01-21T09:20:00Z",
        "updatedAt": "2026-01-21T09:20:00Z"
      }
    ],
    "total": 156
  }
}
```

**字段说明**:
- `list`: 用户列表数组
- `total`: 符合条件的用户总数
- `id`: 用户 ID
- `phone`: 手机号
- `nickname`: 用户昵称
- `avatar`: 头像 URL
- `userType`: 用户类型（1-业主，2-服务商，3-工人，4-管理员）
- `status`: 用户状态（1-正常，2-禁用，3-已删除）
- `createdAt`: 创建时间
- `updatedAt`: 更新时间

### 2. 用户详情
**接口**: `GET /api/v1/admin/users/:id`

**描述**: 获取指定用户的详细信息。

**认证**: 需要管理员 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | uint64 | 是 | 用户 ID |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 123,
    "phone": "13800138000",
    "nickname": "张三",
    "avatar": "http://example.com/avatar.jpg",
    "userType": 1,
    "status": 1,
    "createdAt": "2026-01-20T10:30:00Z",
    "updatedAt": "2026-01-25T15:45:00Z"
  }
}
```

**错误响应**:
```json
{
  "code": 404,
  "message": "用户不存在",
  "data": null
}
```

### 3. 创建用户
**接口**: `POST /api/v1/admin/users`

**描述**: 管理员创建新用户。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| phone | string | 是 | 11位手机号 |
| nickname | string | 否 | 用户昵称 |
| userType | int8 | 否 | 用户类型，默认 1（业主） |
| status | int8 | 否 | 用户状态，默认 1（正常） |

**请求示例**:
```json
{
  "phone": "13800138000",
  "nickname": "新用户",
  "userType": 1,
  "status": 1
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 125,
    "phone": "13800138000",
    "nickname": "新用户",
    "avatar": "",
    "userType": 1,
    "status": 1,
    "createdAt": "2026-01-25T16:00:00Z",
    "updatedAt": "2026-01-25T16:00:00Z"
  }
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

```json
{
  "code": 500,
  "message": "创建失败",
  "data": null
}
```

**业务规则**:
- 手机号必须唯一，不能重复
- 如果未指定 `status`，默认为 1（正常）
- 如果未指定 `userType`，默认为 0（由系统处理）
- 创建的用户没有密码，需要用户通过验证码登录后自行设置

### 4. 更新用户
**接口**: `PUT /api/v1/admin/users/:id`

**描述**: 更新用户信息（昵称、用户类型、状态）。

**认证**: 需要管理员 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | uint64 | 是 | 用户 ID |

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| nickname | string | 否 | 用户昵称 |
| userType | int8 | 否 | 用户类型 |
| status | int8 | 否 | 用户状态 |

**请求示例**:
```json
{
  "nickname": "更新后的昵称",
  "userType": 1,
  "status": 1
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 123,
    "phone": "13800138000",
    "nickname": "更新后的昵称",
    "avatar": "http://example.com/avatar.jpg",
    "userType": 1,
    "status": 1,
    "createdAt": "2026-01-20T10:30:00Z",
    "updatedAt": "2026-01-25T16:10:00Z"
  }
}
```

**错误响应**:
```json
{
  "code": 404,
  "message": "用户不存在",
  "data": null
}
```

```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

```json
{
  "code": 500,
  "message": "更新失败",
  "data": null
}
```

**业务规则**:
- 只更新提供的字段，未提供的字段保持不变
- `nickname` 为空字符串时不更新
- `userType` 为 0 时不更新
- `status` 字段始终会更新（包括 0 值）

### 5. 更新用户状态
**接口**: `PATCH /api/v1/admin/users/:id/status`

**描述**: 快速更新用户状态（启用/禁用/删除）。

**认证**: 需要管理员 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | uint64 | 是 | 用户 ID |

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| status | int8 | 是 | 用户状态：1-正常，2-禁用，3-已删除 |

**请求示例**:
```json
{
  "status": 2
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

```json
{
  "code": 500,
  "message": "更新失败",
  "data": null
}
```

**业务规则**:
- 状态为 2（禁用）时，用户无法登录系统
- 状态为 3（已删除）时，用户数据仍保留在数据库中（软删除）
- 禁用或删除用户不会影响已存在的订单和项目数据

---

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法或缺少必填参数 |
| 401 | 认证失败 | Token 过期或无效 |
| 403 | 权限不足 | 非管理员用户无法访问 |
| 404 | 用户不存在 | 指定的用户 ID 不存在 |
| 500 | 服务器错误 | 数据库操作失败或其他内部错误 |

## 使用示例

### 搜索用户 (Curl)
```bash
curl -X GET "http://localhost:8080/api/v1/admin/users?keyword=张三&page=1&pageSize=10" \
  -H "Authorization: Bearer <admin_token>"
```

### 按用户类型筛选
```bash
curl -X GET "http://localhost:8080/api/v1/admin/users?userType=2&page=1&pageSize=20" \
  -H "Authorization: Bearer <admin_token>"
```

### 创建用户
```bash
curl -X POST http://localhost:8080/api/v1/admin/users \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "13800138000",
    "nickname": "新用户",
    "userType": 1,
    "status": 1
  }'
```

### 禁用用户
```bash
curl -X PATCH http://localhost:8080/api/v1/admin/users/123/status \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": 2
  }'
```

## 注意事项

1. **权限验证**: 所有接口都需要管理员权限，普通用户无法访问。
2. **分页限制**: `pageSize` 最大值为 100，超过会被限制为 100。
3. **关键词搜索**: `keyword` 参数会同时匹配手机号和昵称（模糊匹配）。
4. **状态管理**: 
   - 禁用用户（status=2）会阻止用户登录，但不影响已有数据。
   - 删除用户（status=3）是软删除，数据仍保留在数据库中。
5. **手机号唯一性**: 创建用户时，手机号必须唯一，重复会导致创建失败。
6. **密码管理**: 管理员创建的用户没有初始密码，用户需要通过验证码登录后自行设置。
7. **用户类型**: 修改用户类型时需谨慎，可能影响用户在系统中的权限和功能访问。
