# 管理后台 - 订单与预约管理 API

**最后更新**: 2026-01-25

## 概述
管理后台订单与预约管理模块提供预约管理、退款处理和争议处理功能。管理员可以查看和管理所有预约订单，处理退款申请，以及裁决用户与服务商之间的争议。

## 预约状态说明
- `1` - 待处理（Pending）：用户已创建预约，等待商家响应
- `2` - 已确认（Confirmed）：商家已确认预约
- `3` - 已完成（Completed）：预约服务已完成
- `4` - 已取消（Cancelled）：预约已取消
- `5` - 争议中（Disputed）：预约进入争议处理流程

## 意向金退款说明

### 退款条件
意向金可退款需满足以下条件：
- 意向金已支付（`intent_fee_paid = true`）
- 意向金未退款（`intent_fee_refunded = false`）
- 意向金未抵扣（`intent_fee_deducted = false`）

### 退款场景
1. **商家超时未响应**：商家在48小时内未响应预约，系统自动退款
2. **商家拒绝接单**：商家主动拒绝预约，系统自动退款
3. **方案连续拒绝3次**：用户连续拒绝商家方案3次，系统自动退款
4. **管理员手动退款**：管理员根据实际情况手动触发退款

### 退款流程
1. 验证退款条件（已支付、未退款、未抵扣）
2. 更新预约记录退款状态
3. 发送退款通知给用户
4. 调用支付网关执行退款（待接入）

## API 列表

### 1. 获取预约列表
**接口**: `GET /api/v1/admin/bookings`

**描述**: 获取所有预约列表，支持按状态筛选和分页。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10，最大 100 |
| status | int | 否 | 预约状态筛选：1-待处理，2-已确认，3-已完成，4-已取消，5-争议中 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 123,
        "userId": 456,
        "providerId": 789,
        "serviceType": "design",
        "appointmentDate": "2026-02-01T10:00:00Z",
        "status": 1,
        "intentFee": 100.00,
        "intentFeePaid": true,
        "intentFeeRefunded": false,
        "intentFeeDeducted": false,
        "merchantResponseDeadline": "2026-01-27T10:00:00Z",
        "createdAt": "2026-01-25T10:00:00Z",
        "updatedAt": "2026-01-25T10:00:00Z"
      }
    ],
    "total": 150
  }
}
```

### 2. 更新预约状态
**接口**: `PATCH /api/v1/admin/bookings/:id/status`

**描述**: 管理员手动更新预约状态。

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 是 | 预约 ID |

**请求参数**:
```json
{
  "status": 2
}
```

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| status | int8 | 是 | 新状态：1-待处理，2-已确认，3-已完成，4-已取消，5-争议中 |

**响应示例**:
```json
{

  "message": "success",
  "data": null

```

**接口**: `GET /api/v1/admin/bookings/refundable`

**描述**: 获取所有符合退款条件的预约列表，用于批量退款或定时任务扫描。

**请求参数**: 无

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "bookings": [
      {
        "id": 123,
        "userId": 456,
        "providerId": 789,
        "status": 1,
        "intentFee": 100.00,
        "intentFeePaid": true,
        "intentFeeRefunded": false,
        "intentFeeDeducted": false,
        "merchantResponseDeadline": "2026-01-23T10:00:00Z",
        "createdAt": "2026-01-21T10:00:00Z"
```

**说明**:
- 返回已支付意向金、未退款、未抵扣、状态为待确认、且超过商家响应截止时间的预约
- 这些预约符合自动退款条件

## 争议处理

### 争议状态流转
1. **pending（待处理）** → 预约进入争议状态（status = 5）
2. **resolved（已解决）** → 管理员裁决后，预约状态更新为已取消（status = 4）

### 争议解决方案类型
- `refund_user` - 全额退款给用户
- `refund_partial` - 部分退款（需指定退款比例）
- `cancel_no_refund` - 取消预约但不退款（用户责任）
- `reassign` - 重新分配商家（暂未开放）

### 7. 处理争议
**接口**: `POST /api/v1/admin/disputed-bookings/:id/resolve`

**描述**: 管理员裁决争议预约，支持全额退款、部分退款、不退款等多种处理方式。

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 是 | 预约 ID |

**请求参数**:
```json
{
  "resolution": "refund_user",
  "reason": "经核实，商家方案质量不达标，支持用户诉求",
  "refundRate": 0.5
}
```

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| resolution | string | 是 | 处理方式：`refund_user`（全额退款）、`refund_partial`（部分退款）、`cancel_no_refund`（不退款）、`reassign`（重新分配，暂未开放） |
| reason | string | 否 | 处理原因说明 |
| refundRate | float64 | 条件 | 部分退款比例（0-1），仅当 `resolution = refund_partial` 时必填 |

**响应示例**:
```json
{
  "code": 0,
  "message": "处理成功"
}
```

**错误响应**:
```json
{
  "code": 1,
  "error": "该预约不在争议状态"
}
```

```json
{
  "code": 1,
  "error": "无效的处理方式"
}
```

```json
{
  "code": 1,
  "error": "重新分配功能暂未开放"
}
```

**处理逻辑说明**:

#### 全额退款（refund_user）
- 预约状态更新为已取消（status = 4）
- 标记意向金已退款（`intent_fee_refunded = true`）
- 记录退款原因：`"平台裁定：全额退款 - {reason}"`
- 发送通知给用户

#### 部分退款（refund_partial）
- 预约状态更新为已取消（status = 4）
- 标记意向金已退款（`intent_fee_refunded = true`）
- 计算退款金额：`refundAmount = intentFee × refundRate`
- 记录退款原因：`"平台裁定：部分退款 {refundAmount}元 - {reason}"`

#### 不退款（cancel_no_refund）
- 预约状态更新为已取消（status = 4）
- 标记意向金未退款（`intent_fee_refunded = false`）
- 记录退款原因：`"平台裁定：不予退款 - {reason}"`
- 发送通知给用户说明原因

## 使用示例

### 手动退款请求 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/admin/bookings/123/refund \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <admin_token>" \
  -d '{
    "reason": "用户投诉，经核实商家服务态度不佳"
  }'
```

### 处理争议请求 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/admin/disputed-bookings/123/resolve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <admin_token>" \
  -d '{
    "resolution": "refund_partial",
    "reason": "双方各有责任，协商部分退款",
    "refundRate": 0.5
  }'
```

### 获取可退款预约列表 (Curl)
```bash
curl -X GET http://localhost:8080/api/v1/admin/bookings/refundable \
  -H "Authorization: Bearer <admin_token>"
```
