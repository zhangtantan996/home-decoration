# 财务管理 API

**最后更新**: 2026-01-25

## 概述
财务管理模块负责管理平台的资金托管账户和商家提现申请。管理员可以通过这些接口查看托管账户状态、处理提现申请、审核资金释放等操作。

## 业务流程说明

### 托管账户流程
1. 业主支付项目款项后，资金进入托管账户
2. 托管账户包含总金额、冻结金额、可用金额、已释放金额
3. 项目完成阶段验收后，管理员可释放对应金额给服务商
4. 账户状态：0-待激活，1-正常，2-冻结，3-已清算

### 提现审核流程
1. 商家提交提现申请（状态：pending）
2. 管理员审核提现申请
3. 审核通过（状态：approved → completed）：
   - 更新提现记录状态为成功
   - 更新关联的收入记录状态为已提现
   - 调用银行 API 实际打款（TODO）
   - 发送通知给商家
4. 审核拒绝（状态：rejected）：
   - 更新提现记录状态为失败
   - 记录拒绝原因
   - 收入记录保持已结算状态，商家可重新申请
   - 发送通知给商家

### 提现状态流转
```
pending (0) → approved (1) → completed (1)
            ↘ rejected (2)
```

### 手续费说明
- 提现手续费：根据平台配置收取，通常为提现金额的 0.1%-1%
- 手续费计算：实际到账金额 = 提现金额 - 手续费
- 手续费由商家承担，从提现金额中扣除

## API 列表

## 一、托管账户管理

### 1.1 托管账户列表
**接口**: `GET /api/v1/admin/escrow-accounts`

**描述**: 获取所有托管账户列表，支持分页查询。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10，最大 100 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "projectId": 101,
        "userId": 123,
        "projectName": "三居室现代简约装修",
        "userName": "张三",
        "totalAmount": 100000.00,
        "frozenAmount": 30000.00,
        "availableAmount": 50000.00,
        "releasedAmount": 20000.00,
        "status": 1,
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T15:30:00Z"
      }
    ],
    "total": 50
  }
}
```

**字段说明**:
- `totalAmount`: 托管总金额
- `frozenAmount`: 冻结金额（待验收阶段）
- `availableAmount`: 可用金额（已存入但未分配）
- `releasedAmount`: 已释放金额（已支付给服务商）
- `status`: 账户状态（0-待激活，1-正常，2-冻结，3-已清算）

---

### 1.2 托管账户详情
**接口**: `GET /api/v1/admin/escrow-accounts/:id`

**描述**: 获取指定托管账户的详细信息，包括关联的交易记录。

**认证**: 需要管理员 Token

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "account": {
      "id": 1,
      "projectId": 101,
      "userId": 123,
      "projectName": "三居室现代简约装修",
      "userName": "张三",
      "totalAmount": 100000.00,
      "frozenAmount": 30000.00,
      "availableAmount": 50000.00,
      "releasedAmount": 20000.00,
      "status": 1,
      "createdAt": "2026-01-20T10:00:00Z",
      "updatedAt": "2026-01-25T15:30:00Z"
    },
    "transactions": [
      {
        "id": 1,
        "orderId": "TXN202601251234567890",
        "escrowId": 1,
        "milestoneId": 0,
        "type": "deposit",
        "amount": 50000.00,
        "fromUserId": 123,
        "fromAccount": "用户账户",
        "toUserId": 0,
        "toAccount": "托管账户",
        "status": 1,
        "remark": "首期款",
        "completedAt": "2026-01-20T10:30:00Z",
        "createdAt": "2026-01-20T10:00:00Z"
      }
    ]
  }
}
```

**交易类型说明**:
- `deposit`: 存入（业主支付到托管账户）
- `withdraw`: 提现（从托管账户提现）
- `transfer`: 转账（托管账户转给服务商）
- `refund`: 退款（退回给业主）

**交易状态说明**:
- `0`: 处理中
- `1`: 成功
- `2`: 失败

---

### 1.3 释放托管资金
**接口**: `POST /api/v1/admin/escrow-accounts/:id/release`

**描述**: 管理员手动释放托管账户资金，通常用于阶段验收后支付给服务商。

**认证**: 需要管理员 Token

**请求参数**:
```json
{
  "amount": 30000.00,
  "remark": "水电阶段验收通过，释放第二期款"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| amount | float64 | 是 | 释放金额，必须大于 0 |
| remark | string | 否 | 备注说明 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "资金释放成功",
    "transactionId": 123
  }
}
```

**业务规则**:
1. 释放金额不能超过账户可用金额
2. 账户状态必须为正常（status=1）
3. 释放后自动创建交易记录（type=transfer）
4. 更新账户的 availableAmount 和 releasedAmount
5. 发送通知给服务商

**错误响应**:
```json
{
  "code": 400,
  "message": "余额不足",
  "data": null
}
```

## 二、提现管理

### 2.1 提现申请列表
**接口**: `GET /api/v1/admin/withdrawals`

**描述**: 获取商家提现申请列表，支持按状态和商家筛选。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 20，最大 100 |
| status | int8 | 否 | 提现状态：0-处理中，1-成功，2-失败 |
| providerId | uint64 | 否 | 商家 ID，筛选指定商家的提现记录 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "providerId": 45,
        "providerName": "张三设计工作室",
        "orderNo": "WD20260125123456",
        "amount": 50000.00,
        "bankAccount": "6222 **** **** 1234",
        "bankName": "中国工商银行",
        "status": 0,
        "statusLabel": "处理中",
        "failReason": "",
        "completedAt": null,
        "createdAt": "2026-01-25T10:00:00Z"
      }
    ],
    "total": 30,
    "page": 1,
    "pageSize": 20
  }
}
```

**字段说明**:
- `orderNo`: 提现订单号，格式 WD + 时间戳 + 商家ID
- `bankAccount`: 收款银行账户（脱敏显示）
- `status`: 提现状态（0-处理中，1-成功，2-失败）
- `statusLabel`: 状态标签（处理中、成功、失败）
- `failReason`: 失败原因（status=2 时有值）

### 2.3 审核通过提现
**接口**: `POST /api/v1/admin/withdrawals/:id/approve`

**描述**: 管理员审核通过提现申请，系统将自动处理打款流程。

**认证**: 需要管理员 Token

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "审核通过，提现成功"
  }
}
```

**业务规则**:
1. 只有状态为 0（处理中）的申请可以审核
2. 审核通过后自动执行以下操作：
   - 更新提现记录状态为 1（成功）
   - 记录完成时间
   - 更新关联的收入记录状态为 2（已提现）
   - 记录提现订单号到收入记录
   - 调用银行 API 实际打款（TODO）
   - 发送通知给商家
3. 所有操作在事务中执行，失败时自动回滚

**错误响应**:
```json
{
  "code": 400,
  "message": "该提现申请已处理",
  "data": null
}
```

---

### 2.4 审核拒绝提现
**接口**: `POST /api/v1/admin/withdrawals/:id/reject`

**描述**: 管理员拒绝提现申请，需要提供拒绝原因。

**认证**: 需要管理员 Token

**请求参数**:
```json
{
  "reason": "银行账户信息有误，请核实后重新提交"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| reason | string | 是 | 拒绝原因，必填 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "已拒绝该提现申请"
  }
}
```

**业务规则**:
1. 只有状态为 0（处理中）的申请可以拒绝
2. 拒绝后执行以下操作：
   - 更新提现记录状态为 2（失败）
   - 记录拒绝原因
   - 不修改收入记录状态（保持已结算状态）
   - 商家可以重新提交提现申请
   - 发送通知给商家
3. 拒绝原因会显示给商家，请填写清晰明确的说明

**错误响应**:
```json
{
  "code": 400,
  "message": "该提现申请已处理",
  "data": null
}
```

## 错误码说明

| 错误码 | 说明 | 备注 |
|-------|------|------|
| 400 | 参数错误 | 请求参数不合法或缺少必填参数 |
| 404 | 资源不存在 | 托管账户或提现记录不存在 |
| 500 | 服务器错误 | 数据库操作失败或其他内部错误 |

## 使用示例

### 查询提现申请列表 (Curl)
```bash
curl -X GET "http://localhost:8080/api/v1/admin/withdrawals?status=0&page=1&pageSize=20" \
  -H "Authorization: Bearer <admin_token>"
```

### 审核通过提现 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/admin/withdrawals/1/approve \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json"
```

### 审核拒绝提现 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/admin/withdrawals/1/reject \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "银行账户信息有误，请核实后重新提交"
  }'
```

### 释放托管资金 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/admin/escrow-accounts/1/release \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 30000.00,
    "remark": "水电阶段验收通过，释放第二期款"
  }'
```

## 注意事项

### 托管账户管理
1. **资金安全**: 托管账户资金必须与项目进度绑定，不可随意释放
2. **状态管理**: 账户冻结时（status=2）不允许释放资金
3. **金额校验**: 释放金额必须小于等于可用金额
4. **交易记录**: 每次资金变动都会生成交易记录，便于审计
5. **通知机制**: 资金释放后需通知服务商和业主

### 提现审核
1. **审核时效**: 建议在 24 小时内完成审核，避免商家等待过久
2. **拒绝原因**: 必须填写清晰明确的拒绝原因，帮助商家改进
3. **重复申请**: 拒绝后商家可重新提交，收入记录保持已结算状态
4. **打款流程**: 审核通过后需调用银行 API 实际打款（当前为 TODO）
5. **手续费**: 提现手续费从提现金额中扣除，实际到账金额 = 提现金额 - 手续费

### 手续费计算
1. **费率配置**: 手续费费率在系统配置中设置，通常为 0.1%-1%
2. **最低手续费**: 可设置最低手续费金额，如 1 元
3. **最高手续费**: 可设置最高手续费金额，如 100 元
4. **免手续费**: 特定商家或活动期间可设置免手续费
5. **手续费记录**: 手续费金额记录在交易记录中，便于财务对账

### 权限控制
- 所有接口需要管理员 Token
- 建议记录操作日志（操作人、操作时间、操作内容）
- 敏感操作（释放资金、审核通过）建议二次确认
- 大额提现（如超过 10 万）建议增加审批流程

### 性能优化
1. 列表接口使用分页，避免一次性加载大量数据
2. pageSize 限制最大 100，防止 DoS 攻击
3. 提现审核使用事务，确保数据一致性
4. 通知发送使用异步队列，避免阻塞主流程

### 安全建议
1. **银行账户**: 存储时脱敏，显示时只显示后 4 位
2. **金额校验**: 严格校验金额范围，防止负数或超大金额
3. **并发控制**: 使用数据库锁防止并发审核同一申请
4. **审计日志**: 记录所有财务操作，便于追溯和审计
5. **风控规则**: 设置单日提现限额、单笔提现限额等风控规则
