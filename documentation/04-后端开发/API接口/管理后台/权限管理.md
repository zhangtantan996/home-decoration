# 权限管理 API

**最后更新**: 2026-01-25

## 概述
权限管理模块基于 RBAC（Role-Based Access Control）模型实现，提供管理员账号管理、角色管理、菜单权限管理等功能。系统通过"管理员-角色-菜单"的多对多关系实现灵活的权限控制。

## RBAC 模型说明

### 核心概念
- **管理员（SysAdmin）**: 系统操作者，可分配多个角色
- **角色（SysRole）**: 权限集合的载体，如"系统管理员"、"财务专员"、"客服"
- **菜单（SysMenu）**: 包含目录、菜单页面、按钮权限，每个菜单可关联权限标识
- **权限标识（Permission）**: 格式为 `模块:资源:操作`，如 `system:user:list`

### 关系模型
```
管理员 (SysAdmin) ←→ sys_admin_roles ←→ 角色 (SysRole) ←→ sys_role_menus ←→ 菜单 (SysMenu)
```

### 权限计算规则
1. **超级管理员**: `isSuperAdmin=true` 拥有所有权限 `*:*:*`，可访问所有菜单
2. **普通管理员**: 通过角色获取权限
   - 权限列表 = 所有角色关联的菜单中的 `permission` 字段（去重）
   - 菜单树 = 所有角色关联的菜单（type=1目录或2菜单，visible=true，status=1启用）

### 菜单类型
- **Type 1 - 目录**: 一级导航，如"系统管理"
- **Type 2 - 菜单**: 二级页面，如"用户列表"
- **Type 3 - 按钮**: 页面内操作权限，如"新增"、"删除"

## 一、管理员管理

### 1. 获取管理员列表
**接口**: `GET /api/v1/admin/admins`

**描述**: 分页查询管理员列表，支持关键词搜索（用户名/手机号/昵称）。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10 |
| keyword | string | 否 | 搜索关键词 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "username": "admin",
        "nickname": "系统管理员",
        "phone": "13800138000",
        "email": "admin@example.com",
        "avatar": "http://example.com/avatar.jpg",
        "status": 1,
        "isSuperAdmin": true,
        "lastLoginAt": "2026-01-25T10:30:00Z",
        "lastLoginIp": "192.168.1.100",
        "roles": [
          {
            "id": 1,
            "name": "超级管理员",
            "key": "super_admin",
            "status": 1
          }
        ],
        "createdAt": "2026-01-01T00:00:00Z",
        "updatedAt": "2026-01-25T10:30:00Z"
      }
    ],
    "total": 15
  }
}
```

### 2. 创建管理员
**接口**: `POST /api/v1/admin/admins`

**描述**: 创建新管理员并分配角色。密码自动加密存储（bcrypt）。

**请求参数**:
```json
{
  "username": "zhangsan",
  "phone": "13900139000",
  "email": "zhangsan@example.com",
  "password": "Password123!",
  "nickname": "张三",
  "roleIds": [2, 3]
}
```

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| username | string | 是 | 用户名，唯一 |
| phone | string | 否 | 手机号，唯一 |
| email | string | 否 | 邮箱 |
| password | string | 是 | 密码，最少 6 位 |
| nickname | string | 否 | 昵称 |
| roleIds | []uint64 | 是 | 角色 ID 列表 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 10,
    "username": "zhangsan",
    "nickname": "张三",
    "phone": "13900139000",
    "email": "zhangsan@example.com",
    "status": 1,
    "isSuperAdmin": false,
    "roles": [
      {
        "id": 2,
        "name": "财务专员",
        "key": "finance"
      }
    ],
    "createdAt": "2026-01-25T11:00:00Z"
  }
}
```

**错误码**:
- `400`: 用户名已存在 / 手机号已存在 / 参数错误

### 3. 更新管理员
**接口**: `PUT /api/v1/admin/admins/:id`

**描述**: 更新管理员基本信息和角色。密码为可选字段，提供时才更新。

**请求参数**:
```json
{
  "username": "zhangsan_new",
  "phone": "13900139001",
  "email": "zhangsan_new@example.com",
  "nickname": "张三（新）",
  "password": "NewPassword123!",
  "roleIds": [2, 4]
}
```

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| username | string | 否 | 用户名 |
| phone | string | 否 | 手机号 |
| email | string | 否 | 邮箱 |
| nickname | string | 否 | 昵称 |
| password | string | 否 | 新密码，最少 6 位 |
| roleIds | []uint64 | 否 | 角色 ID 列表 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 10,
    "username": "zhangsan_new",
    "nickname": "张三（新）",
    "roles": [
      {
        "id": 2,
        "name": "财务专员"
      },
      {
        "id": 4,
        "name": "客服专员"
      }
    ],
    "updatedAt": "2026-01-25T12:00:00Z"
  }
}
```

**错误码**:
- `404`: 管理员不存在
- `400`: 用户名已被占用 / 手机号已被占用

---

### 4. 删除管理员
**接口**: `DELETE /api/v1/admin/admins/:id`

**描述**: 删除管理员及其角色关联。超级管理员不可删除。

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**错误码**:
- `404`: 管理员不存在
- `400`: 不能删除超级管理员

### 5. 修改管理员密码
**接口**: `PATCH /api/v1/admin/admins/:id/password`

**描述**: 管理员修改自己或其他管理员的密码（需权限）。

**请求参数**:
```json
{
  "password": "NewPassword123!"
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**说明**: 此接口在 `AdminUpdateAdmin` 中通过 `password` 字段实现，也可单独实现专用接口。

**接口**: `DELETE /api/v1/admin/roles/:id`


### 10. 分配角色菜单权限
**接口**: `POST /api/v1/admin/roles/:id/permissions`

**描述**: 为角色分配菜单权限。会先删除原有关联，再创建新关联（全量替换）。

**请求参数**:
```json
{
  "menuIds": [1, 2, 3, 10, 11, 12]
}
```

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| menuIds | []uint64 | 是 | 菜单 ID 列表 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**说明**: 
- 传入空数组 `[]` 表示清空该角色的所有权限
- 菜单 ID 应包含目录、菜单、按钮的完整权限树

## 权限分配流程

### 典型场景：为新员工配置权限

#### 步骤 1: 创建角色（如果不存在）
```bash
POST /api/v1/admin/roles
{
  "name": "订单管理员",
  "key": "order_manager",
  "remark": "管理订单和项目"
}
```

#### 步骤 2: 为角色分配菜单权限
```bash
POST /api/v1/admin/roles/5/permissions
{
  "menuIds": [1, 5, 6, 50, 51, 52]  # 包含目录、菜单、按钮
}
```

#### 步骤 3: 创建管理员并分配角色
```bash
POST /api/v1/admin/admins
{
  "username": "lisi",
  "password": "Password123!",
  "nickname": "李四",
  "roleIds": [5]
}
```

#### 步骤 4: 管理员登录获取权限
```bash
POST /api/v1/admin/auth/login
{
  "username": "lisi",
  "password": "Password123!"
}

# 响应包含：
# - permissions: ["order:list", "order:create", "order:update"]
# - menus: 菜单树（前端渲染导航）
```

## 菜单树结构说明

### 三层结构示例
```
系统管理 (目录, type=1, parentId=0)
├── 用户管理 (菜单, type=2, parentId=1)
│   ├── 新增用户 (按钮, type=3, parentId=2, permission="system:user:create")
│   ├── 编辑用户 (按钮, type=3, parentId=2, permission="system:user:update")
│   └── 删除用户 (按钮, type=3, parentId=2, permission="system:user:delete")
└── 角色管理 (菜单, type=2, parentId=1)
    ├── 新增角色 (按钮, type=3, parentId=3, permission="system:role:create")
    └── 分配权限 (按钮, type=3, parentId=3, permission="system:role:assign")
```

### 前端权限控制
```typescript
// 按钮权限控制
const hasPermission = (permission: string) => {
  return permissions.includes(permission) || permissions.includes('*:*:*');
};

// 使用示例
{hasPermission('system:user:create') && <Button>新增用户</Button>}
```

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法或业务规则限制 |

## 使用示例

### 获取角色列表并分配权限 (Curl)
```bash
# 1. 获取所有角色
curl -X GET http://localhost:8080/api/v1/admin/roles \
  -H "Authorization: Bearer <admin_token>"

# 2. 获取菜单树
curl -X GET http://localhost:8080/api/v1/admin/menus \
  -H "Authorization: Bearer <admin_token>"

# 3. 为角色分配权限
curl -X POST http://localhost:8080/api/v1/admin/roles/5/permissions \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "menuIds": [1, 2, 3, 10, 11, 12]
  }'
```

### 创建管理员并分配角色 (JavaScript)
```javascript
const createAdmin = async () => {
  const response = await fetch('http://localhost:8080/api/v1/admin/admins', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${adminToken}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      username: 'wangwu',
      password: 'Password123!',
      nickname: '王五',
      phone: '13700137000',
      roleIds: [2, 3]  // 财务专员 + 客服专员
    })
  });
  
  const result = await response.json();
  console.log('管理员创建成功:', result.data);
};
```

## 安全建议

1. **密码策略**: 
   - 最少 6 位（建议 8 位以上）
   - 包含大小写字母、数字、特殊字符
   - 使用 bcrypt 加密存储

2. **权限最小化原则**:
   - 仅分配必要的角色和权限
   - 定期审计管理员权限

3. **超级管理员保护**:
   - 不可删除
   - 不可禁用
   - 建议限制超级管理员数量

4. **操作日志**:
   - 所有权限变更应记录操作日志
   - 包含操作人、时间、IP、变更内容

5. **Token 管理**:
   - 管理员 Token 有效期 60 分钟
   - 登录失败 5 次锁定 30 分钟
   - 使用 Redis 记录失败次数

```
模块:资源:操作
```

### 示例
- `system:user:list` - 查看用户列表
- `system:user:create` - 创建用户
- `system:user:update` - 更新用户
- `system:user:delete` - 删除用户
- `system:role:assign` - 分配角色权限
- `order:project:approve` - 审批项目
- `finance:escrow:release` - 释放托管资金
- `*:*:*` - 所有权限（超级管理员）

### 模块分类
- `system` - 系统管理
- `user` - 用户管理
- `provider` - 服务商管理
- `order` - 订单管理
- `finance` - 财务管理
- `content` - 内容管理
- `audit` - 审核管理
