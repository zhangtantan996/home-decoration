# 商家管理 API

**最后更新**: 2026-01-25

## 概述
商家管理模块负责管理平台上的服务商（设计师、装修公司、工长）、商家入驻申请审核以及主材门店管理。管理员可以通过这些接口对商家进行审核、认证、状态管理等操作。

## 服务商类型说明

| 类型值 | 类型名称 | 说明 |
|-------|---------|------|
| 1 | 设计师 | 独立设计师或设计工作室 |
| 2 | 装修公司 | 装修施工公司 |
| 3 | 工长 | 独立工长或施工队 |

## 审核状态说明

### 商家申请状态
| 状态值 | 状态名称 | 说明 |
|-------|---------|------|
| 0 | 待审核 | 申请已提交，等待管理员审核 |
| 1 | 审核通过 | 申请通过，已创建商家账号 |
| 2 | 审核拒绝 | 申请被拒绝，商家可重新提交 |

### 认证状态
| 状态 | 说明 |
|------|------|
| verified: true | 已认证，显示认证标识 |
| verified: false | 未认证，不显示认证标识 |

### 服务商状态
| 状态值 | 状态名称 | 说明 |
|-------|---------|------|
| 1 | 正常 | 可正常接单和提供服务 |
| 2 | 禁用 | 被封禁，无法接单 |

## 一、服务商管理

### 1.1 服务商列表
**接口**: `GET /api/v1/admin/providers`

**描述**: 获取服务商列表，支持按类型和认证状态筛选。

|-------|------|------|------|

| pageSize | int | 否 | 每页数量，默认 10，最大 100 |
| type | int8 | 否 | 服务商类型：1-设计师，2-装修公司，3-工长 |
| verified | string | 否 | 认证状态：true-已认证，false-未认证 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {

      {
        "id": 1,
        "userId": 123,
        "providerType": 1,
        "subType": "personal",
        "companyName": "张三设计工作室",

        "specialty": "[\"现代简约\",\"北欧风格\"]",

        "yearsExperience": 5,
        "rating": 4.8,
```

**字段说明**:
- `subType`: 子类型（personal-个人，studio-工作室，company-公司）

### 1.2 创建服务商
**接口**: `POST /api/v1/admin/providers`

**描述**: 管理员手动创建服务商账号。

**认证**: 需要管理员 Token

**请求参数**:
```json
{
  "userId": 123,
  "providerType": 1,
  "companyName": "张三设计工作室",
  "subType": "personal",
  "specialty": "现代简约,北欧风格",
  "yearsExperience": 5,
  "status": 1
}
```

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| userId | uint64 | 否 | 关联的用户 ID（如不提供需先创建用户） |
| providerType | int8 | 是 | 服务商类型：1-设计师，2-装修公司，3-工长 |
| companyName | string | 是 | 公司/工作室名称 |
| subType | string | 否 | 子类型，默认 personal |
| specialty | string | 否 | 擅长领域 |
| yearsExperience | int | 否 | 从业年限 |
| status | int8 | 否 | 状态，默认 1（正常） |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "userId": 123,
    "providerType": 1,
    "companyName": "张三设计工作室",
    "subType": "personal",
    "status": 1,
    "verified": false,
    "createdAt": "2026-01-25T16:00:00Z"
  }
}
```

3. status 字段始终更新（包括值为 0 的情况）

### 2.3 审核通过
**接口**: `POST /api/v1/admin/merchant-applications/:id/approve`

**描述**: 审核通过商家入驻申请，自动创建用户账号和服务商账号。

**认证**: 需要管理员 Token

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "审核通过",
    "userId": 123,
    "providerId": 45
  }
}
```

**业务规则**:
1. 只有状态为 0（待审核）的申请可以审核
2. 审核通过后自动执行以下操作：
   - 创建 User 账号（userType=2，status=1）
   - 创建 Provider 账号（verified=true，status=1）
   - 迁移作品集到 ProviderCase 表（showInInspiration=true）
   - 创建 MerchantServiceSetting（acceptBooking=true）
   - 更新申请状态为 1（审核通过）
   - 记录审核人和审核时间
   - 异步同步商家到腾讯云 IM
3. 作品风格未填写时，回退到商家擅长风格的第一个选项
4. 作品风格为空时，保底使用"现代简约"
5. 所有操作在事务中执行，失败时自动回滚

**错误响应**:
```json
{
  "code": 400,
  "message": "该申请已处理",
  "data": null
}
```

---

### 2.4 审核拒绝
**接口**: `POST /api/v1/admin/merchant-applications/:id/reject`

**描述**: 拒绝商家入驻申请，需要提供拒绝原因。

**认证**: 需要管理员 Token

**请求参数**:
```json
{
  "reason": "提交的资质证明不清晰，请重新上传"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| reason | string | 是 | 拒绝原因 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "已拒绝"
  }
}
```

**业务规则**:
1. 只有状态为 0（待审核）的申请可以拒绝
2. 拒绝后商家可以重新提交申请
3. 更新申请状态为 2（审核拒绝）
4. 记录拒绝原因、审核人和审核时间

**错误响应**:
```json
{
  "code": 400,
  "message": "该申请已处理",
  "data": null
}
```

```bash
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 4.8,
```

### 封禁服务商 (Curl)
```bash
curl -X PATCH http://localhost:8080/api/v1/admin/providers/1/status \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": 2
  }'
```

## 注意事项

### 商家申请审核
1. **审核流程**: 待审核 → 审核通过/拒绝
2. **审核通过**: 自动创建用户和服务商账号，迁移作品集
3. **审核拒绝**: 商家可重新提交申请
4. **作品集要求**: 至少 3 个作品案例
5. **服务区域**: 支持区域代码和名称自动转换

### 服务商管理
1. **认证标识**: verified 字段控制是否显示认证标识
2. **状态管理**: status=2 时服务商无法接单
3. **评分系统**: rating（综合评分）、restoreRate（还原度）、budgetControl（预算控制力）
4. **服务区域**: 使用区域代码数组，支持自动转换

### 主材门店管理
1. **门店类型**: 瓷砖、地板、卫浴、橱柜、灯具等
2. **认证门店**: 优先展示，显示认证标识
3. **删除操作**: 物理删除，数据无法恢复
4. **收藏功能**: 用户可收藏门店，favoriteCount 记录收藏数

### 权限控制
- 所有接口需要管理员 Token
- 建议记录操作日志（审核人、操作时间、操作内容）
- 敏感操作（删除、封禁）建议二次确认

### 性能优化
1. 列表接口使用分页，避免一次性加载大量数据
2. pageSize 限制最大 100，防止 DoS 攻击
3. 服务区域转换使用缓存，减少数据库查询
4. 作品集迁移使用事务，确保数据一致性
