# 管理后台 - 项目管理 API

**最后更新**: 2026-01-25

## 概述
项目管理模块为管理员提供项目全生命周期的管理能力，包括项目列表查询、详情查看、状态更新、阶段管理以及施工日志管理。管理员可以监控所有项目的进展情况，协调各方资源，确保项目顺利推进。

## 项目状态说明

项目状态（`status`）采用 `int8` 类型，定义如下：

| 状态值 | 状态名称 | 说明 |
| :--- | :--- | :--- |
| -1 | 待创建 | 已生成设计方案但未完善项目信息 |
| 0 | 待准备 | 项目已创建，等待开工准备 |
| 1 | 进行中 | 项目正在施工 |
| 2 | 已完成 | 项目已竣工验收 |
| 3 | 已暂停 | 项目因故暂停施工 |
| 4 | 已取消 | 项目已取消 |

**状态流转规则**：
- 待准备 → 进行中 → 已完成（正常流程）
- 任意状态 → 已暂停 → 进行中（可恢复）
- 任意状态 → 已取消（不可逆）

## 项目阶段类型说明

项目阶段（`phaseType`）定义了装修施工的标准流程：

| 阶段类型 | 中文名称 | 顺序 | 预估工期 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| preparation | 准备阶段 | 1 | 4天 | 现场交接、图纸确认、材料进场 |
| demolition | 拆除阶段 | 2 | 7天 | 墙体拆除、地面拆除、垃圾清运 |
| electrical | 水电阶段 | 3 | 10天 | 水管布置、电路布线、水电验收 |
| masonry | 泥木阶段 | 4 | 15天 | 瓷砖铺贴、木工制作、吊顶施工 |
| painting | 油漆阶段 | 5 | 10天 | 墙面处理、乳胶漆施工 |
| installation | 安装阶段 | 6 | 7天 | 灯具、洁具、五金安装 |
| inspection | 验收阶段 | 7 | 3天 | 全屋保洁、设备调试、交付验收 |

**阶段状态**（`status`）：
- `pending`：待开始
- `in_progress`：进行中
- `completed`：已完成

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 0 | 成功 | 操作成功 |
| 1 | 业务错误 | 具体错误信息见 error 字段 |
| 400 | 参数错误 | 请求参数不合法 |
| 401 | 认证失败 | Token 过期或无效 |
| 403 | 权限不足 | 非管理员用户无权访问 |

## 使用示例

### 获取项目列表（Curl）
```bash
curl -X GET "http://localhost:8080/api/v1/admin/projects?page=1&pageSize=20&status=1" \
  -H "Authorization: Bearer <admin_token>"
```

### 更新项目状态（Curl）
```bash
curl -X PUT http://localhost:8080/api/v1/admin/projects/1001/status \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": 1,
    "currentPhase": "水电阶段"
  }'
```

### 创建施工日志（Curl）
```bash
curl -X POST http://localhost:8080/api/v1/admin/projects/1001/phases/3/logs \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "水电布线进度更新",
    "description": "客厅和主卧电路布线完成",
    "photos": "[\"https://cdn.example.com/photo1.jpg\"]",
    "logDate": "2026-01-25"
  }'
```

## 注意事项

1. **权限要求**：所有接口均需要管理员权限，请求时必须携带有效的管理员 Token。
2. **日期格式**：所有日期参数统一使用 `YYYY-MM-DD` 格式（如 `2026-01-25`）。
3. **照片字段**：`photos` 字段为 JSON 数组字符串，需要进行 JSON 序列化。
4. **状态流转**：更新项目状态时需遵循状态流转规则，避免非法状态转换。
5. **阶段顺序**：项目阶段按照 `seq` 字段顺序执行，建议按顺序推进。
6. **数据完整性**：删除施工日志为物理删除，操作不可逆，请谨慎使用。
