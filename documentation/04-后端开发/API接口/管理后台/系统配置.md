# 管理后台 - 系统配置 API

**最后更新**: 2026-01-25

## 概述
系统配置模块为管理员提供平台配置管理功能,包括系统设置、业务配置、字典管理、区域管理和操作日志查询。管理员可以动态调整平台参数、管理数据字典、配置行政区划,并追踪操作记录。

## 配置类型说明

### 系统设置 (SystemSettings)
通用系统设置,存储在 `system_settings` 表中,支持分类管理。

**分类说明**:
| 分类 | 说明 | 示例 |
| :--- | :--- | :--- |
| basic | 基础设置 | 网站名称、Logo、联系方式 |
| security | 安全设置 | 密码策略、登录限制 |
| payment | 支付设置 | 支付方式、支付配置 |
| sms | 短信设置 | 短信服务商、模板配置 |

### 系统配置 (SystemConfig)
业务配置项,存储在 `system_configs` 表中,支持类型化管理。

**配置类型**:
| 类型 | 说明 | 示例 |
| :--- | :--- | :--- |
| string | 字符串 | 文本配置 |
| number | 数字 | 金额、比例 |
| boolean | 布尔值 | 开关配置 |
| json | JSON对象 | 复杂配置 |

**常用配置项**:
| 配置键 | 说明 | 类型 | 默认值 |
| :--- | :--- | :--- | :--- |
| booking.intent_fee | 意向金金额 | number | 500 |
| booking.intent_fee_refundable | 意向金是否可退 | boolean | true |
| order.design_fee_unlock_download | 支付设计费后解锁下载 | boolean | true |
| fee.platform.intent_fee_rate | 意向金抽成比例 | number | 0.05 |
| fee.platform.design_fee_rate | 设计费抽成比例 | number | 0.10 |
| fee.platform.construction_fee_rate | 施工费抽成比例 | number | 0.08 |
| withdraw.min_amount | 最小提现金额 | number | 100 |
| im.tencent_enabled | 是否启用腾讯云IM | boolean | false |
| im.tencent_sdk_app_id | 腾讯云IM SDKAppID | string | - |

## API 列表

## 一、系统设置

### 1. 获取系统设置
**接口**: `GET /api/v1/admin/settings`

**描述**: 获取所有系统设置,返回键值对格式。

**认证**: 需要管理员 Token

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "site_name": "装修设计一体化平台",
    "site_logo": "https://example.com/logo.png",
    "contact_phone": "400-123-4567",
    "contact_email": "support@example.com",
    "im_tencent_enabled": "true",
    "im_tencent_sdk_app_id": "1400123456"
  }
}
```

**字段说明**:
- 返回的是 `key: value` 格式的对象
- 所有值都是字符串类型
- 前端需要根据业务逻辑进行类型转换

---

### 2. 更新系统设置
**接口**: `PUT /api/v1/admin/settings`

**描述**: 批量更新系统设置。支持同时更新 `system_settings` 和 `system_configs` 两个表。

**认证**: 需要管理员 Token

**请求参数**:
```json
{
  "site_name": "新的网站名称",
  "contact_phone": "400-999-8888",
  "im_tencent_enabled": "true",
  "im_tencent_sdk_app_id": "1400999888"
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**特殊说明**:
- 前端使用 `im_tencent_*` 格式,后端自动转换为 `im.tencent_*` 格式存储
- 如果配置项不存在,`im.*` 开头的配置会自动创建到 `system_configs` 表
- 更新后会自动清除配置缓存

---

## 二、系统配置

### 3. 获取系统配置列表
**接口**: `GET /api/v1/admin/system-configs`

**描述**: 获取所有系统配置项,包含完整的配置信息。

**认证**: 需要管理员 Token

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "configs": [
      {
        "id": 1,
        "key": "booking.intent_fee",
        "value": "500",
        "type": "number",
        "description": "意向金金额（元）",
        "editable": true,
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T15:30:00Z"
      },
      {
        "id": 2,
        "key": "im.tencent_enabled",
        "value": "true",
        "type": "boolean",
        "description": "是否启用腾讯云IM",
        "editable": true,
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T16:00:00Z"
      }
    ],
    "count": 2
  }
}
```

**字段说明**:
- `key`: 配置键,使用点号分隔的命名空间
- `value`: 配置值（字符串格式）
- `type`: 配置类型（string/number/boolean/json）
- `description`: 配置说明
- `editable`: 是否允许后台修改

**接口**: `PUT /api/v1/admin/system-configs/:key`

**描述**: 更新指定的系统配置项。

**认证**: 需要管理员 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |

**请求参数**:
```json
{

  "description": "意向金金额（元）- 已调整"

  "data": null
}
```

- 只能更新 `editable: true` 的配置项

| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码,默认 1 |
| pageSize | int | 否 | 每页数量,默认 10 |
| categoryCode | string | 否 | 分类代码筛选 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [

        "label": "公寓",
        "description": "普通住宅公寓",
        "sortOrder": 1,
        "enabled": true,
        "extraData": {
          "icon": "apartment"
        },
        "parentValue": "",
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T15:00:00Z"
      },
      {
        "id": 2,
        "categoryCode": "house_type",
        "value": "villa",
        "label": "别墅",
        "description": "独栋或联排别墅",
        "sortOrder": 2,
        "enabled": true,
        "extraData": {
          "icon": "villa"

      }
    ],
    "total": 25,

    "pageSize": 10

```

**字段说明**:
- `categoryCode`: 字典分类代码
- `value`: 字典值（用于程序逻辑）

## 四、区域管理

### 14. 区域列表
**接口**: `GET /api/v1/admin/regions`

**描述**: 获取行政区划列表,支持分页和层级筛选。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码,默认 1 |
| pageSize | int | 否 | 每页数量,默认 20 |
| level | int | 否 | 层级筛选：1-省,2-市,3-区/县 |
| parentCode | string | 否 | 父级区域代码 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "code": "110000",
        "name": "北京市",
        "level": 1,
        "parentCode": "",
        "sortOrder": 1,
        "enabled": true,
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T15:00:00Z"
      },
      {
        "id": 2,
        "code": "110100",
        "name": "北京市",
        "level": 2,
        "parentCode": "110000",
        "sortOrder": 1,
        "enabled": true,
        "createdAt": "2026-01-20T10:00:00Z",
        "updatedAt": "2026-01-25T15:00:00Z"
      }
    ],
    "total": 3500
  }
}
```

**字段说明**:
- `code`: 区域代码（国标行政区划代码）
- `name`: 区域名称
- `level`: 层级（1-省,2-市,3-区/县）
- `parentCode`: 父级区域代码
- `sortOrder`: 排序顺序
- `enabled`: 是否启用

### 16. 启用/禁用区域
**接口**: `PUT /api/v1/admin/regions/:id/toggle`

**描述**: 启用或禁用指定区域,支持级联操作（会同时更新所有子区域）。

**认证**: 需要管理员 Token

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | uint64 | 是 | 区域ID |

**请求参数**:
```json
{
  "enabled": false
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**特殊说明**:
- 禁用省级区域会级联禁用该省下所有市、区/县
- 禁用市级区域会级联禁用该市下所有区/县
- 启用操作同样会级联启用所有子区域

## 五、操作日志

### 17. 操作日志列表
**接口**: `GET /api/v1/admin/operation-logs`

**描述**: 获取管理员操作日志,支持分页和筛选。

**认证**: 需要管理员 Token

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | int | 否 | 页码,默认 1 |
| pageSize | int | 否 | 每页数量,默认 10 |
| adminId | uint64 | 否 | 管理员ID筛选 |
| action | string | 否 | 操作类型关键词筛选 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1001,
        "adminId": 1,
        "adminName": "超级管理员",
        "action": "更新系统配置",
        "targetType": "system_config",
        "targetId": 5,
        "ip": "192.168.1.100",
        "createdAt": "2026-01-25 16:30:00"
      },
      {
        "id": 1000,
        "adminId": 2,
        "adminName": "运营管理员",
        "action": "创建字典值",
        "targetType": "dictionary",
        "targetId": 26,
        "ip": "192.168.1.101",
        "createdAt": "2026-01-25 15:45:00"
      }
    ],
    "total": 1523
  }
}
```

**字段说明**:
- `adminId`: 操作管理员ID
- `adminName`: 操作管理员名称
- `action`: 操作类型描述
- `targetType`: 操作目标类型（对应数据库表或资源类型）
- `targetId`: 操作目标ID
- `ip`: 操作IP地址
- `createdAt`: 操作时间

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法 |
| 401 | 认证失败 | Token 过期或无效 |
| 403 | 权限不足 | 无管理员权限 |
| 404 | 资源不存在 | 配置项、字典或区域不存在 |
| 500 | 服务器错误 | 内部错误 |

## 最佳实践

### 配置管理
1. **配置分类**: 使用命名空间组织配置项（如 `booking.*`、`fee.*`、`im.*`）
2. **类型化**: 明确配置类型,前端根据类型进行验证和转换
3. **缓存策略**: 配置更新后自动清除缓存,确保实时生效
4. **权限控制**: 敏感配置（如密钥）应设置 `editable: false`

### 字典管理
1. **分类规范**: 使用清晰的分类代码（如 `house_type`、`decoration_style`）
2. **级联字典**: 使用 `parentValue` 实现级联选择
3. **扩展数据**: 使用 `extraData` 存储额外信息（如图标、颜色）
4. **排序控制**: 使用 `sortOrder` 控制显示顺序

### 区域管理
1. **三级联动**: 省-市-区三级联动选择
2. **懒加载**: 使用 `GET /regions/children/:parentCode` 实现懒加载
3. **级联操作**: 禁用上级区域会自动禁用所有下级区域
4. **国标代码**: 使用国家标准行政区划代码

### 操作日志
1. **审计追踪**: 记录所有敏感操作
2. **定期归档**: 定期归档历史日志
3. **异常监控**: 监控异常操作和高频操作
