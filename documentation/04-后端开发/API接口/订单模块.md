# 订单模块 API

**最后更新**: 2026-01-25

## 概述
订单模块负责处理用户的订单创建、支付、取消等操作。系统支持三种订单类型：意向金订单、设计费订单和施工款订单。订单支付支持微信支付和支付宝两种方式，施工款订单支持分期付款。

## 订单类型说明
- **意向金订单** (`intent_fee`): 用户预约服务商时支付的意向金
- **设计费订单** (`design`): 用户确认设计方案后生成的设计费订单
- **施工款订单** (`construction`): 项目施工阶段的施工费用订单

## 订单状态流转
```
pending (待支付) → paid (已支付) → completed (已完成)
                 ↘ cancelled (已取消)
                 ↘ refunded (已退款)
```

**状态码说明**:
- `0` - pending: 待支付
- `1` - paid: 已支付
- `2` - cancelled: 已取消
- `3` - refunded: 已退款

## 支付方式
- `wechat` - 微信支付
- `alipay` - 支付宝

## API 列表

### 1. 获取待支付列表
**接口**: `GET /api/v1/orders/pending-payments`

**描述**: 获取当前用户所有待支付的订单，包括未付意向金的预约和未付设计费的订单。列表按创建时间倒序排列。

**认证**: 需要 Bearer Token

**请求参数**: 无

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "type": "intent_fee",
        "id": 123,
        "orderNo": "BK00000123",
        "amount": 500.00,
        "providerId": 456,
        "providerName": "张设计师",
        "address": "北京市朝阳区某小区3号楼101",
        "expireAt": null,
        "createdAt": "2026-01-20T10:30:00Z"
      },
      {
        "type": "design_fee",
        "id": 789,
        "orderNo": "OD20260120001",
        "amount": 4500.00,
        "providerId": 456,
        "providerName": "张设计师",
        "address": "",
        "expireAt": "2026-01-22T10:30:00Z",
        "createdAt": "2026-01-20T10:30:00Z"
      }
    ],
    "total": 2
  }
}
```

**字段说明**:
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| type | string | 订单类型：`intent_fee`（意向金）、`design_fee`（设计费）、`construction_fee`（施工款） |
| id | uint64 | 订单ID（意向金为 Booking ID，设计费为 Order ID） |
| orderNo | string | 订单号（意向金格式：BK + 8位数字，设计费格式：OD + 日期 + 序号） |
| amount | float64 | 应付金额（设计费已扣除意向金抵扣） |
| providerId | uint64 | 服务商ID |
| providerName | string | 服务商名称（优先显示昵称，无昵称则显示公司名） |
| address | string | 装修地址（仅意向金订单有值） |
| expireAt | string/null | 支付过期时间（设计费订单为48小时，意向金无过期时间） |
| createdAt | string | 订单创建时间 |

### 2. 获取订单详情
**接口**: `GET /api/v1/orders/:id`

**描述**: 获取指定订单的详细信息。系统会验证订单归属权限，只有订单所属用户才能查看。

**认证**: 需要 Bearer Token

**请求参数**:
| 参数名 | 类型 | 位置 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| id | uint64 | path | 是 | 订单ID |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 789,
    "projectId": 0,
    "proposalId": 456,
    "bookingId": 123,
    "orderNo": "OD20260120001",
    "orderType": "design",
    "totalAmount": 5000.00,
    "paidAmount": 0.00,
    "discount": 500.00,
    "status": 0,
    "expireAt": "2026-01-22T10:30:00Z",
    "paidAt": null,
    "createdAt": "2026-01-20T10:30:00Z",
    "updatedAt": "2026-01-20T10:30:00Z"
  }
}
```

**字段说明**:
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | uint64 | 订单ID |
| projectId | uint64 | 项目ID（设计费订单为0，施工款订单有值） |
| proposalId | uint64 | 关联的设计方案ID |
| bookingId | uint64 | 关联的预约ID（用于追踪意向金） |
| orderNo | string | 订单号 |
| orderType | string | 订单类型：`design`、`construction`、`material` |
| totalAmount | float64 | 订单总金额 |
| paidAmount | float64 | 已支付金额 |
| discount | float64 | 优惠金额（意向金抵扣） |
| status | int8 | 订单状态：0-待支付，1-已支付，2-已取消，3-已退款 |
| expireAt | string/null | 支付过期时间 |
| paidAt | string/null | 支付完成时间 |

**错误响应**:
```json
{
  "code": 404,
  "message": "订单不存在"
}
```

```json
{
  "code": 403,
  "message": "无权查看此订单"
}
```

### 3. 支付订单
**接口**: `POST /api/v1/orders/:id/pay`

**描述**: 支付指定订单。支付成功后订单状态变更为"已支付"，并记录支付时间。

**认证**: 需要 Bearer Token

**请求参数**:
| 参数名 | 类型 | 位置 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| id | uint64 | path | 是 | 订单ID |

**请求体**: 无（实际支付逻辑在 service 层处理）

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 789,
    "orderNo": "OD20260120001",
    "orderType": "design",
    "totalAmount": 5000.00,
    "paidAmount": 4500.00,
    "discount": 500.00,
    "status": 1,
    "paidAt": "2026-01-20T14:30:00Z",
    "createdAt": "2026-01-20T10:30:00Z",
    "updatedAt": "2026-01-20T14:30:00Z"
  }
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "订单不存在或已支付"
}
```

```json
{
  "code": 400,
  "message": "订单已过期"
}
```

```json
{
  "code": 400,
  "message": "无权支付此订单"
}
```

### 4. 取消订单
**接口**: `DELETE /api/v1/orders/:id`

**描述**: 取消待支付的订单。只有待支付状态的订单才能被取消，已支付或已取消的订单无法再次取消。

**认证**: 需要 Bearer Token

**请求参数**:
| 参数名 | 类型 | 位置 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| id | uint64 | path | 是 | 订单ID |

**请求体**: 无

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "订单已取消"
  }
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "订单不存在"
}
```

```json
{
  "code": 400,
  "message": "订单状态不允许取消"
}
```

```json
{
  "code": 400,
  "message": "无权取消此订单"
}
```

### 5. 支付分期款项
**接口**: `POST /api/v1/orders/plans/:planId/pay`

**描述**: 支付施工款订单的分期款项。施工款订单支持按里程碑分期支付（如开工款、水电款、泥木款、竣工款等）。

**认证**: 需要 Bearer Token

**请求参数**:
| 参数名 | 类型 | 位置 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| planId | uint64 | path | 是 | 支付计划ID |

**请求体**: 无（实际支付逻辑在 service 层处理）

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 101,
    "orderId": 789,
    "type": "milestone",
    "seq": 1,
    "name": "开工款",
    "amount": 15000.00,
    "percentage": 30.0,
    "status": 1,
    "dueAt": "2026-02-01T00:00:00Z",
    "paidAt": "2026-01-25T10:30:00Z",
    "createdAt": "2026-01-20T10:30:00Z",
    "updatedAt": "2026-01-25T10:30:00Z"
  }
}
```

**字段说明**:
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | uint64 | 支付计划ID |
| orderId | uint64 | 关联的订单ID |
| type | string | 支付类型：`milestone`（里程碑）、`onetime`（一次性） |
| seq | int | 期数顺序（第几期） |
| name | string | 款项名称（如"开工款"、"水电款"） |
| amount | float64 | 本期应付金额 |
| percentage | float32 | 占总金额的百分比 |
| status | int8 | 支付状态：0-待支付，1-已支付 |
| dueAt | string/null | 应付日期 |
| paidAt | string/null | 实际支付时间 |

**错误响应**:
```json
{
  "code": 400,
  "message": "支付计划不存在"
}
```

```json
{
  "code": 400,
  "message": "该期款项已支付"
}
```

```json
{
  "code": 400,
  "message": "无权支付此款项"
}
```

## 业务规则说明

### 订单过期时间
- **意向金订单**: 无过期时间限制
- **设计费订单**: 创建后48小时内必须完成支付，否则订单自动过期
- **施工款订单**: 按分期计划的应付日期（dueAt）为准

### 意向金抵扣规则
- 用户支付意向金后，确认设计方案时会自动生成设计费订单
- 设计费订单的 `discount` 字段记录意向金抵扣金额
- 实际应付金额 = `totalAmount` - `discount`

### 分期支付规则
- 施工款订单支持按里程碑分期支付
- 常见分期方案：开工款（30%）、水电款（20%）、泥木款（20%）、竣工款（30%）
- 每期款项按顺序支付，前一期未支付时后续期数无法支付

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法或业务规则校验失败 |
| 401 | 认证失败 | Token 过期或无效 |
| 403 | 权限不足 | 无权访问该订单 |
| 404 | 资源不存在 | 订单或支付计划不存在 |
| 500 | 服务器错误 | 系统内部错误 |

## 使用示例

### 获取待支付列表 (Curl)
```bash
curl -X GET http://localhost:8080/api/v1/orders/pending-payments \
  -H "Authorization: Bearer <your_token_here>"
```

### 支付订单 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/orders/789/pay \
  -H "Authorization: Bearer <your_token_here>"
```

### 取消订单 (Curl)
```bash
curl -X DELETE http://localhost:8080/api/v1/orders/789 \
  -H "Authorization: Bearer <your_token_here>"
```

### 支付分期款项 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/orders/plans/101/pay \
  -H "Authorization: Bearer <your_token_here>"
```
