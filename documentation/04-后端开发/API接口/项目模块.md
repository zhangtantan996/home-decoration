# 项目模块 API

**最后更新**: 2026-01-25

## 概述
项目模块是平台的核心业务流程载体，涵盖了从项目发布、施工管理、阶段验收预测到资金托管的全生命周期。系统采用**里程碑式托管支付**模型，业主预存资金到平台，待各施工节点验收通过后分批释放给服务商，确保双方权益。

---

## API 列表

### 项目管理

#### 1. 发布项目
**接口**: `POST /api/v1/projects`
**描述**: 业主在确认设计方案或达成意向后，正式发起一个装修项目。
**认证**: 需要 JWT Token (业主)

**请求参数**:
```json
{
    "name": "浦东新区120平米全屋装修",
    "providerId": 1,
    "address": "上海市浦东新区张江高科技园区",
    "area": 120,
    "budget": 150000,
    "startDate": "2025-03-01",
    "expectedEndDate": "2025-06-30",
    "description": "现代简约风格，含水电改造及软装"
}
```

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "id": 1,
        "name": "浦东新区120平米全屋装修",
        "status": "pending"
    }
}
```

#### 2. 项目列表
**接口**: `GET /api/v1/projects`
**描述**: 分页获取当前用户的项目列表，支持状态筛选。
**认证**: 需要 JWT Token

**Query 参数**:
- `status`: 项目状态筛选 (pending/in_progress/completed/archived)
- `page`: 页码（默认 1）
- `pageSize`: 每页数量（默认 20）

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "items": [
            {
                "id": 1,
                "name": "浦东新区120平米全屋装修",
                "providerName": "XX装修公司",
                "status": "in_progress",
                "progress": 35,
                "currentPhase": "水电改造",
                "startDate": "2025-03-01"
            }
        ],
        "total": 1
    }
}
```

#### 3. 项目详情
**接口**: `GET /api/v1/projects/:id`
**描述**: 获取项目的全量信息，包括基本属性、当前阶段、里程碑节点及施工队伍信息。
**认证**: 需要 JWT Token

**响应数据**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "id": 1,
        "name": "浦东新区120平米全屋装修",
        "status": "in_progress",
        "currentPhase": "水电改造",
        "progress": 35,
        "phases": [
            { "id": 1, "name": "拆除与清理", "status": "completed" },
            { "id": 2, "name": "水电改造", "status": "in_progress" }
        ],
        "milestones": [...]
    }
}
```

#### 4. 更新状态
**接口**: `PUT /api/v1/projects/:id/status` (管理端) 或 `PUT /api/v1/projects/:id` (普通更新)
**描述**: 更新项目生命周期状态。
**认证**: 需要 JWT Token

**请求参数**:
```json
{
    "status": "in_progress"
}
```

### 资金托管

#### 1. 资金托管 (充值)
**接口**: `POST /api/v1/projects/:id/deposit` 或 `POST /api/v1/escrow/deposit`
**描述**: 业主为特定施工阶段存入托管资金。资金将由平台暂管并处于冻结状态。
**认证**: 需要 JWT Token (业主)

**请求参数**:
```json
{
    "amount": 30000,
    "milestoneId": 1,
    "paymentMethod": "wechat"
}
```

**响应数据**:
```json
{
    "code": 0,
    "message": "充值成功",
    "data": {
        "transactionId": "TXN20250301001",
        "newBalance": 30000
    }
}
```

#### 2. 节点验收
**接口**: `POST /api/v1/projects/:id/accept` 或 `POST /api/v1/escrow/milestone`
**描述**: 业主确认特定里程碑节点施工合格。验收通过后，对应的托管资金将变为可释放状态。
**认证**: 需要 JWT Token (业主)

**请求参数**:
```json
{
    "milestoneId": 1,
    "rating": 5,
    "comment": "做工细致，准时交付"
}
```

#### 3. 资金释放
**接口**: `POST /api/v1/projects/:id/release` 或 `POST /api/v1/escrow/release`
**描述**: 将托管账户中对应的里程碑款项打入服务商账户。
**认证**: 需要 JWT Token

**请求参数**:
```json
{
    "milestoneId": 1
}
```

**响应数据**:
```json
{
    "code": 0,
    "message": "资金释放成功",
    "data": {
        "milestoneId": 1,
        "releasedAmount": 30000
    }
}
```

## 使用示例

### 发布并初始化项目
```bash
curl -X POST http://localhost:8080/api/v1/projects \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "新家装修项目",
    "providerId": 5,
    "budget": 200000,
    "address": "上海市静安区..."
  }'
```

### 业主存入托管资金
```bash
curl -X POST http://localhost:8080/api/v1/projects/1/deposit \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 50000,
    "milestoneId": 1,
    "paymentMethod": "wechat"
  }'
```

### 完成节点验收
```bash
curl -X POST http://localhost:8080/api/v1/projects/1/accept \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "milestoneId": 1,
    "rating": 5,
    "comment": "验收通过"
  }'
```
