# 认证模块 API

**最后更新**: 2026-01-25

## 概述
认证模块负责处理用户身份验证相关的操作，包括注册、登录、发送验证码、Token 刷新以及第三方登录（微信小程序）。系统采用 JWT（JSON Web Token）进行状态无无关的认证。

## 认证流程
1. 客户端通过手机号和验证码或密码发起登录请求。
2. 后端验证成功后，返回 `token`（Access Token）和 `refreshToken`。
3. 客户端在后续请求的 Header 中携带 `Authorization: Bearer <token>`。
4. `token` 过期后，客户端使用 `refreshToken` 调用刷新接口获取新的 `token`。

## API 列表

### 1. 发送验证码
**接口**: `POST /api/v1/auth/send-code`

**描述**: 向指定手机号发送短信验证码。

**请求参数**:
```json
{
  "phone": "13800138000"
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "验证码已发送",
  "data": {
    "message": "测试验证码: 123456"
  }
}
```

---

### 2. 用户注册
**接口**: `POST /api/v1/auth/register`

**描述**: 使用手机号和验证码进行用户注册。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| phone | string | 是 | 11位手机号 |
| code | string | 是 | 验证码 |
| password | string | 否 | 登录密码（需符合复杂度要求） |
| nickname | string | 否 | 用户昵称 |
| userType | int8 | 否 | 用户类型：1-业主（默认），2-服务商，3-工人 |

**响应示例**:
```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 7200,
    "user": {
      "id": 123,
      "phone": "13800138000",
      "nickname": "张三",
      "avatar": "http://example.com/avatar.jpg",
      "userType": 1
    }
  }
}
```

---

### 3. 用户登录
**接口**: `POST /api/v1/auth/login`

**描述**: 支持验证码登录和密码登录。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| phone | string | 是 | 11位手机号 |
| type | string | 否 | 登录方式：`code`（默认）或 `password` |
| code | string | 条件 | 验证码登录时必填 |
| password | string | 条件 | 密码登录时必填 |

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 7200,
    "user": {
      "id": 123,
      "phone": "13800138000",
      "nickname": "张三",
      "avatar": "http://example.com/avatar.jpg",
      "userType": 1
    }
  }
}
```

---

### 4. Token 刷新
**接口**: `POST /api/v1/auth/refresh`

**描述**: 使用 `refreshToken` 换取新的 `token`。

**请求参数**:
```json
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
}
```

**响应示例**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "新的 Access Token",
    "refreshToken": "新的 Refresh Token",
    "expiresIn": 7200
  }
}
```

---

### 5. 微信小程序登录
**接口**: `POST /api/v1/auth/wechat/mini/login`

**描述**: 微信小程序一键登录，通过 `wx.login` 获取的 code。

**请求参数**:
```json
{
  "code": "微信登录 code"
}
```

**响应示例（已绑定手机号）**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "...",
    "user": { ... }
  }
}
```

---

### 6. 用户登出
**接口**: `POST /api/v1/auth/logout` (建议)

**描述**: 客户端清除本地存储的 Token 即可。后端目前不强制销毁 Token，如需实现黑名单功能，可通过 Redis 存储已失效的 Token。

---

## 错误码说明

| 错误码 | 说明 | 备注 |
| :--- | :--- | :--- |
| 400 | 参数错误 | 请求参数不合法 |
| 401 | 认证失败 | Token 过期或无效 |
| 1001 | 验证码错误 | 验证码不匹配或已过期 |
| 1002 | 用户已存在 | 该手机号已被注册 |
| 1003 | 密码错误 | 手机号或密码不正确 |
| 1004 | 账号已锁定 | 登录失败次数过多，需等待一段时间 |
| 1005 | 账号已禁用 | 账号违反规定被平台封禁 |

## 使用示例

### 登录请求 (Curl)
```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "13800138000",
    "type": "password",
    "password": "Password123!"
  }'
```

### 携带 Token 的请求
```bash
curl -X GET http://localhost:8080/api/v1/user/profile \
  -H "Authorization: Bearer <your_token_here>"
```
